平台变体 — Web 浏览器 / 手机 native

同一套设计，平台差异：Web 浏览器先自动连本机 daemon（同桌面走 S2），连不上才落选择器、无扫码；手机无本机 daemon → 自动连失败兜底到选择器、扫码优先。各屏当前主机+状态由主文案表达（不显端口）。

平台差异
  1. W
    Web 浏览器：「开始使用」先进 S2 自动连本机 daemon（经 ws 探本机，不显端口，过场主文案「正在连接本机 daemon…」）→ 连上落主页；连不上才落选择器（兜底 · 无扫码 · 带「重试本地」· 副标题转「没检测到本地 daemon」）。一次性品牌欢迎仍保留（首次访问）。
  2. M
    手机 native：本机无 daemon、自动连探不到 → 兜底到选择器（纯远程），扫码连电脑主机为主 CTA；粘链/直连为次。连上后即远程遥控本机代理（relay E2E）。
  3. D
    桌面 Electron：directPipe 自动连本机 → 走 S1→S2 主路径（与浏览器同路径，仅 transport 不同），连不上落 S4 错误态。
  4. R
    重开自愈（全平台）：已看过欢迎/老用户后续启动不显欢迎→静默重试本机一次→连上进主页，否则落 picker（桌面落 S4）；「连接远程」是会话级意图、不跨重启，重开仍先重试本机。
