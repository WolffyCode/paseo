## 1. 目标 & 用户价值

定义 Helm 在「主壳左下『设置』入口点开后」进入的**设置界面**——一个 Codex 式干净的 **master-detail 设置控制台**：左侧 section 导航分「应用」「主机」两段，右侧详情面随选中 section 切换。

- 用户为什么需要：
  - **配通一条能用的链路**：选哪个 CLI 提供方、接哪个中转站（API 供应商）、放出哪些模型——三层选型的深度管理一处搞定。
  - **主机是根上下文**：连接与配对（含手机远程）/ 守护进程 / 模型与提供方 / 用量，全部挂在「当前选中的主机」下，**主机配置是设置的一等公民**；主机段默认进**当前主机**（有明确「当前」标识）。
  - **给 agent 装能力**（插件 = MCP / 技能 / 目录）：**本轮延后保留、隐藏**，后续回归。
  - **调教这台设备的 App**：外观、快捷键、语言、本机守护进程、权限、关于与更新。

主壳（项目①）只露「设置」入口，级联里只露「管理中转站…」入口；**所有深度配置都在本设置界面**。本设置是主壳明确推迟划出的模块。

## 2. 范围

**做（完整设置界面 · 11 个 tab：应用 8 + 主机 3 · 两级 IA · 全交互态 · 桌面）**

- **设置外壳**：master-detail；左导航两段（应用 App 级 / 主机 主机级）+ 主机段头 **host 选择器**（默认当前主机 + 「当前」标识）；进入 / 退出 / 深链 / 默认落点 / 外壳级状态（加载 / 空 / 错误 / 主机离线）。
- **应用段（8 tab，跟设备走 · 本轮不动）**：通用 · 外观 · 快捷键 · 守护进程 · 集成 · 权限 · 诊断 · 关于。
- **主机段（3 tab，跟所选主机走 · 本轮重构）**：① 主机（连接与配对 + 主机运维合并）· ② 模型与提供方（provider-centric 三级下钻）· ③ 用量。
- **全交互态**：默认 / 悬浮 / 活跃·选中 / 运行 / 禁用 / 空 / 加载 / 错误 / 离线 + 右键菜单（提供方行、中转站行、连接行等，逐项列全）+ 即时持久化语义。

**不做（删除 / 延后 / 划走）**

- **删除**：Agents tab（主壳已覆盖，无价值）· 工作区 tab（主壳已能操作工作区）· 终端 tab（其命令配置归入提供方「命令」字段；PTY 终端 profiles 归属见开放项）。
- **延后保留、本轮隐藏**：插件（MCP / 技能 / 目录）——三块也重要，本轮不进主机 nav，设计稿留作附录，后续回归（**内容不删**）。
- **手机 / 紧凑态**：本轮桌面 only，紧凑态后续统一适配（只在注解里标差异）。
- **添加主机流程**：host 选择器只「切换已连主机」；添加主机归主壳 / onboarding，设置内只在配对子流程露二维码生成。
- **主壳工作视角**：对话、级联选择器本身、主壳侧栏目录 / 对话列表——设置只做「配置 / 管理」视角，不重做主壳实时工作界面。

## 3. 全局 IA 与设置外壳

### 3.1 结构

单一 master-detail（左导航 + 右详情）。左导航分两段：

- **段 A · 应用（App 级）** —— 跟当前设备 / App 走，对所有已连主机生效。
- **段 B · 主机（主机级）** —— 段头一个 **host 选择器**（当前 active 主机 + 在线状态色 + **「当前」标识** + 展开），下方 **3 个 tab**（主机 / 模型与提供方 / 用量）跟「所选主机」走；切主机后右详情面重载该主机数据，**默认进当前主机**。host 一等公民由整段 B 承载。

### 3.2 进入 / 退出 / 深链

- **进入**（3 个入口）：① 主壳左下「设置」→ 落「通用」（或记忆上次 section）；② 主壳三层级联的「管理中转站…」→ **深链**直达主机段「模型与提供方」tab 对应提供方 / 中转站；③ onboarding 本机 daemon 连接失败的「查看诊断」→ 落应用段「守护进程」/「诊断」（具体落点见开放项）。
- **退出**：返回主壳（顶部「返回主壳」或 Esc）。
- 一次只一个 section active；切 section 右详情面整块替换。

### 3.3 host 选择器

- 段 B 段头：当前主机名 + 在线 / 连接中 / 离线状态点 + relay/本地标识 + 展开下拉（列已连主机 + 当前项✓）。
- 选另一台 → 段 B 所有 tab 的右详情重载该主机数据。**不负责添加主机**（添加归主壳/onboarding）。
- 离线主机：状态点转红，仍可进入查看缓存，可改项禁用。

### 3.4 外壳级状态

- **加载**：切 section / 切主机 → 右详情面页级 spinner；行级数据各自 inline spinner。
- **空**：无已连主机 → 段 B 显「未连接主机」占位 +「去连接」（回主壳）；段 B tab 不可进。
- **错误**：主机级数据拉取失败 → 详情面顶 Alert + 重试。
- **离线**：所选主机离线 → host 选择器红点 + 详情面顶 callout（主机离线 + 重新连接）；只读项显缓存值、可改项禁用。
- **即时持久化**：所有控件改动**立即生效落盘**（App 级写设备本地、主机级写该主机配置），无独立「保存」按钮；输入类失焦/回车提交。

UI：`ui.html#shell`

## 4. tab 清单与每 tab WHAT

> 每 tab：作用 / 子功能 / 关键流程 / 状态要点（全态见 ui.html）/ 边界 / UI 锚点。状态命名统一：默认·悬浮·活跃·运行·禁用·空·加载·错误·离线。

### 段 A · 应用（App 级，跟设备走）

#### A1 通用（General）
- 作用：App 全局基础行为。
- 子功能：回车发送行为（排队 / 中断）、界面语言、服务 URL 打开方式（内置浏览器 / 系统浏览器）、终端回滚行数。
- 流程：改任一项 → 立即生效（语言切换即时重渲）。
- 边界：全为真实已有项；纯设备级偏好。
- UI：`ui.html#shell`（通用即设置外壳的金标准详情面）

#### A2 外观（Appearance）
- 作用：视觉呈现偏好。
- 子功能：主题（**下拉**，含浅 / 深 / 跟随系统及暗色变体；Helm 品牌方向浅色）、UI 字体族、代码字体族、UI 字号（11–24）、代码字号（9–22）、语法高亮方案、工作区标题来源（标题 / 分支）；带一块代码预览卡实时反映。
- 流程：改任一项 → 预览卡与全局即时反映。
- 状态：字体族空 = placeholder「系统默认」；预览卡渲染 diff 增删行。
- 边界：主题用下拉（保留全部变体，不收敛为三段——见开放项）；"工作区标题来源"与主壳侧栏显示偏好为**同一偏好、双向同步**（非两个真相源）。
- UI：`ui.html#ap1`

#### A3 快捷键（Keyboard shortcuts）
- 作用：查看 / 覆盖快捷键（桌面）。
- 子功能：按组列出动作 + 当前键位 + 捕获新键位 / 重置单项 / 全部重置。
- 流程：点动作键位 → 进「按下快捷键…」捕获态 → 按键提交 / Esc 取消；重置回默认。
- 状态：捕获中高亮；已覆盖项显「重置」。
- 边界：本轮不做冲突消解 UI（见开放项）；非桌面置灰。
- UI：`ui.html#ap2`

#### A4 守护进程（本机内建 daemon · 设备级）
- 作用：管理**本机内建守护进程**（桌面 Electron 内建，设备级）的生命周期。
- 子功能：状态（运行中 / 未运行 / 启动中 / 错误）+ 版本、随 Helm 启动开关、退出后保持运行开关、日志文件入口（复制路径 / 打开）、完整状态查看。
- 流程：切「随 Helm 启动」= 托管启停；停止走破坏性确认（连接的 agents/terminals 会停）。
- 边界：**双 daemon 铁律**——本 tab 只管「本机内建 daemon」，与主机段「主机」tab 的「所选 host 的 daemon」严格区分；不做 OS 级开机自启（见开放项）；不在此重启远程主机。桌面专属。
- UI：`ui.html#rt1`

#### A5 集成（Integrations · 本机命令行工具）
- 作用：本机命令行工具的安装 / 更新 / 卸载（桌面）。
- 子功能：列出可用 CLI runtime（Helm CLI + 各 agent CLI：Claude Code / Codex / Copilot / OpenCode / Pi 等），每行 图标 + 名称 + 已装版本 / 未安装 / 失败 + 安装 / 更新 / 卸载 + 行级安装进度。
- 流程：点安装 → 行级进度 → 完成显版本；失败显错误 + 重试。
- 边界：真实已有 = Helm CLI 安装（二元已装态）；**agent runtime 的「版本 / 更新 / 卸载」管理面为 Helm 新造方向**（见开放项），可用性检测可复用既有 provider 可用性能力；Skill 已迁至主机段「插件」tab，本 tab 不含 Skill。桌面专属。
- UI：`ui.html#rt2`

#### A6 权限（Permissions）
- 作用：App 系统权限（桌面 Electron 原生）。
- 子功能：列出权限（**真实 = 通知 + 麦克风**；摄像头 / 辅助功能为按需扩展，见开放项），每项 状态 badge（已授权 / 已拒绝 / 待请求 / 不可用）+ 去授予 / 打开系统设置 + 刷新。
- 流程：点请求 → 系统弹权限 → 状态更新；受限项引导「打开系统隐私设置」。
- 边界：只反映 / 引导系统权限，不代授予。桌面专属。
- UI：`ui.html#rt3`

#### A7 诊断（Diagnostics）
- 作用：本地排障集中处。
- 子功能：音频播放测试（thinking tone）、查看 daemon 日志入口、（轻量）连接自检。
- 流程：点测试 → 播放并报结果；点日志 → 打开 daemon 日志。
- 边界：**不重复实现** daemon 日志（其能力归守护进程 tab，诊断只复用入口）；「连接自检」为薄能力、是否保留见开放项；onboarding 失败「查看诊断」的落点见开放项。
- UI：`ui.html#ap3`

#### A8 关于（About）
- 作用：版本与更新。
- 子功能：Helm 品牌头 + App 版本、已连主机 daemon 版本（不一致高亮 / 离线）、检查 / 安装更新（桌面 updater）、发布频道（stable / beta）、社区链接。
- 流程：自动静默检查 → 有更新显提示 → 确认 → 安装中 → 重启生效；切频道改更新来源。
- 边界：不内嵌 changelog 浏览；不在关于内升级主机 daemon（那归主机段）。
- UI：`ui.html#ap4`

### 段 B · 主机（主机级，跟所选主机走 · 本轮重构为 3 tab）

> 主机段默认进**当前主机**（host 选择器带「当前」标识）。原 8 tab 重构为 3 tab：主机 / 模型与提供方 / 用量。

#### B1 主机（Host · 连接与运维合并）
- 作用：所选主机的连接、配对、运维——原「连接与配对」+「主机」**合并为一个 tab**。
- 子功能：
  - **顶部一处身份**：主机名 + 在线状态点 + daemon 版本 + 「当前主机」标识 + 重命名（**仅顶部一处，不再重复**）。
  - **这台主机**：健康概览（= 诊断：状态 / 活动连接类型 / 版本 / 最近错误）· 重启守护进程 · 服务代理（service-proxy 折叠高级：localhost 始终可用 + 公开 baseUrl + listen + 文档入口）· 危险操作（移除主机）。
  - **连接与配对**：连接列表（**relay 远程 E2E / 本地直连 / TCP** + 实时延迟 + 活动标识 + 逐条移除）· relay 中继配置（启用 / endpoint / TLS / 公开 endpoint / 公开 TLS）· 手机配对（生成 / 重新生成二维码 + `#offer=` 链接，relay E2E，"链接像密码"提示）。
- 流程：配 relay → 即时落盘；配对 → 展开二维码 / 链接 → 手机扫码接入；重启 → 确认 → 重连；移除主机 → 危险确认 → 回主壳。
- 状态：**离线态**——host 选择器红点 + 顶部 callout；缓存连接显延迟超时；重启 / relay / service-proxy / 配对禁用，重命名 / 移除仍可用。
- 边界：**术语铁律** relay = E2E 远程通道 **≠ 中转站 vendor**；配对只「生成 offer」，**设备列表 / 吊销不做**（无持久设备身份，见开放项）；**双 daemon**——本 tab 管「所选 host 的 daemon」，与应用段「守护进程」（本机内建）区分；service-proxy 浅档（DNS / 反代不入应用）。
- UI：`ui.html#hm1`、`ui.html#hm1-pair`、`ui.html#hm1-offline`

#### B2 模型与提供方（Models & Providers · provider-centric 三级下钻 · 本轮重头）
- 作用：以**提供方为中心**的三级深度管理——提供方（CLI）→ 中转站（API 供应商 base_url+key）→ 放出模型。原「模型与中转站」vendor-centric 级联重做为 provider-centric 三级 drill-in（**三级都是页面、带面包屑返回，不用弹框**）。
- 结构：
  - **L1 提供方列表**：显示所有提供方，**已添加 / 已启用在上、未添加 / 未装在下**；每行 启用 / 关闭开关 + **该提供方 CLI 已装 / 未装状态**（不显版本号，见边界）+ 中转站数 / 模型数 + ⚙ 设置（→ L2）；顶部「+ 新增提供方」。**提供方可增删**（从🔒反转，按真实能力分型，见边界）。
  - **L2 提供方设置**（带返回 L1）：基础信息（名称 / 图标 / **命令**）+ 该提供方下的中转站配置列表（+ 新增中转站 / cc-switch 同步入口）+ 提供方管理（停用 / 删除）。
  - **L3 中转站详情**（drill-in 页，带返回 L2，**非弹框**）：base_url / key（密码态）/ 通用配置 JSON 编辑器 / 花费限额 + 倍率 / 端点测速 / **放出模型**（自动拉 + 手动加 + 多选放出 + 设默认 + thinking）/ 删除中转站。点中转站行或「新增中转站」都进此页。
- 流程：L1 启停 / 新增 / 删除提供方；进 L2 配命令 / 管中转站；进 L3 配 base_url+key → 测速 → 放出模型 → 设默认 → 设为当前；切 host / section → 栈重置回 L1；主壳级联「管理中转站…」深链直达 L2/L3。
- 状态：默认 / 悬浮 / 活跃 / 运行（测速中）/ 禁用（未装）/ 空（无中转站）/ 加载 / 错误（不健康 401 / 拉模型失败）/ 离线。
- **边界与能力门（董事长要求扎根真实，逐条诚实标注）**：
  - **删除提供方**：内置（claude/codex/copilot/opencode/pi/omp）**不能删、只能禁用**；自定义提供方**删除当前无后端 mutation 路径，需新增后端能力**（UI 可呈现删除、对内置不出现）。
  - **新增提供方**：① in-app 从 ACP 精选目录添加 = **真实**；② extends claude/codex 的自定义（Z.AI/Qwen/自定义二进制）= **in-app 编辑 UI 为 Helm 新造**（后端 schema + patch 已支持，现状只能手改配置文件 / 仅接通追加模型）。
  - **CLI 状态**：只显**已装 / 未装 + 模型数**，**无结构化版本号字段**（不显版本）。
  - **命令字段** = 提供方 agent CLI 的启动命令（真实）；**≠ 原终端 PTY profile**——终端 tab 删除后，**PTY 终端 profiles 的归属是开放项**（两套独立机制、数据模型不可合并）。
  - **花费限额 + 倍率** = 提供方配置无此字段、无内建核算，本轮至多「设值 + 进度占位」，超限自动行为（暂停 / 切换 / 阻断）不做。
  - 术语：中转站 vendor（base_url+key）**≠ relay 远程**；用量在「用量」tab、本 tab 只露深链。
- UI：`ui.html#mp1`（L1 提供方列表）、`ui.html#mp2`（L2 提供方设置）、`ui.html#mp3`（L3 中转站详情 + 模型放出）、`ui.html#mp4`（新增提供方：ACP 目录 / 自定义两路）、`ui.html#mp5`（空 / 错 / 状态合屏）

#### B3 用量（Usage）
- 作用：所选主机的用量监控（只读）——顶部概览卡 + 时间窗切换 + 每提供方 / 模型。
- 子功能：
  - 顶部 3–4 张概览卡：总用量 + 估算总金额 + 活跃提供方 + 最近窗口重置；右上角 **今天 / 本周 / 本月 / 全局** 切换。
  - 每提供方一块：余额（USD / credits / requests / tokens）+ 多窗口进度 + planLabel + 该提供方下**按具体模型的明细行**。
  - 顶部刷新 + 门控提示。
- 流程：进入 / 刷新 → 拉取渲染；切时间窗 → 重算。
- **边界与能力门（董事长要求扎根，三处缺口诚实标注 + 视觉双标"目标态 vs 现状"）**：
  - **时间切换（今天 / 本周 / 本月 / 全局）**：底层当前**只各家 API 固定窗口（Session / Weekly…）**，**用户时间切换需聚合能力**（标"目标态"）。
  - **估算总金额**：**无定价 / 成本计算逻辑**，金额只能聚合各家 API 已算好的余额（标"目标态·需定价能力"）。
  - **按模型明细**：真实只到**提供方维**，模型为各家窗口标签近似，**结构化按模型需新能力**（标"目标态"）。
  - **不到中转站**：使用中会切中转站、中转维不准，**移除中转站维**（正确）。
  - 真实可显：提供方维 + API 窗口（usedPct / resetsAt / runsOutAt / shortfallPct）+ 余额四单位 + planLabel；**依赖能力门 providerUsageList**（缺失显「更新主机」）。
- UI：`ui.html#us1`（用量总览）、`ui.html#us2`（门控 / 空 / 加载 / 离线态）

## 5. 核心流程（端到端 · 每条有入口有出口）

1. **配通一条模型链路（provider-centric 三级）**：主壳级联「管理中转站…」或设置「模型与提供方」→ L1 选 / 启用提供方 → L2 选 / 新建中转站 → L3 配 base_url + key → 测速 → 放出模型 + 设默认 → 设为当前 → 回主壳级联即生效。入口 = 级联 / 设置；出口 = 可在对话用该提供方 + 中转站 + 模型。
2. **新增 / 删除提供方**：L1「+ 新增提供方」→ 从 ACP 目录选 或 自定义（extends claude/codex + 命令 + base_url + key）→ 落为新提供方行；删除仅对自定义（内置只禁用，删除需新后端能力）。
3. **配对手机远程**：主机段「主机」→ 配对设备 → 展开二维码 / 链接 → 手机扫码（relay E2E）→ 手机接入遥控本机代理。
4. **看用量**：主机段「用量」→ 概览卡 + 切时间窗（目标态）+ 每提供方 / 模型；缺能力门时显「更新主机」。
5. **调外观**：应用段「外观」→ 改主题 / 字体 / 字号 → 预览卡 + 全局即时反映。
6. **切主机重载**：host 选择器 → 选另一台在线主机 → 段 B 3 tab 重载该主机数据，默认进当前主机。
7. **边界分叉**：无主机（段 B 占位 + 去连接）/ 加载（页级 / 行级 spinner）/ 错误（Alert + 重试）/ 主机离线（红点 + callout + 可改项禁用）/ CLI 未装（unavailable 行）/ 测速失败（不健康红字）/ 能力缺口（门控提示，不写降级）——每条有入口有出口、无死胡同。

## 6. 全局状态与规则

- **两级 scope**：应用段写设备本地、对所有主机生效；主机段写「所选主机」配置、跟 host 选择器走。
- **即时持久化**：无独立保存按钮；改动即落盘；输入类失焦 / 回车提交；写入失败必须可见反馈。
- **双 daemon 铁律**：应用段「守护进程」= 本机内建 daemon（设备级）；主机段「主机」= 所选 host 的 daemon。全程用对偶限定语，不出现无限定「daemon」。
- **术语铁律**：中转站（vendor，base_url+key）≠ relay / 中继（E2E 远程通道）；二者都含「中转」但完全不同，UI 全程不混用。
- **桌面专属 section**（守护进程 / 快捷键 / 集成 / 权限）：非桌面置灰（隐藏 vs 置灰见开放项）；本轮桌面 only。
- **能力门**：用量依赖 providerUsageList，缺失显「更新主机」；本设置不为缺失能力写降级路径。
- **本设置无老用户 / 迁移逻辑**（fork 未上线）。

## 7. 决策与边界（迭代 2 · 已在各 tab 锁定）

- **主机段 8 tab → 3 tab**：主机（连接 + 运维合并，身份顶部一处去重）/ 模型与提供方（provider-centric 三级）/ 用量。应用段 8 tab 不动。
- **模型与提供方 = provider-centric 三级下钻**（L1 提供方列表 → L2 提供方设置 → L3 中转站详情，**页面 + 面包屑、非弹框**），取代旧 vendor-centric 级联。
- **提供方升为可增删一等实体**（从🔒反转）：但**按真实能力分型**——内置只启停、自定义可删（删除需新后端能力）、in-app 自定义新增为新 UI（后端已支持）。
- **终端 tab 删除**：命令归入提供方「命令」字段（= agent CLI 启动命令）；**PTY 终端 profiles 与之是两套独立机制、不可合并**，其归属列开放项。
- **用量重做**：概览卡 + 时间切换 + 提供方/模型两维（**不到中转站**）；时间切换 / 估算金额 / 按模型三处底层缺口**诚实标"目标态·需新能力"**。
- **Agents / 工作区 tab 删除**（主壳已覆盖）；**插件延后保留、本轮隐藏**（内容留附录）。
- 配对只生成 offer，不做设备列表 / 吊销；service-proxy 浅档；relay ≠ 中转站、双 daemon 区分。

## 10. 术语表（本设置补充，余见 docs/helm/glossary 与 product.md）

- **提供方（Provider）**：CLI 代理（Claude Code / Codex / Copilot / OpenCode / Pi + 自定义）。「模型与提供方」三级**第一层**，对话中🔒锁定但**设置中为可增删一等实体**（内置只启停、自定义可删 / 新增，按真实能力分型）。
- **中转站（Vendor）**：配 base_url + key 的 API 供应商（官方直连 / 第三方转发）。三级**第二层**，挂在某提供方下；可增删改。**≠ relay**。
- **模型（Model）**：中转站放出的具体模型。三级**第三层**，可多选放出 + 设默认。
- **relay / 中继**：E2E 加密远程访问通道（手机 / 远程连主机的 transport）。在「主机」tab。**≠ 中转站**。
- **本机内建 daemon**：桌面 Electron 内建、设备级守护进程（应用段「守护进程」）。
- **所选主机的 daemon**：host 选择器选中那台主机的后端进程（主机段「主机」）。
- **命令（Command）**：提供方的 agent CLI 启动命令（argv，替换默认二进制）。**≠ 终端 PTY profile**（两套独立机制）。
- **插件（Plugins）**：给 agent 装能力的统一伞（MCP + 技能 + 精选目录）。**本轮延后保留、隐藏**。
- **作用域**：软件级（设备 / user）/ 目录级（项目 / project）/ 对话级（local，归主壳）。
