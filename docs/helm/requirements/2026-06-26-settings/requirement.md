## 1. 目标 & 用户价值

定义 Helm 在「主壳左下『设置』入口点开后」进入的**设置界面**——一个 Codex 式干净的 **master-detail 设置控制台**：左侧 section 导航分「应用」「主机」两段，右侧详情面随选中 section 切换。

- 用户为什么需要：
  - **配通一条能用的链路**：选哪个 CLI 提供方、接哪个中转站（API 供应商）、放出哪些模型——三层选型的深度管理一处搞定。
  - **主机是根上下文**：连接 / 配对手机远程 / 守护进程 / 提供方与中转站 / 用量 / 工作区 / Agents / 终端 / 插件，全部挂在「当前选中的主机」下，**主机配置是设置的一等公民**。
  - **给 agent 装能力**：MCP 服务器、技能（Skill）、精选插件目录，一处管理。
  - **调教这台设备的 App**：外观、快捷键、语言、本机守护进程、权限、关于与更新。

主壳（项目①）只露「设置」入口，级联里只露「管理中转站…」入口；**所有深度配置都在本设置界面**。本设置是主壳明确推迟划出的模块。

## 2. 范围

**做（完整设置界面 · 16 个 tab · 两级 IA · 全交互态 · 桌面）**

- **设置外壳**：master-detail；左导航两段（应用 App 级 / 主机 主机级）+ 主机段头 **host 选择器**；进入 / 退出 / 深链 / 默认落点 / 外壳级状态（加载 / 空 / 错误 / 主机离线）。
- **应用段（8 tab，跟设备走）**：通用 · 外观 · 快捷键 · 守护进程 · 集成 · 权限 · 诊断 · 关于。
- **主机段（8 tab，跟所选主机走）**：连接与配对 · 主机 · Agents · 工作区 · 终端 · 模型与中转站（三层级联）· 用量 · 插件（MCP / 技能 / 目录）。
- **全交互态**：默认 / 悬浮 / 活跃·选中 / 运行 / 禁用 / 空 / 加载 / 错误 / 离线 + 右键菜单（中转站行、MCP 行、终端 profile 行、连接行、Skill 行等，逐项列全）+ 即时持久化语义。

**不做（划走 / 后续）**

- **手机 / 紧凑态**：本轮桌面 only，紧凑态后续统一适配（只在注解里标差异）。
- **添加主机流程**：host 选择器只「切换已连主机」；添加主机（直连 / 粘链 / 扫码）归主壳 / onboarding，设置内只在配对子流程露二维码生成。
- **主壳工作视角**：对话、级联选择器本身、主壳侧栏的目录 / 对话列表——设置只做「配置 / 概览 / 管理」视角，不重做主壳的实时工作界面。
- **对话级作用域**：MCP / Skill 的「对话级」作用域归主壳；设置内只露「软件级 / 目录级」。
- **第三方插件市场**：不造打包格式 / registry / 评分 / 发布 / 付费；插件目录 = 精选可安装项（扎根真实 ACP 目录 + 技能安装）。

## 3. 全局 IA 与设置外壳

### 3.1 结构

单一 master-detail（左导航 + 右详情）。左导航分两段：

- **段 A · 应用（App 级）** —— 跟当前设备 / App 走，对所有已连主机生效。
- **段 B · 主机（主机级）** —— 段头一个 **host 选择器**（当前 active 主机 + 在线状态色 + 展开），下方各 tab 跟「所选主机」走；切主机后右详情面重载该主机数据。host 一等公民由整段 B 承载。

### 3.2 进入 / 退出 / 深链

- **进入**（3 个入口）：① 主壳左下「设置」→ 落「通用」（或记忆上次 section）；② 主壳三层级联的「管理中转站…」→ **深链**直达主机段「模型与中转站」tab；③ onboarding 本机 daemon 连接失败的「查看诊断」→ 落应用段「守护进程」/「诊断」（具体落点见开放项）。
- **退出**：返回主壳（顶部「返回主壳」或 Esc）。
- 一次只一个 section active；切 section 右详情面整块替换。

### 3.3 host 选择器

- 段 B 段头：当前主机名 + 在线 / 连接中 / 离线状态点 + relay/本地标识 + 展开下拉（列已连主机 + 当前项✓）。
- 选另一台 → 段 B 所有 tab 的右详情重载该主机数据。**不负责添加主机**（添加归主壳/onboarding）。
- 离线主机：状态点转红，仍可进入查看缓存，可改项禁用。

### 3.4 外壳级状态

- **加载**：切 section / 切主机 → 右详情面页级 spinner；行级数据各自 inline spinner。
- **空**：无已连主机 → 段 B 显「未连接主机」占位 +「去连接」（回主壳）；段 B tab 不可进。
- **错误**：主机级数据拉取失败 → 详情面顶 Alert + 重试。
- **离线**：所选主机离线 → host 选择器红点 + 详情面顶 callout（主机离线 + 重新连接）；只读项显缓存值、可改项禁用。
- **即时持久化**：所有控件改动**立即生效落盘**（App 级写设备本地、主机级写该主机配置），无独立「保存」按钮；输入类失焦/回车提交。

UI：`ui.html#shell`

## 4. tab 清单与每 tab WHAT

> 每 tab：作用 / 子功能 / 关键流程 / 状态要点（全态见 ui.html）/ 边界 / UI 锚点。状态命名统一：默认·悬浮·活跃·运行·禁用·空·加载·错误·离线。

### 段 A · 应用（App 级，跟设备走）

#### A1 通用（General）
- 作用：App 全局基础行为。
- 子功能：回车发送行为（排队 / 中断）、界面语言、服务 URL 打开方式（内置浏览器 / 系统浏览器）、终端回滚行数。
- 流程：改任一项 → 立即生效（语言切换即时重渲）。
- 边界：全为真实已有项；纯设备级偏好。
- UI：`ui.html#shell`（通用即设置外壳的金标准详情面）

#### A2 外观（Appearance）
- 作用：视觉呈现偏好。
- 子功能：主题（**下拉**，含浅 / 深 / 跟随系统及暗色变体；Helm 品牌方向浅色）、UI 字体族、代码字体族、UI 字号（11–24）、代码字号（9–22）、语法高亮方案、工作区标题来源（标题 / 分支）；带一块代码预览卡实时反映。
- 流程：改任一项 → 预览卡与全局即时反映。
- 状态：字体族空 = placeholder「系统默认」；预览卡渲染 diff 增删行。
- 边界：主题用下拉（保留全部变体，不收敛为三段——见开放项）；"工作区标题来源"与主壳侧栏显示偏好为**同一偏好、双向同步**（非两个真相源）。
- UI：`ui.html#ap1`

#### A3 快捷键（Keyboard shortcuts）
- 作用：查看 / 覆盖快捷键（桌面）。
- 子功能：按组列出动作 + 当前键位 + 捕获新键位 / 重置单项 / 全部重置。
- 流程：点动作键位 → 进「按下快捷键…」捕获态 → 按键提交 / Esc 取消；重置回默认。
- 状态：捕获中高亮；已覆盖项显「重置」。
- 边界：本轮不做冲突消解 UI（见开放项）；非桌面置灰。
- UI：`ui.html#ap2`

#### A4 守护进程（本机内建 daemon · 设备级）
- 作用：管理**本机内建守护进程**（桌面 Electron 内建，设备级）的生命周期。
- 子功能：状态（运行中 / 未运行 / 启动中 / 错误）+ 版本、随 Helm 启动开关、退出后保持运行开关、日志文件入口（复制路径 / 打开）、完整状态查看。
- 流程：切「随 Helm 启动」= 托管启停；停止走破坏性确认（连接的 agents/terminals 会停）。
- 边界：**双 daemon 铁律**——本 tab 只管「本机内建 daemon」，与主机段「主机」tab 的「所选 host 的 daemon」严格区分；不做 OS 级开机自启（见开放项）；不在此重启远程主机。桌面专属。
- UI：`ui.html#rt1`

#### A5 集成（Integrations · 本机命令行工具）
- 作用：本机命令行工具的安装 / 更新 / 卸载（桌面）。
- 子功能：列出可用 CLI runtime（Helm CLI + 各 agent CLI：Claude Code / Codex / Copilot / OpenCode / Pi 等），每行 图标 + 名称 + 已装版本 / 未安装 / 失败 + 安装 / 更新 / 卸载 + 行级安装进度。
- 流程：点安装 → 行级进度 → 完成显版本；失败显错误 + 重试。
- 边界：真实已有 = Helm CLI 安装（二元已装态）；**agent runtime 的「版本 / 更新 / 卸载」管理面为 Helm 新造方向**（见开放项），可用性检测可复用既有 provider 可用性能力；Skill 已迁至主机段「插件」tab，本 tab 不含 Skill。桌面专属。
- UI：`ui.html#rt2`

#### A6 权限（Permissions）
- 作用：App 系统权限（桌面 Electron 原生）。
- 子功能：列出权限（**真实 = 通知 + 麦克风**；摄像头 / 辅助功能为按需扩展，见开放项），每项 状态 badge（已授权 / 已拒绝 / 待请求 / 不可用）+ 去授予 / 打开系统设置 + 刷新。
- 流程：点请求 → 系统弹权限 → 状态更新；受限项引导「打开系统隐私设置」。
- 边界：只反映 / 引导系统权限，不代授予。桌面专属。
- UI：`ui.html#rt3`

#### A7 诊断（Diagnostics）
- 作用：本地排障集中处。
- 子功能：音频播放测试（thinking tone）、查看 daemon 日志入口、（轻量）连接自检。
- 流程：点测试 → 播放并报结果；点日志 → 打开 daemon 日志。
- 边界：**不重复实现** daemon 日志（其能力归守护进程 tab，诊断只复用入口）；「连接自检」为薄能力、是否保留见开放项；onboarding 失败「查看诊断」的落点见开放项。
- UI：`ui.html#ap3`

#### A8 关于（About）
- 作用：版本与更新。
- 子功能：Helm 品牌头 + App 版本、已连主机 daemon 版本（不一致高亮 / 离线）、检查 / 安装更新（桌面 updater）、发布频道（stable / beta）、社区链接。
- 流程：自动静默检查 → 有更新显提示 → 确认 → 安装中 → 重启生效；切频道改更新来源。
- 边界：不内嵌 changelog 浏览；不在关于内升级主机 daemon（那归主机段）。
- UI：`ui.html#ap4`

### 段 B · 主机（主机级，跟所选主机走）

#### B1 连接与配对（Connection & Pairing）
- 作用：所选主机的连接管理 + 手机远程配对。
- 子功能：① 主机标识（在线状态 / daemon 版本 / 重命名）；② 连接列表（**relay 远程 E2E / 本地直连 / TCP** 三类型 + 地址 + 实时延迟 + 活动标识 + 逐条移除）；③ relay 中继配置（启用 / 中继 endpoint / 中继 TLS / 公开 endpoint / 公开 TLS）；④ 配对设备（手机远程）——生成 / 重新生成二维码 + `#offer=` 配对链接（relay E2E，含 daemon 公钥，"链接像密码"安全提示）+ 复制。
- 流程：配 relay → 即时落盘；配对 → 展开二维码 / 链接 → 手机扫码 → 远程接入。
- 状态：**离线态**——host 选择器红点 + 顶部 callout；缓存连接显延迟超时；relay 与配对禁用、移除 / 重命名仍可用。
- 边界：**术语铁律**——此处 relay = E2E 远程访问通道，**≠** 中转站 vendor（那是「模型与中转站」tab 的 base_url+key）；配对只做「生成 offer」，**「已配对设备列表 / 吊销」不做**（relay 手机端密钥每连临时生成、无持久设备身份，见开放项）；重命名归本 tab。
- UI：`ui.html#connect`、`ui.html#connect-pair`、`ui.html#connect-offline`

#### B2 主机（Host）
- 作用：所选主机的枢纽与高级配置。
- 子功能：健康概览（= 主机诊断：状态 / 活动连接类型 / 版本 / 最近错误）、重启守护进程、服务代理（service-proxy 折叠高级区：localhost 始终可用 + 公开 baseUrl + listen + 文档入口）、危险操作（移除主机）。
- 流程：点重启 → 确认 → 重连过场；移除主机 → 危险确认 → 回主壳。
- 边界：只管「所选 host 的 daemon」（可能远程），与应用段「守护进程」（本机内建）区分；service-proxy 仅展示状态 + 配置 + 文档指引，DNS / 反代不入应用内（见开放项）；重启是否限本地、移除主机最后一条连接策略见开放项。
- UI：`ui.html#host`

#### B3 Agents
- 作用：所选主机的代理默认行为 + 概览。
- 子功能：代理默认配置（注入 Helm 工具开关、追加系统提示词）；（可选）agent 概览列表（provider 图标 + 标题 + 状态点 + 项目 / 分支 / 时间）。
- 流程：改默认 → 即时落盘（写入失败必须可见反馈，不静默）；列表行 = 打开（跳主壳）/ 归档 / 复制 ID / 在 Finder。
- 边界：**设置 = 配置 / 概览视角**，实时对话 / 工作归主壳；真实已有 = 代理默认配置；概览列表为 Helm 增量、与主壳列表防双真相源（见开放项）。
- UI：`ui.html#hw1`

#### B4 工作区（Workspaces）
- 作用：所选主机的工作区生命周期默认 + 概览。
- 子功能：工作区生命周期默认（PR 合并后自动归档干净工作区开关）；（可选）工作区概览列表（项目 / worktree 隔离 badge + 路径 + agent 数）。
- 流程：改默认 → 即时落盘；列表行 = 打开 / 移出 / 在 Finder。
- 边界：同 B3——配置视角，列表为增量、防双真相源（见开放项）。
- UI：`ui.html#hw2`

#### B5 终端（Terminals）
- 作用：所选主机的终端 profile 管理。
- 子功能：终端代理状态通知开关；profile 列表（图标 + 名称 + 命令 + 上移 / 下移排序）+ 新建 / 编辑 / 删除；编辑表单（名称 / 命令 / 参数；Helm 增强：描述 / 图标选择）。
- 流程：新建 / 编辑 → 表单 → 保存落盘；删除 → 确认。
- 状态：**空态两态**——删空（真空）vs 未设（回退内置默认 Claude Code / Codex / OpenCode 三项），需区分。
- 边界：真实字段 = 名称 / 命令 / 参数 + 图标按命令自动派生；**描述 / 手选图标为 Helm 增强、需扩 schema**（见开放项）。
- UI：`ui.html#hw3`、`ui.html#hw3b`

#### B6 模型与中转站（三层级联 · 本轮重头）
- 作用：三层选型的深度管理——用哪个 CLI 提供方 → 接哪个中转站（API 供应商）→ 放出哪些模型。
- IA：tab 内**两栏嵌套 master-detail + 模型层内嵌于中转站详情**（否决三列 miller，避免左导航占宽后拥挤；"放出哪些模型"本质是某中转站的属性）。
  - 内左列 = **提供方层**（锁定轴）。
  - 内主区 = 选中提供方 → 其中转站列表（+ 新建 / cc-switch 同步 / 故障转移折叠区）；选中中转站 → 其详情（含模型放出，带返回）。
- 子功能：
  - ① **提供方（provider）**：内置 CLI（Claude Code / Codex / Copilot / OpenCode / Pi）。**提供方🔒锁定——只启用 / 禁用 + 看状态 / 模型数 / 诊断，不可新增 / 删除提供方本身**（ACP 自定义为保留的既有高级能力、不计入本轮「新增提供方」主流程）。
  - ② **中转站（vendor）**：可增删改的一等实体——名称 / base_url / key（密码态）/ 通用配置（JSON 编辑器 + 便捷字段双向同步）/ 花费限额（日 / 月）+ 倍率 / 端点测速·健康（延迟 / 不健康红字）。
  - ③ **模型（model）**：自动拉取 + 拉不到时手动添加 + **多选「放出哪些」** + 设默认 + thinking 强度。
  - **cc-switch 同步**：读外部 cc-switch 数据（只读）→ 导入预览（勾选）→ 按关联键增量导入（更新不覆盖本地改动、不删非 cc-switch 项）。
  - **代理 / 故障转移**：折叠高级区（反代 + 熔断 + 故障转移队列），可后置（见开放项）。
- 流程：选提供方 → 选 / 新建中转站 → 配 base_url+key → 测速 → 放出模型 → 设默认 → 设为当前；主壳级联「管理中转站…」深链落本 tab 对应中转站。
- 状态：空（无中转站 → 新建 / 同步引导）、运行（测速中）、错误（中转站不健康红字 / 拉模型失败）、离线（改 / 测速 / 设当前禁用）。
- 边界：术语 中转站 vendor ≠ relay 远程；提供方不可增删；花费上限本轮只「设值 + 展示进度」，超限行为（告警 / 自动切换 / 阻断）见开放项。
- UI：`ui.html#mr1`（级联总览）、`ui.html#mr2`（中转站编辑 + 模型放出）、`ui.html#mr3`（cc-switch 同步）、`ui.html#mr4`（代理 / 故障转移）、`ui.html#mr6`（空 / 错 / 加载态）

#### B7 用量（Usage）
- 作用：所选主机各提供方 / 中转站的用量监控（只读）。
- 子功能：仪表盘——余额（credits / USD / requests / tokens）+ 多窗口进度（Session / Weekly / Biweekly）+ 限额 + 明细 + 定价；顶部刷新。
- 流程：进入 / 刷新 → 拉取并渲染；不支持用量的主机显门控提示。
- 边界：**只读监控**，配置在 B6；**依赖能力门 providerUsageList**（主机不支持则显「更新主机以查看用量」）；与 B6 数据同源、互相深链。
- UI：`ui.html#mr5`

#### B8 插件（Plugins · 给 agent 装能力的统一面）
- 作用：给所选主机的 agent 装能力。"插件"= 统一伞，实体落到 MCP + 技能 + 目录。
- 结构：tab 内顶部三平级子分段（MCP 服务器 / 技能 / 插件目录，默认落 MCP）。
- 子功能：
  - **MCP 服务器**：列表（名称 + 命令 / URL + 状态点 连接 / 失败 + 作用域 badge + 启用）+ 新建 / 编辑 / 删除 + 全局「注入到 agent」总闸 + **作用域（软件级 / 目录级）**。
  - **技能（Skill）**：已装列表（名称 + 来源 + 启用 + 更新 / 卸载）+ 安装（GitHub URL / ZIP）。
  - **插件目录（catalog）**：**精选**可安装项（扎根真实 ACP 目录：Gemini CLI / Qwen Code / Kimi Code / Cursor / DeepAgents 等 + 技能），每卡 图标 + 名称 + 类型 + 简介 + 来源·版本 + 安装 / 已安装（去管理）。
- 流程：装 MCP → 表单（类型 stdio/http + 命令 / URL + env + 作用域）→ 落盘；装 skill → 输入来源 → 安装中 → 完成；从目录安装 → 跳对应管理。
- 边界：**不造第三方市场**（无评分 / 发布 / 上传 / 付费）；**对话级作用域归主壳**，设置只露软件级 / 目录级；目录级须有「选工作区」约束；插件市场做不做见开放项。
- UI：`ui.html#pl1`（MCP）、`ui.html#pl2`（技能）、`ui.html#pl3`（目录）、`ui.html#pl4`（新建 MCP 弹窗）、`ui.html#pl5`（空 / 装中 / 错态）

## 5. 核心流程（端到端 · 每条有入口有出口）

1. **配通一条模型链路**：主壳级联「管理中转站…」或设置「模型与中转站」→ 选提供方 → 新建 / 选中转站 → 配 base_url + key → 测速 → 放出模型 + 设默认 → 设为当前 → 回主壳级联即生效。入口 = 级联 / 设置；出口 = 可在对话用该中转站 + 模型。
2. **cc-switch 一键同步**：「模型与中转站」→ cc-switch 同步 → 读外部数据 → 勾选导入 → 增量合并（不覆盖本地改动）→ 中转站列表出现导入项。
3. **配对手机远程**：主机段「连接与配对」→ 配对设备 → 展开二维码 / 链接 → 手机扫码（relay E2E）→ 手机接入遥控本机代理。
4. **给 agent 装 MCP**：主机段「插件」→ MCP 子分段 → 新建 → 填类型 / 命令 / env / 作用域 → 落盘 + 注入。
5. **调外观**：应用段「外观」→ 改主题 / 字体 / 字号 → 预览卡 + 全局即时反映。
6. **切主机重载**：host 选择器 → 选另一台在线主机 → 段 B 所有 tab 重载该主机数据。
7. **边界分叉**：无主机（段 B 占位 + 去连接）/ 加载（页级 / 行级 spinner）/ 错误（Alert + 重试）/ 主机离线（红点 + callout + 可改项禁用）/ 安装失败（行级错误 + 重试）/ 测速失败（不健康红字）——每条有入口有出口、无死胡同。

## 6. 全局状态与规则

- **两级 scope**：应用段写设备本地、对所有主机生效；主机段写「所选主机」配置、跟 host 选择器走。
- **即时持久化**：无独立保存按钮；改动即落盘；输入类失焦 / 回车提交；写入失败必须可见反馈。
- **双 daemon 铁律**：应用段「守护进程」= 本机内建 daemon（设备级）；主机段「主机」= 所选 host 的 daemon。全程用对偶限定语，不出现无限定「daemon」。
- **术语铁律**：中转站（vendor，base_url+key）≠ relay / 中继（E2E 远程通道）；二者都含「中转」但完全不同，UI 全程不混用。
- **桌面专属 section**（守护进程 / 快捷键 / 集成 / 权限）：非桌面置灰（隐藏 vs 置灰见开放项）；本轮桌面 only。
- **能力门**：用量依赖 providerUsageList，缺失显「更新主机」；本设置不为缺失能力写降级路径。
- **本设置无老用户 / 迁移逻辑**（fork 未上线）。

## 7. 决策与边界（已在各 tab 锁定）

- 三层级联落「两栏嵌套 + 模型内嵌中转站详情」，否决三列 miller。
- 用量独立 tab（与 B6 同源深链），不并入 B6。
- 插件 = 伞，收敛 MCP / 技能 / 目录，不造第三方市场。
- 提供方固定（只启停、不可增删），可增删的一等实体严格限定在中转站。
- 配对只生成 offer，不做设备列表 / 吊销（无持久设备身份）。
- service-proxy 浅档（localhost + publicBaseUrl + listen + 文档），DNS / 反代不入应用。
- Agents / 工作区设置 tab = 配置 / 概览视角，不重做主壳实时列表。

## 8. 开放项（送董事长闸 1 定夺）

1. **默认主题**：是否把默认改浅色 / 收敛暗色变体为三段？（现状默认跟随系统、下拉含全部变体）
2. **诊断「连接自检」**：做还是不做？（倾向不做，YAGNI + 与守护进程 / 连接诊断重叠）

4. **权限范围**：是否从真实的「通知 + 麦克风」扩到摄像头 / 辅助功能？
5. **双 daemon 表述**：本机内建 vs 所选 host 的对偶文案是否够清晰、需否更强区分？
6. **配对设备身份**：是否后续引入持久设备身份以支持「设备列表 + 吊销」？（现无底层支撑）
7. **主机操作边界**：远程主机是否开放配对 / 重启是否限本地 / 删到最后一条连接的策略？
8. **service-proxy 深浅**：保持浅档（localhost + publicBaseUrl + listen + docs）确认？
9. **Agents / 工作区 概览列表**：要不要在设置里做只读概览（主壳已有列表，防双真相源），还是只做默认配置？
10. **终端 profile 增强**：是否做描述 / 手选图标（需扩 schema），还是仅名称 / 命令 / 参数？
11. **代理 / 故障转移**：本轮做还是后置？（重度高级，倾向先做「展示 + 单中转站编辑」）
12. **花费上限行为**：超限后告警 / 自动切换 / 阻断本轮做不做？（倾向后置）
13. **插件市场**：要不要真做第三方市场？（倾向不做，先精选目录）
14. **host 选择器**：复用主壳主机切换器控件，还是设置内专用？
15. **非桌面 section**：置灰可见 vs 隐藏？

## 9. 验收标准（逐条可判定 · 抽样）

1. 主壳左下「设置」进入 → master-detail，左导航两段（应用 / 主机），主机段头 host 选择器显当前主机 + 在线色；默认落「通用」（或上次 section）。
2. 切左导航 section → 右详情面整块替换、选中行 accent 高亮、一次只一个 active。
3. host 选择器切另一台在线主机 → 段 B 所有 tab 右详情重载该主机数据；离线主机红点、可进查看缓存、可改项禁用。
4. 主壳级联「管理中转站…」→ 深链直达「模型与中转站」对应中转站。
5. 三层级联：提供方🔒只启停不可增删；中转站可新建 / 编辑（base_url+key+JSON+限额+倍率）/ 测速 / 删除；模型可多选放出 + 设默认；cc-switch 同步增量导入不覆盖本地。
6. 用量在 providerUsageList 缺失时显「更新主机」，存在时显余额 / 多窗口 / 限额 / 明细。
7. 连接与配对：连接列表三类型 + 实时延迟；relay E2E 配置即时落盘；配对生成二维码 / 链接；离线态红点 + callout + 配对禁用；术语全程 relay≠中转站。
8. 插件三子分段（MCP / 技能 / 目录）；MCP 可增删 + 作用域（软件级 / 目录级）+ 注入总闸；技能可装（GitHub / ZIP）；目录为精选、无市场元素。
9. 终端 profile 可增删改 + 排序；空态区分「删空」与「回退内置默认」。
10. 双 daemon：应用段「守护进程」管本机内建、主机段「主机」管所选 host，文案不混。
11. 所有控件即时持久化、无保存按钮；写入失败可见反馈。
12. 抽查每屏交互态与 ui.html 一致（各取一处 悬浮 / 活跃 / 运行 / 加载 / 错误 / 离线 / 右键核对，逐屏对齐）。

## 10. 术语表（本设置补充，余见 docs/helm/glossary 与 product.md）

- **提供方（Provider）**：CLI 代理（Claude Code / Codex / Copilot / OpenCode / Pi）。三层第一层，对话中🔒锁定，设置中只启停。
- **中转站（Vendor）**：配 base_url + key 的 API 供应商（官方直连 / 第三方转发）。三层第二层，可增删改的一等实体。**≠ relay**。
- **模型（Model）**：中转站放出的具体模型。三层第三层，可多选放出 + 设默认。
- **relay / 中继**：E2E 加密远程访问通道（手机 / 远程连主机的 transport）。在「连接与配对」tab。**≠ 中转站**。
- **本机内建 daemon**：桌面 Electron 内建、设备级守护进程（应用段「守护进程」）。
- **所选主机的 daemon**：host 选择器选中那台主机的后端进程（主机段「主机」）。
- **插件（Plugins）**：给 agent 装能力的统一伞，落到 MCP + 技能 + 精选目录。
- **作用域**：软件级（设备 / user）/ 目录级（项目 / project）/ 对话级（local，归主壳）。
