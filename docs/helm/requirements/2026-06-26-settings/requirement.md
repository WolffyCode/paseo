## 1. 目标 & 用户价值

定义 Helm 在「主壳左下『设置』入口点开后」进入的**设置界面**——一个 Codex 式干净的 **master-detail 设置控制台**：左侧 section 导航分「应用」「主机」两段，右侧详情面随选中 section 切换。

- 用户为什么需要：
  - **普通用户也能轻松配通一条链路**：选内置提供方（CLI）→ 接中转站（API 供应商 base_url+key）→ 放出模型；界面只留常用项、少术语，**会用就行**。
  - **主机是根上下文**：连接与配对 / 运维 / 模型与提供方 / 用量，全部挂在「当前选中的主机」下，**主机配置是设置的一等公民**；主机段默认进**当前主机**（有明确「当前」标识）。
  - **高级用户有逃生舱**：一个 **「打开配置文件（JSON）」** 入口（VSCode settings.json 式），界面没暴露的高级项都能在 JSON 里直接改——有它，GUI 才敢做简单。
  - **应用段（外观 / 快捷键 / 守护进程 / 权限 / 关于…）+ 插件**：**本版整段搁置 / 延后隐藏**，后续阶段回归（设计稿留附录、内容不删）。

主壳（项目①）只露「设置」入口，级联里只露「管理中转站…」入口；**所有深度配置都在本设置界面**。本设置是主壳明确推迟划出的模块。

## 2. 范围

**做（本版 = 主机段 3 tab + 配置文件 · 桌面 · 普通用户优先）**

- **设置外壳**：master-detail；左导航 = **主机段**（host 选择器默认当前主机 + 「当前」标识；主机 / 模型与提供方 / 用量）+ **配置文件（JSON）** 入口 + 灰置「应用设置 · 后续阶段」占位；进入 / 退出 / 默认落点（当前主机「主机」tab）/ 外壳级状态（加载 / 空 / 错误 / 主机离线）。
- **主机段 3 tab（跟所选主机走 · 做减法）**：① **主机**（连接与配对 + 运维合并；常用留界面，relay 全字段 / TCP / service-proxy 折叠「高级」或入 config.json）· ② **模型与提供方**（**内置 5 + 从目录新增的 ACP 提供方，启停 / 删除（目录真实·内置需新后端门控）/ CLI 检测 + 未装一键安装**；中转站扎实，L1 提供方 → L2 设置 → L3 中转站详情；高级折叠或入 config.json）· ③ **用量**（普通用户看得懂的余额 / 用量；三处缺口克制标「目标态」）。
- **★ 配置文件（JSON）编辑器**：**接一个现成 JSON 编辑器组件（Monaco / CodeMirror 类）**，映射真实 `config.json`；界面没暴露的高级项在此改（GUI 简单 + JSON 兜底完整）。
- **全交互态**：默认 / 悬浮 / 活跃·选中 / 运行 / 禁用 / 空 / 加载 / 错误 / 离线 + 右键菜单（提供方行、中转站行、连接行、JSON 编辑器等）+ 即时持久化语义。

**不做（搁置 / 删除 / 延后）**

- **应用段 8 tab 整段搁置**：通用 / 外观 / 快捷键 / 守护进程 / 集成 / 权限 / 诊断 / 关于——本版不做、移出 nav（灰置「后续阶段」占位）；设计稿留附录，后续回归（**内容不删**）。
- **不做「自造命令 / 名字」的自定义提供方**：「+ 新增提供方」**只从真实 ACP 目录（catalog）选已存在的**加入（Gemini CLI / Qwen Code / Kimi / Cursor / DeepAgents 等），**不提供任意命令 / 名字的自造入口**。要更高级的自定义走「配置文件（JSON）」手改。从目录新增复用真实 ACP catalog 能力，非新造后端。
  - **删除（董事长本次加回，右键 danger）**：目录新增的＝真实删除（从 `agents.providers` 移除，回目录可再加）；**内置 5 的删除＝需新主机能力**（给入口但门控、不假装能删）。
  - **一键安装 agent CLI（新增）＝可能需新主机能力**（标注，待架构）；Helm CLI 安装 + 安装指引外链为真实兜底。
- **删除 tab**：Agents（主壳已覆盖）· 工作区（主壳已能操作）· 终端（PTY profiles 归属见开放项，本版可经 config.json 改）。
- **延后隐藏**：插件（MCP / 技能 / 目录）——不进本版 nav，设计稿留附录（内容不删）。
- **手机 / 紧凑态**：本轮桌面 only。**添加主机流程**：host 选择器只切换已连主机，添加归主壳 / onboarding。

## 3. 全局 IA 与设置外壳

### 3.1 结构

单一 master-detail（左导航 + 右详情）。本版左导航 = **主机段为主**：

- 段头 **host 选择器**（当前 active 主机 + 在线状态色 + **「当前」标识** + 展开切换）。
- **主机段 3 tab**（主机 / 模型与提供方 / 用量）跟「所选主机」走；切主机后右详情面重载该主机数据，**默认进当前主机「主机」tab**。
- **配置文件（JSON）** 入口（导航行）→ config.json 编辑器。
- **灰置「应用设置 · 后续阶段」** 占位（不可点，标本版搁置）；应用段（App 级）回归见开放项。

### 3.2 进入 / 退出 / 深链

- **进入**：① 主壳左下「设置」→ 落**当前主机「主机」tab**（或记忆上次 section）；② 主壳级联「管理中转站…」→ **深链**直达「模型与提供方」对应提供方 / 中转站（L2 / L3）；③ onboarding 本机 daemon 失败「查看诊断」→ 落「主机」tab 健康概览（应用段诊断已搁置，见开放项）。
- **退出**：返回主壳（顶部「返回主壳」或 Esc）。
- 一次只一个 section active；切 section 右详情面整块替换。

### 3.3 host 选择器

- 段 B 段头：当前主机名 + 在线 / 连接中 / 离线状态点 + relay/本地标识 + 展开下拉（列已连主机 + 当前项✓）。
- 选另一台 → 段 B 所有 tab 的右详情重载该主机数据。**不负责添加主机**（添加归主壳/onboarding）。
- 离线主机：状态点转红，仍可进入查看缓存，可改项禁用。

### 3.4 外壳级状态

- **加载**：切 section / 切主机 → 右详情面页级 spinner；行级数据各自 inline spinner。
- **空**：无已连主机 → 段 B 显「未连接主机」占位 +「去连接」（回主壳）；段 B tab 不可进。
- **错误**：主机级数据拉取失败 → 详情面顶 Alert + 重试。
- **离线**：所选主机离线 → host 选择器红点 + 详情面顶 callout（主机离线 + 重新连接）；只读项显缓存值、可改项禁用。
- **即时持久化**：GUI 控件改动**立即落盘**（写该主机 config.json），无独立「保存」；配置文件（JSON）编辑器走校验后保存。

UI：`ui.html#hm1`（主机 tab = 本版外壳现役范例）

## 4. tab 清单与每 tab WHAT

> 每 tab：作用 / 子功能 / 关键流程 / 状态要点（全态见 ui.html）/ 边界 / UI 锚点。状态命名统一：默认·悬浮·活跃·运行·禁用·空·加载·错误·离线。

### 段 A · 应用（App 级）—— 后续阶段 · 本版搁置

> 应用段 8 tab（通用 / 外观 / 快捷键 / 守护进程 / 集成 / 权限 / 诊断 / 关于）**本版整段搁置**、移出 nav（导航仅留灰置「应用设置 · 后续阶段」占位）。其详细 WHAT 与设计稿保留于 [ui.html](./ui.html) 附录与版本历史（**内容不删**），后续阶段回归。回归时机与设备级（本地）↔ 主机级（config.json）边界划分见开放项。

### 段 B · 主机（主机级 · 本版 3 tab + 配置文件 · 做减法）

> 普通用户优先：每页只留常用项、少术语；高级 / 少用项折叠「高级」或给「在 config.json 编辑」链接，由配置文件（JSON）编辑器兜底完整。

#### B1 主机（Host · 连接与运维合并 · 简化）
- 作用：所选主机的连接、配对、运维，原「连接与配对」+「主机」合并一屏；普通用户看得懂。
- **常用（留界面）**：顶部一处身份（名 + 在线状态 + 版本 + 「当前主机」+ 重命名，仅一处）· 连接状态概览（当前连接方式 + 延迟 + 在线）· 重启主机服务 · 远程访问开关（弱化术语）· 配对手机（二维码 + 链接）。
- **高级（折叠 / 入 config.json）**：relay 中继全字段（endpoint / TLS / 公开 endpoint…）· TCP / 多连接列表 · 服务代理 service-proxy —— 收进「高级」折叠区并给「在 config.json 编辑」链接。
- 流程：配对 → 二维码 / 链接 → 手机扫码接入；重启 → 确认 → 重连；移除主机 → 危险确认（收进 host 选择器溢出 / 高级，简化屏不堆危险区，无死胡同）。
- 状态：**离线态**——host 选择器红点 + 顶部 callout；常用项禁用、高级折叠标「离线只读，config.json 仍可编辑」；重命名 / 重连仍可用。
- 边界：术语弱化（relay = 加密远程通道 **≠ 中转站 vendor**；daemon = 这台主机的后台服务，与已搁置的应用段「本机内建 daemon」区分）；配对只生成 offer、不做设备列表 / 吊销。
- UI：`ui.html#hm1`、`ui.html#hm1-pair`、`ui.html#hm1-offline`

#### B2 模型与提供方（Models & Providers · 内置 + 从目录 · 三级 · 中转站扎实）
- 作用：以提供方为中心配通链路——提供方（CLI）→ 中转站（API 供应商 base_url+key）→ 放出模型。
- **L1 提供方列表**：**内置 5**（Claude Code / Codex / Copilot / OpenCode / Pi）+ **从目录新增的 ACP 提供方**（如 Qwen Code）；每行 图标 + 名称 + **CLI 已装 / 未装检测（每行都有）** + 模型数 + 开关 + ⚙ 设置（→ L2）。顶部 **「+ 新增提供方」= 从真实 ACP 目录（catalog）选已存在的加入**（目录选择器，已加入的不在列）；**不做自造命令 / 名字的自定义新增**。
  - **删除（董事长本次加回 · 右键 danger，每行都有「删除」入口）**：**目录新增的 ACP 提供方 ＝ 真实删除**（二次确认 → 从 `agents.providers` 移除该项及其中转站 / 放出模型，回目录可再加）；**内置 5 ＝需新主机能力**——给「删除」入口但**标注「需新主机能力」、点击落能力门「更新主机后可删」，不假装能删**（架构阶段给方案）。config.json 始终可兜底。
  - **一键安装 CLI（新增 · CLI 未安装行）**：未安装行（如 OpenCode）给「一键安装」→ 安装中（loading）→ 成功转「CLI 已安装」可启用；**失败 → 行内红字 + 重试**。**诚实标注：** Helm CLI 安装 + 安装指引（外链）＝真实可用；**agent CLI（claude/codex/opencode…）一键装 / 更新可能需新主机能力**——缺则按钮转「需更新主机」回落安装指引（架构阶段给方案）。
- **L2 提供方设置**（面包屑返回 L1）：基础信息（名称 / 图标 / 命令）+ 该提供方的中转站列表 +「新增中转站」。**启停只在顶部一处 toggle**（删除了底部重复的「提供方管理 / 停用」块）。
- **L3 中转站详情**（drill-in 页 · 非弹框 · 扎实）：**常用突出**——base_url / key（密码态）/ 端点测速 / **放出模型**（多选 + 设默认 + thinking）；**高级折叠**——通用配置 JSON / 花费限额 / 倍率，并给「在 config.json 编辑全部高级项」链接；删除中转站（中转站可增删改）。
- **跨模块依赖（关键）**：中转站配置是**唯一真相源**，驱动 home-shell 对话框 composer 的中转站 / 模型选择与默认（「当前」中转站 = composer 默认 vendor）；composer「管理中转站…」深链回 L2 / L3；切 host 整体随之切换。
- 流程：L1 启停提供方 / 顶部「+ 新增提供方」从目录加入 / 行右键删除（目录真实·内置门控）/ 未装行一键安装 CLI → L2 配命令 / 管中转站 → L3 配 base_url+key → 测速 → 放出模型 → 设默认 → 设为当前；切 host / section → 栈重置回 L1。
- 状态：默认 / 悬浮 / 活跃 / 运行（测速中 / **CLI 安装中** / 删除中）/ 禁用（CLI 未装 / **内置删除门控**）/ 空（无中转站）/ 加载 / 错误（不健康 401 / 拉模型失败 / **CLI 安装失败**）/ 离线。
- 边界与能力门：**CLI 只显已装 / 未装，无结构化版本号**；命令 = 提供方 agent CLI 启动命令（**≠ 终端 PTY profile**，后者归 config.json / 开放项）；**删除内置提供方、一键安装 / 更新 agent CLI ＝ 需新主机能力（缺口，诚实标注、待架构）**，目录新增删除 + Helm CLI 安装 + 安装指引为真实；花费限额 + 倍率无内建核算，本版只「设值占位」或入 config.json；术语 中转站 vendor ≠ relay；用量在用量 tab。
- UI：`ui.html#mp1`（L1）、`ui.html#mp1-add`（从目录新增 · 目录选择器）、`ui.html#mp2`（L2）、`ui.html#mp3`（L3 + 模型放出 + 高级折叠）、`ui.html#mp5`（空 / 错 / 状态合屏）

#### B3 用量（Usage · 简化）
- 作用：所选主机的用量监控（只读），普通用户看得懂。
- **常用**：每提供方一块——余额 / 已用 / 还能用 + 一条额度条，直观清楚；顶部刷新 + 门控提示。
- **克制呈现（标「目标态」）**：今天 / 本周 / 本月 / 全局 时间切换、估算总金额、按具体模型明细——底层缺口，弱化 / 默认折叠并标「目标态」，不喧宾夺主。
- 边界与能力门：真实只到**提供方维**（无结构化模型维、无中转站维）；窗口为各家 API 固定（无用户时间切换）；无定价 / 估算逻辑；**依赖能力门 providerUsageList**（缺失显「更新主机」）。三处缺口要做需新能力（开放项）。
- UI：`ui.html#us1`（用量总览）、`ui.html#us2`（门控 / 空 / 加载 / 离线态）

#### B4 ★ 配置文件（JSON）编辑器（逃生舱 · 新增）
- 作用：VSCode settings.json 式的 config.json 编辑器——**界面简单 + JSON 兜底完整**；界面没暴露的高级项都能在此直接改、修正。
- 子功能：**接一个现成 JSON 编辑器组件（Monaco / CodeMirror 类）**——行号 + 语法高亮 + JSON 校验（非法标红、禁存）+ 查找 + **格式化 / 恢复 / 保存** 三个标准动作，均为组件自带标准能力；顶部说明「界面没暴露的高级项都能在这改」。**不自造区段导航、不自定义右键**（右键＝编辑器自带标准菜单）。
- 映射真实持久化：所选主机的 `config.json`（跟 host 选择器走、经守护进程读写，**非设备级偏好**，见开放项）；覆盖被界面简化掉的全部高级项（relay 全字段 / mcp / terminalProfiles / serviceProxy / log / auth / metadataGeneration / 各提供方 env·command / 语音听写 等）。**单一真相源**：GUI 与 JSON 重叠字段双向反映。
- 流程：进入 → 显当前 config.json → 改 → 语法 + schema 校验（非法标红、禁存）→ 保存（需重启的字段给提示）；离线只读。
- 边界：本版仅「编辑文件」；危险护栏强度（改密码 / 监听可能锁死连接）见开放项；config scope（所选主机 vs 设备）见开放项。
- UI：`ui.html#cfg1`

## 5. 核心流程（端到端 · 每条有入口有出口）

1. **配通一条模型链路（普通用户 · 三级）**：设置「模型与提供方」L1 启用内置提供方 → ⚙ 进 L2 → 选 / 新建中转站 → L3 配 base_url + key → 测速 → 放出模型 + 设默认 → 设为当前 → home 对话框 composer 的中转站 / 模型即跟随生效（跨模块真相源）；主壳级联「管理中转站…」深链直达 L2 / L3。
2. **配对手机远程**：主机段「主机」→ 配对手机 → 展开二维码 / 链接 → 手机扫码 → 手机接入遥控本机代理。
3. **看用量**：主机段「用量」→ 每提供方余额 / 用量额度条直观；时间切换 / 估算金额 / 按模型为「目标态」克制呈现；缺能力门显「更新主机」。
4. **改高级项（逃生舱）**：左导航「配置文件（JSON）」→ 显当前主机 config.json → 改界面没暴露的高级项（relay 全字段 / mcp / 终端 / 日志 / 各提供方 env·command…）→ 校验通过保存（需重启字段给提示）。各页「在 config.json 编辑」链接也直达此处。
5. **切主机重载**：host 选择器 → 选另一台在线主机 → 主机段 3 tab + 配置文件重载该主机数据，默认进当前主机「主机」tab。
6. **边界分叉**：无主机（占位 + 去连接）/ 加载（页级 / 行级 spinner）/ 错误（Alert + 重试）/ 主机离线（红点 + callout + 可改项禁用，config.json 离线只读）/ CLI 未装（unavailable 行）/ 测速失败（不健康红字）/ 能力缺口（门控 / 目标态提示，不写降级）——每条有入口有出口、无死胡同。

## 6. 全局状态与规则

- **本版 scope = 主机级**：主机段 3 tab 与配置文件都写「所选主机」的 `config.json`、跟 host 选择器走、经守护进程读写。应用段（设备级 / 本地）整段搁置，其设备级 scope 待回归时定（开放项）。
- **GUI 简单 + JSON 兜底**：界面只暴露常用项；高级 / 少用项折叠或移到配置文件（JSON）编辑器；GUI 与 JSON 是**单一真相源**、重叠字段双向反映。
- **即时持久化**：GUI 控件改动即落盘、无独立保存；JSON 编辑器走「校验通过后保存」（可批量改、需重启字段给提示）；写入失败必须可见反馈。
- **术语弱化（普通用户）**：少用专业词；中转站（vendor，base_url+key）≠ relay / 中继（加密远程通道），二者全程不混用；daemon = 「这台主机的后台服务」。
- **能力门 / 目标态**：用量依赖 providerUsageList，缺失显「更新主机」；底层没有的能力（用量三缺口等）标「目标态」、不写降级路径。
- **本设置无老用户 / 迁移逻辑**（fork 未上线）。

## 7. 决策与边界（迭代 3 · 简化 · 已锁定）

- **本版 = 主机段 3 tab + 配置文件（JSON）**；**应用段 8 tab 整段搁置**（后续阶段，移出 nav，留附录）；插件延后隐藏。
- **普通用户优先 · 做减法**：每页只留常用项、少术语；高级 / 少用项折叠「高级」或入 config.json。**GUI 简单 + JSON 兜底完整**——加 VSCode settings.json 式 config.json 编辑器作逃生舱。
- **提供方新增 = 从目录（董事长修正）**：内置 5 始终在列 + **「+ 新增提供方」从真实 ACP 目录（catalog）选已存在的加入**；**不做「自造命令 / 名字」的自定义新增**；删「底部提供方管理块」（启停收敛顶部一处）。
- **提供方删除 = 右键「删除」（董事长本次加回）**：目录新增的＝真实删除（`agents.providers` 移除，回目录可再加）；**内置 5 的删除＝需新主机能力**（给入口但门控、诚实标注、不假装能删，架构给方案）。
- **CLI 存在检测每行覆盖 + 一键安装**：5 内置 + 目录新增每行都有「已装 / 未装」检测；未装行「一键安装」（**agent CLI 一键装 / 更新可能需新主机能力**，缺则回落安装指引外链——真实兜底）。

- **中转站做扎实（跨模块）**：L3（base_url / key / 测速 / 放出模型 / 设默认 / 设当前）是 home-shell composer 中转站 / 模型选择的**唯一真相源**；高级（JSON / 限额 / 倍率 / 故障转移）折叠或入 config.json。
- **用量简化**：每提供方余额 / 额度直观；时间切换 / 估算金额 / 按模型三处缺口克制标「目标态」。
- **终端 PTY profiles**：本版经 config.json 改，专用 GUI 归属列开放项；relay ≠ 中转站、配对只生成 offer。

## 10. 术语表（本设置补充，余见 docs/helm/glossary 与 product.md）

- **提供方（Provider）**：CLI 代理。内置 5（Claude Code / Codex / Copilot / OpenCode / Pi）+ **从目录新增的 ACP 代理**（真实 ACP catalog 已知项，如 Gemini CLI / Qwen Code / Kimi / Cursor / DeepAgents）。「模型与提供方」三级**第一层**；均可启停；**删除**＝右键 danger：目录新增的真实删除（回目录可再加）、**内置 5 需新主机能力**（门控）；每行 CLI 检测、未装可一键安装（agent CLI 一键装可能需新主机能力）；**不做自造命令 / 名字的自定义新增**。
- **中转站（Vendor）**：配 base_url + key 的 API 供应商（官方直连 / 第三方转发）。三级**第二层**，挂在某提供方下；可增删改；**是 home-shell composer 中转站 / 模型选择的真相源**。**≠ relay**。
- **配置文件（JSON）**：VSCode settings.json 式的 config.json 编辑器；映射所选主机真实持久化配置；GUI 没暴露的高级项在此改（GUI 简单 + JSON 兜底）。
- **模型（Model）**：中转站放出的具体模型。三级**第三层**，可多选放出 + 设默认。
- **relay / 中继**：E2E 加密远程访问通道（手机 / 远程连主机的 transport）。在「主机」tab。**≠ 中转站**。
- **本机内建 daemon**：桌面 Electron 内建、设备级守护进程（应用段「守护进程」）。
- **所选主机的 daemon**：host 选择器选中那台主机的后端进程（主机段「主机」）。
- **命令（Command）**：提供方的 agent CLI 启动命令（argv，替换默认二进制）。**≠ 终端 PTY profile**（两套独立机制）。
- **插件（Plugins）**：给 agent 装能力的统一伞（MCP + 技能 + 精选目录）。**本轮延后保留、隐藏**。
- **作用域**：软件级（设备 / user）/ 目录级（项目 / project）/ 对话级（local，归主壳）。
