## 1. 目标 & 用户价值

**一句话目标**：给 Helm 一个独立的「指挥舱」视图，一屏纵览（已连接主机上）所有 AI 编码代理的实时状态与无限嵌套关系，并把「需要你」的事聚合成一条可秒清的待办队列，支持**原地处置**——不必逐个切进对话。

**用户为什么需要它（场景）**：

- Helm 的产品前提就是**同时跑很多代理**（每个还能无限嵌套 subagent），分布在本机/远程多主机上。但主壳是 Codex 式**单对话**——一次只看得见一个对话。
- 于是用户**丢失全局**：谁卡在等你批权限、谁跑完了、谁报错、谁在空闲待命，全都被压在一个个对话 tab 里。用户**自己成了瓶颈**，却要逐个 tab 翻找才能处理，越跑越多越乱。
- 指挥舱把已有的多路数据流（代理状态、代理输出、权限请求/解决、父子嵌套关系、工作区活跃聚合）**重新编排**成「纵览 + 行动」的信息架构：一眼看清谁在干嘛，一处清掉所有「需要你」的事。这是**编排层（软件层）创新**，不依赖任何新模型能力。

## 2. 范围

**做（本需求）**

- 侧栏新增**顶层入口「指挥舱」**，带一个「需要你」计数徽标（待批权限 + 等你输入 + 报错 的总数；为 0 时不显）。徽标是同一份聚合的派生，不新增能力。
- **指挥舱总览**（`#mc1`）：状态分桶计数（运行 / 空闲待命 / 需关注 / 报错 / 已完成）+ 按主机/项目分组的活跃总览 + 「需要你」队列入口。
- **「需要你」原地处置队列**（`#mc2`）：三类行——
  - **待批权限**：工具调用摘要 + 命令/文件/diff **紧凑预览** + 批准 / 拒绝 / 总是允许 / 进入对话。
  - **等你输入**：**行内迷你 composer**（纯文本回话 + 发送）+ 进入对话。
  - **报错**：错误摘要 + 重试 / 进入对话。
- **全代理嵌套树 + 筛选**（`#mc3`）：root → subagent 无限嵌套展开；每行状态点 + 当前动作 + 主机/项目/提供方标签；筛选条（主机 / 项目 / 提供方 / 状态）；行 hover 动作与右键菜单。
- **状态合集**（`#mc4`）：空态 / 加载骨架 / 某主机离线降级 / 全局错误。
- 「进入对话」作为**交接桥**：把主壳导航并聚焦到该代理的对话（指挥舱不取代单对话主壳）。

**明确不做（防范围蔓延 / YAGNI）**

- **不做单对话主壳**：那是已有主界面；指挥舱只「纵览 + 处置」，「进入对话」交接给它。
- **不做设置/选型**：提供方 · 中转站 · 模型、relay、诊断等都不在本需求。
- **不重新设计权限弹窗、归档确认、级联归档语义、审查/diff 面板**：一律**复用既有能力**，指挥舱只做紧凑预览与原地动作。
- **不做完整的「新建代理」流程**：仅空态提供「起一个代理」引导，点击**交接给既有新对话流程**，本需求不重做创建。
- **不做批量操作**（批量批准 / 批量归档 / 一键清空）：v1 不做，留待确有需要再议。
- **不做主动推送 / 通知系统**：指挥舱是「你打开来看」的视图（+ 一个应用内计数徽标）；操作系统级推送是另一回事。
- **不做历史/审计日志**（过往审批记录、用量报表）：v1 不做。
- **不做行内迷你 composer 的富能力**：slash / @ / 附件 / 语音 / 队列轨等都不做，需要富输入就「进入对话」。
- **不做手机/紧凑布局**：本轮桌面 only。
- **不修改协议 / RPC / `server_info.features.*` 的实现**：跨主机等新能力在本文只以「依赖 / 能力门」表述（见 §5），不写实现。
- **不做跨主机同时聚合所需的新增 daemon/客户端能力**：主价值锁定「当前已连接主机」，跨多主机为能力门（见 §5）。

## 3. 流程（端到端，含失败/取消/降级分叉）

**A. 打开指挥舱并分诊（主路径）**

1. 用户点侧栏「指挥舱」入口（入口徽标已显示 N 件「需要你」）。
2. 进入总览 `#mc1`：5 个分桶计数 + 按主机/项目分组的活跃总览 + 「需要你」队列入口。计数随代理状态实时更新。
3. 分叉——加载：初次聚合期间显示**加载骨架** `#mc4`；无任何代理显示**空态** `#mc4`；聚合整体失败显示**全局错误** `#mc4`（可重试）。
4. 点任一分桶计数 → 跳到对应子集（如点「报错」→ 队列定位到报错行 / 树筛到报错）。点「需要你」队列入口 → 进入 `#mc2`。

**B. 原地处置 · 待批权限**（`#mc2`）

1. 用户展开一条「待批权限」行 → 看到工具调用摘要 + 命令/文件/diff 紧凑预览。
2. 点**批准** → 复用既有权限流放行 → 该代理继续运行 → 该行**离开队列**（计数与入口徽标随之减少），进入「运行」桶。
3. 分叉：
   - **拒绝** → 既有权限流拒绝 → 代理按自身逻辑跳过该工具/结束回合 → 该行离开队列，按代理新状态归入对应桶。
   - **总是允许** → 复用既有「总是允许」作用域记忆 + 放行 → 该行离开队列，代理继续运行。
   - **进入对话** → 交接：主壳导航并聚焦该代理对话（离开指挥舱视图；侧栏入口随时可返回）。
   - **处置失败**（权限请求已超时/被撤销、主机离线、代理已往前走）→ 该行**就地显示失败原因**，转为不可批准并提供「进入对话」/「重试」，**绝不静默丢失**。
   - **被他端抢先**（另一客户端已批/拒该请求）→ 该行实时更新为「已处理」并自动离开队列。

**C. 原地处置 · 等你输入**（`#mc2`）

1. 用户展开一条「等你输入」行 → 出现行内迷你 composer。
2. 输入回话并发送 → 复用既有发送 prompt 能力投递 → 代理进入运行 → 该行离开队列，进入「运行」桶。
3. 分叉：
   - **取消**（关闭 composer 未发送）→ 该行保留在队列，不变更。
   - **发送失败**（主机离线等）→ composer 就地显示失败 + 可重试 / 进入对话，草稿不丢。

**D. 原地处置 · 报错**（`#mc2`）

1. 用户看到「报错」行（错误摘要）。
2. 点**重试** → 该代理重新运行 → 该行离开队列进入「运行」桶；若再次失败 → 回到「报错」行（摘要更新）。
3. 点**进入对话** → 交接到该代理对话。

**E. 浏览全代理树并筛选**（`#mc3`）

1. 用户切到「全代理嵌套树」。
2. 展开 root → 逐层展开 subagent（无限嵌套）。每行：状态点 + 当前动作 + 主机/项目/提供方标签。
3. 应用筛选（主机 / 项目 / 提供方 / 状态，可叠加）→ 树收窄；无匹配 → 显示「无匹配，清除筛选」。
4. 行 hover → 出现「进入对话」快捷动作；右键 → 菜单（进入对话 / 归档）。
5. 点「进入对话」→ 交接到该代理对话。点「归档」→ 复用既有归档确认（运行中/含子代理时弹既有确认并级联），归档后该代理（及其子树）实时从树中消失。

**F. 降级 / 离线 与恢复**

1. 某已连接主机在浏览中掉线 → 该主机分组**降级**：标记离线、其下代理与队列行显示为陈旧（last-known）且**动作禁用**并附「主机离线 · 重连」，**不静默移除**（用户仍知道有这些事）。其它主机不受影响。
2. 该主机**重连** → 分组刷新，计数/队列/树恢复可操作。
3. 若**唯一**已连接主机离线 → 整个指挥舱进入离线/重连态（与「空态」区分：不误报为「没有代理」）。
4. 聚合整体无法加载 → 全局错误 `#mc4` + 重试。

**G. 空态**

- 已连接主机上确实没有任何代理 → 空态 `#mc4`：引导「起一个代理」，点击**交接给既有新对话流程**；代理出现后回到正常总览。

> **连通性自证**：每条原地处置结束后，该行都明确「离开队列 → 归入某桶」，用户停留在指挥舱（队列缩短），无死胡同；「进入对话」是离开视图的唯一出口，侧栏入口恒可返回；离线/失败/取消都有就地恢复路径，无悬挂态。

## 4. 屏与全状态（逐屏对应 [ui.html](./ui.html)）

### `#mc1` 指挥舱总览（默认·有活跃代理）

- **默认**：顶部 5 个分桶计数卡（运行 / 空闲待命 / 需关注 / 报错 / 已完成）；中部「需要你」队列入口（显条数）；下部按**主机 → 项目**分组的活跃总览。
- **hover**：分桶卡 hover 高亮可点；分组头 hover 显示折叠/展开手柄；队列入口 hover 高亮。
- **活跃/选中**：被选中的分桶/分组高亮（surface 强调）。
- **运行态**：计数与分组随代理状态**实时跳动**（无需手动刷新）。
- **禁用**：仅连一台主机时不渲染多主机层级（单组，非禁用）；跨主机聚合不可用时不显其它主机组（见 §5 能力门①）。
- **空**：无代理 → `#mc4` 空态。
- **加载**：初次聚合 → `#mc4` 骨架（分桶/分组占位）。
- **错误**：聚合失败 → `#mc4` 全局错误。
- **离线**：某主机离线 → 该组降级（见 `#mc4`）；唯一主机离线 → 整页离线态。
- **右键菜单**：分桶卡与分组头**无**右键菜单（代理级菜单在 `#mc3`）。
- **点击效果**：点分桶计数 → 跳到该子集（报错→队列报错行/树筛报错，依此类推）；点「需要你」队列入口 → 进入 `#mc2`；点分组头 → 折叠/展开该组；点（若启用的）成本/token 指标 → 仅展开说明，非核心（见 §5 能力门②）。

### `#mc2` 「需要你」原地处置队列

- **默认**：按类型分组的行列表（待批权限 / 等你输入 / 报错），每行折叠为一句摘要 + 主机/项目/提供方标签。
- **hover**：行 hover 高亮并显露主要动作按钮。
- **活跃**：展开行——待批权限行展开为命令/文件/diff 紧凑预览 + 动作；等你输入行展开为行内迷你 composer；报错行展开错误摘要。
- **运行态**：处置后该行**动画离场**（代理转入运行）；新「需要你」事项实时插入。
- **禁用**：所属主机离线 → 该行置灰、动作禁用、附「主机离线 · 重连」。
- **空**：无任何「需要你」事项 → 队列「全部清空（inbox zero）」态（区别于无代理空态）。
- **加载**：骨架行。
- **错误**：单行处置失败 → **就地**红字失败原因 + 重试/进入对话；整体加载失败 → `#mc4` 全局错误。
- **离线**：见「禁用」；重连后恢复可操作。
- **右键菜单**：行右键 = 「进入对话」（1 项；批准/拒绝/总是允许/发送/重试为行内按钮，不入右键）。报错行右键追加「重试」。
- **点击效果**：批准→放行该代理继续、行离场；拒绝→拒绝该工具、行离场；总是允许→记住作用域并放行、行离场；发送→投递回话、代理运行、行离场；重试→代理重跑；进入对话→交接主壳；展开/折叠箭头→展开该行。

### `#mc3` 全代理嵌套树 + 筛选

- **默认**：顶部筛选条（主机 / 项目 / 提供方 / 状态 多选）；下方 root 行（折叠，显子代理数）。
- **hover**：行 hover 显「进入对话」快捷动作 + 展开箭头。
- **活跃/选中**：选中行高亮；已展开节点显其 subagent 子树（缩进体现层级）。
- **运行态**：运行中行状态点为运行色并轻动；「当前动作」标签**实时更新**（源自该代理最新输出活动）。
- **禁用**：筛选无匹配 → 空筛选态「无匹配，清除筛选」；离线主机行置灰、动作禁用。
- **空**：无代理 → `#mc4` 空态。
- **加载**：骨架树。
- **错误**：加载失败 → `#mc4` 全局错误。
- **离线**：离线主机组置灰 + 离线标记，仍可展开看 last-known 树（陈旧），动作禁用；重连恢复。
- **右键菜单**：行右键 = 「进入对话」「归档」（2 项）。归档复用既有确认（运行中/含子代理弹既有确认 + 级联归档），不在此重做语义。
- **点击效果**：点箭头→展开/折叠该节点；状态点→（仅释义，hover 提示其含义）；进入对话→交接主壳；筛选 chip→切换该筛选维度；清除筛选→还原全树。

### `#mc4` 状态合集（叠加在 mc1/mc2/mc3 上的态，集中存档）

- **空态（无代理）**：插画 + 一句话 + 「起一个代理」CTA → 交接既有新对话流程。**与离线态明确区分**（空 = 主机在线但没代理）。
- **加载骨架**：总览/队列/树各自的占位骨架，无闪烁、无误导空。
- **某主机离线降级**：该主机组置灰 + 离线标记 + 「重连」入口 + 陈旧 last-known 数据 + 动作禁用；其它主机不受影响；唯一主机离线 → 整页离线/重连态。
- **全局错误**：聚合彻底失败 → 错误说明 + 「重试」。
- **右键菜单**：无（这些是态，不是独立屏）。
- **点击效果**：空态 CTA→新对话流程；离线「重连」→重连该主机；全局错误「重试」→重新聚合。

## 5. 边界与依赖

**复用的既有能力（不新增、不重做）**

- **状态聚合来源**：代理生命周期状态（初始化 / 空闲 / 运行 / 报错 / 已结束）、代理输出流（驱动「当前动作」标签）、权限请求/解决流、父子嵌套关系（父代理关系 / 子代理轨道 / 级联归档）、工作区活跃聚合、回合完成/失败/取消的终结信号（驱动「已完成」桶）。这些**都已存在**。
- **原地处置**：复用既有权限流（代理→服务端→客户端→用户→服务端→代理）的批准/拒绝/总是允许，复用既有发送 prompt 能力（行内回话），复用既有重试与归档（含确认 + 级联）。指挥舱不重新设计这些机制，只提供纵览入口与紧凑预览。
- **分桶定义（互斥，按优先级各归一桶）**：报错（报错态）＞需关注（有未决权限请求 或 在等你回话）＞运行（运行中且无未决权限）＞已完成（会话已结束/终结）＞空闲待命（空闲且无人等你）。「需要你」队列 = 需关注 + 报错（即三类行）。
- **筛选与「进入对话」**：筛选是对**已聚合代理**的客户端内过滤（当前主机无需新查询能力）；进入对话是对主壳的导航交接。

**能力门（诚实标注 · 依赖，不写实现）**

- **能力门①｜跨主机同时聚合**：主价值锁定「**当前已连接的主机**（常见＝1 台）上的多代理纵览」，**无需新增能力**。同时聚合**多台主机**需要客户端**同时持有多个主机的实时连接并合流**——属能力门：能力具备则总览/树/筛选按主机分组扩展到多主机；不具备则视图自然收敛到当前已连接主机（不报错、不假装有别的主机）。
- **能力门②｜成本/token 脉冲**：若聚合用量数据**尚不可得**，则此指标为**可选、能力门控**，**不作为核心**；不可得时整块不出现，不影响纵览与处置主价值。
- **能力门③｜「等你输入」与普通空闲的区分**：队列要可信（秒清），就需要一个可靠的「代理在等你回话」信号把它与「单纯空闲待命」区分开。该信号可靠可得则填充「等你输入」行；**不可靠/不可得时取保守口径**——这些代理留在「空闲待命」桶而非涌入队列，宁可少报不可错报，保住队列的可信度。
- **协议/兼容口径**：上述任何新能力，按仓库精神以**主机能力探测**对待——具备则启用该子能力，不具备则就地提示「需升级主机以使用该功能」，**不写降级/回退实现路径**（本文只表述为依赖，不写 HOW）。

**平台**：本轮**桌面 only**；浅色主题；accent `#20744A`。手机/紧凑布局不在本需求。

**与现有功能的关系**：指挥舱是**侧栏新顶层入口的独立视图**，与单对话主壳并列；不取代主壳、不改主壳布局，仅通过「进入对话」单向交接；不触碰设置 IA。

## 6. 验收标准（逐条可判定 · 能/否）

1. 侧栏存在「指挥舱」顶层入口；当存在「需要你」事项时，入口显示其总数徽标（待批权限+等你输入+报错），为 0 时不显徽标。
2. 打开指挥舱，总览 `#mc1` 显示 5 个分桶计数（运行 / 空闲待命 / 需关注 / 报错 / 已完成）+ 按主机/项目分组的活跃总览 + 「需要你」队列入口。
3. 分桶计数与真实代理状态一致：每个（已连接主机上的）代理恰好计入一个桶，各桶之和等于代理总数（可抽查：制造一条报错、一条待批权限，分别落入「报错」「需关注」而非「运行」）。
4. 点某分桶计数能跳到对应子集（如点「报错」定位到报错项）；点「需要你」入口进入 `#mc2`。
5. `#mc2` 队列含且仅含三类行：待批权限 / 等你输入 / 报错；每行带主机/项目/提供方标签与一句摘要。
6. 展开「待批权限」行能看到工具调用摘要 + 命令/文件/diff 紧凑预览，并提供 批准 / 拒绝 / 总是允许 / 进入对话 四个动作。
7. 点「批准」后：该权限被放行、对应代理继续运行、该行离开队列、入口徽标计数随之减少（端到端真生效，非仅界面变化）。
8. 点「拒绝」「总是允许」分别产生既有权限流的拒绝/记住作用域并放行效果，且该行离开队列。
9. 展开「等你输入」行出现行内迷你 composer；输入并发送后该代理进入运行、该行离开队列；该迷你 composer **不含** slash/@/附件/语音/队列轨（富输入须经「进入对话」）。
10. 「报错」行提供 重试 / 进入对话；点重试使该代理重跑，再次失败则回到报错行（摘要更新）。
11. 任一行的「进入对话」能把主壳导航并聚焦到该代理对话；从该对话可经侧栏「指挥舱」入口返回。
12. 处置失败（请求超时/被撤销、主机离线、被他端抢先）时，对应行**就地**显示失败/已处理原因并给出重试或进入对话，**不静默丢失**。
13. `#mc3` 能从 root 逐层展开 subagent 至任意深度（无限嵌套），每行显示状态点 + 当前动作 + 主机/项目/提供方标签。
14. `#mc3` 筛选条可按 主机 / 项目 / 提供方 / 状态 过滤（可叠加）；无匹配时显示「无匹配，清除筛选」，清除后还原全树。
15. `#mc3` 行 hover 显示「进入对话」快捷动作；右键菜单恰为「进入对话 / 归档」两项；归档走既有确认（运行中/含子代理弹确认并级联），确认后该代理及其子树从树中消失。
16. 空态、加载骨架、全局错误三态均存在且可区分；空态（主机在线无代理）提供「起一个代理」并交接既有新对话流程。
17. 某主机离线时，该主机分组/行降级为离线（置灰 + 离线标记 + 重连 + 陈旧数据 + 动作禁用），其它主机不受影响；唯一主机离线时整页进入离线/重连态而**非**误报空态；重连后恢复可操作。
18. **当前已连接主机**上的纵览与处置**不依赖任何新增 daemon/客户端能力**即可工作；跨多主机同时聚合按能力门处理（能力不具备时视图收敛到当前主机，不报错、不虚构其它主机）。
19. 成本/token 脉冲为可选指标：聚合用量不可得时该块整体不出现，且不影响纵览与处置主价值。
20. 「等你输入」与「空闲待命」可区分：可靠信号不可得时，相关代理留在「空闲待命」桶而不涌入「需要你」队列（队列不出现纯空闲噪声）。
21. 每屏的完整交互态（默认/hover/活跃/运行/禁用/空/加载/错误/离线/右键/点击效果）与配套 [ui.html](./ui.html) 的 `#mc1`/`#mc2`/`#mc3`/`#mc4` 注解逐态一致（抽查 hover / 活跃 / 加载 / 离线 各至少一处）。
22. 指挥舱**不重新实现**权限弹窗、归档确认、级联归档、审查/diff 面板、新建代理流程——以上均复用既有能力（可核查：归档确认文案/行为与既有一致，diff 仅为紧凑预览，富审查经「进入对话」）。
