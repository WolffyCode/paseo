## 1. 目标 & 用户价值

**一句话目标**：让桌面用户在一处**看全 Helm 的全部键盘快捷键、并能把任意一条改成自己习惯的键**，按用途分 5 组、可单条恢复默认、可一键全部恢复默认。

**用户为什么需要**：

- **「这个操作的快捷键是什么 / 能不能改？」** —— 重度桌面用户靠键盘驱动一切（切工作区、拆窗格、开终端、唤起命令中心）。一张**分组的全量快捷键表**让人不必猜、不必翻文档；**改键**让冲突或不顺手的默认键可被纠正。
- **分组降认知负担** —— 41 个动作平铺会淹没人。按**导航 / 标签与窗格 / 项目 / 面板 / 代理输入** 5 组归类，用户按「我想干的事属于哪类」快速定位。
- **改错了能退回** —— 改键难免试错；**行级「重置」**让单条回默认、**段头「全部重置」**让一键全清自定义，用户敢于尝试。
- **诚实对待平台** —— 快捷键依赖物理键盘；**移动端没有键盘**，整段显示「移动端不可用」而非给一堆点不动的行。

> **本轮边界（董事长定 · 关键）**：现有键位映射**不一定准**（已有实证，见 §3 脚注）。**本轮只搭架子**——分组、每行结构、改键 / 重置交互全部做完整、可用；**但每个动作的「当前键帽」只作占位、统一标注「待校正」**。准确的键位映射在后续轮次**统一修正**，本轮**不逐条改键位**。

## 2. 范围

### 做（本 tab 覆盖）

- **5 组共 41 个动作的清单**（分组与动作见 §3），固定组序：导航 → 标签与窗格 → 项目 → 面板 → 代理输入。
- **每行三件套**：动作名（中文 + 英文 label）+ 当前键帽（占位·待校正）+「改键」入口。
- **改键交互全流程**：进入捕获 → 实时显示所按组合 → 完成保存 / 取消 / 离开页面自动取消；支持多段 chord 序列；Backspace 删最后一段。
- **行级「重置」**：该行存在自定义时显示，恢复该行默认。
- **段头「全部重置」**：存在任意自定义时显示，恢复全部默认。
- **平台渲染规则**：mod 键帽随平台（Mac ⌘ / 其它 Ctrl）；移动端整段显示「移动端不可用」。

### 边界与依赖

- **平台**：面向**有物理键盘的平台**（桌面 App + Web 浏览器）；**移动端（native）整段只显示「移动端不可用」占位**，不列任何快捷键、不可改键。（inventory §1 标「桌面 only」，精确含义即「非移动端」。）
- **键位准确性边界**：见 §1 / §3 脚注——本轮键帽为占位，准确映射后续统一校正。
- **关系**：本 tab 属应用段（设备级 / App 自身），与「当前选中主机」无关；自定义键位是**本设备 / 本 App** 偏好。

### 不做（本轮显式排除）

- **不实现、不改任何代码、不碰 daemon**（本轮只到内容明细 + 设计稿）。
- **不在本轮逐条校正键位映射**（架子优先；准确键位统一修正留后续轮次）。
- **不新增 / 不删减动作**（就这 41 条，对齐现役全量帮助项；不臆造新快捷键）。
- **不做**「导入 / 导出快捷键方案」「多套预设方案切换」「按动作搜索过滤」——本轮无人提出需求，YAGNI，不加。
- **不做**冲突检测器（改键撞车时的自动告警/阻止）——列入开放项（§6），本轮不承诺。

## 4. 核心流程（端到端 · 每条有入口有出口、无死胡同）

### A · 改键（Rebind）
1. **入口**：某行点「改键 / Rebind」→ 该行进入**捕获态**（行高亮，期间全局快捷键被抑制，避免误触发别的动作）。
2. **等待**：未按任何键时，行内显示提示 **Press shortcut…**（按下快捷键…）。
3. **实时反馈**：用户按键时，行内**实时显示已按下的组合**；若只按住修饰键（如仅 ⌘/Shift）尚未构成完整组合，则实时显示当前按住的修饰键。
4. **多段 chord**：每按下一个完整组合即追加为序列中的一段，支持「先 A 再 B」的多段快捷键。
5. **删错**：按 **Backspace** 删除已捕获序列的**最后一段**（逐段回退）。
6. **出口·完成**：捕获到至少一段后出现「完成 / Done」→ 点击**保存为该行自定义**、退出捕获、该行随即出现「重置」。
7. **出口·取消**：捕获态点「取消 / Cancel」→ 放弃本次捕获、恢复该行原键帽、不写入。
8. **出口·离开自动取消**：捕获中若**离开本页（页面失焦）**→ **自动取消**本次捕获，不残留半截捕获态，无死胡同。

### B · 行级重置（Reset）
- **入口**：某行已有自定义（override）时，该行显示「重置 / Reset」。
- **动作**：点击 → 该行**恢复默认键帽**、自定义清除、「重置」按钮随之消失。
- **出口**：回到该行默认态。

### C · 段头全部重置（Reset all）
- **入口**：**任意一行存在自定义**时，列表段头出现「全部重置 / Reset all」。
- **动作**：点击 → **所有自定义一次清空、全部恢复默认**、各行「重置」与段头「全部重置」一并消失。
- **出口**：整张表回到全默认态。

### D · 移动端（native）
- **入口**：移动端进入「快捷键」tab。
- **表现**：整段只显示一句 **Keyboard shortcuts are only available on desktop**（键盘快捷键仅在桌面端可用）占位，**不列动作、无改键**。
- **出口**：用户知情，非错误、非空白，无死胡同。

## 5. 全交互态枚举（逐屏 · 引用 ui.html 锚点）

> 与 UI 设计师同步：下列每个状态都要在 `shortcuts/ui.html` 有对应屏/态。锚点名供 UI 对齐，最终以 ui.html 逐屏注解为准。

### `ui.html#s-shortcuts`（快捷键 tab · 桌面默认态）
- 5 组分段标题 + 各组若干行；每行：左动作名，右侧「键帽 + 改键」。
- 默认：键帽显示当前（占位·待校正）值；行无自定义时**不显**「重置」；无任何自定义时段头**不显**「全部重置」。
- hover：行 hover、「改键」按钮 hover 反馈；键帽本身不可点。
- 点击效果：点「改键」→ 进入捕获态（见下）。

### `ui.html#s-shortcuts-capture-empty`（捕获态 · 等待）
- 目标行高亮；键帽位置显示「Press shortcut…」提示；出现「取消」（尚无「完成」，因未捕获）。

### `ui.html#s-shortcuts-capture-combo`（捕获态 · 已按组合）
- 目标行高亮；实时显示已捕获的组合 / 多段序列（或仅按住的修饰键）；出现「完成」+「取消」；Backspace 可逐段回退。

### `ui.html#s-shortcuts-override`（某行有自定义）
- 该行键帽显示自定义值；该行出现「重置」；段头出现「全部重置」。

### `ui.html#s-shortcuts-reset-all`（段头全部重置可见）
- 存在任意自定义时，列表段头显示「全部重置」；点击后回到全默认（→ 回 `s-shortcuts`）。

### `ui.html#s-shortcuts-mobile`（移动端不可用）
- 整段单条占位文案「键盘快捷键仅在桌面端可用」，无列表、无操作。

## 6. 开放项（送董事长闸-1 定夺）

1. **★ 键位统一校正的时机与口径**：本轮键帽全为占位·待校正（已知至少「审查变更」显示 Ctrl 实为 ⌘ 的实证）。**何时、按什么口径统一校正**全部 41 条键位（哪一轮、是否需要逐条人工核对实际触发组合）？请闸-1 定。
2. **★ 改键冲突检测**：用户把某动作改成已被别的动作占用的组合时，是否需要**冲突告警 / 阻止 / 覆盖提示**？本轮架子**不含**冲突检测，列为开放项——闸-1 决定是否纳入后续轮次（避免范围蔓延，先不做）。
3. **★ Web 浏览器端是否完全开放改键**：本 tab 在 Web 浏览器（有键盘）默认与桌面一致可用；但部分组合可能被浏览器自身占用（如 ⌘W / ⌘T）。是否需要对 Web 端某些系统级组合给出提示，还是统一交给「键位校正」轮次处理？
4. **★ 动作中文命名定稿**：41 个动作的**中文名**（如「工作树 worktree」「侧边聊天 side chat」「命令中心 command center」）是否与全局术语表 / 主壳 UI 用词一致？请在闸-1 对齐术语，避免设置里与主界面叫法不一。
