交互注解 · 快捷键
点击效果（逐控件）
  1. 1
    左导航 · 快捷键（选中）：桌面专属（带笔记本图标）。选中 = sidebar-accent 淡灰填充（#eaeef2）+ 深灰字，无左竖条。点其它 tab 切换右详情。
  2. 2
    页眉「键位映射待统一校正」橙 badge：整页架子标识——提醒键位为占位、后续统一校正。纯提示，不可点。
  3. 3
    全部重置（首段段头右）：仅当存在任意 override 时出现（hasOverrides）。点击 resetAll() 清除所有自定义键位，恢复默认。
  4. 4
    改键（每行常驻 · faint）：默认低对比，行 hover 时加深（本行示范 hover 态）。点击进入捕获态：行高亮 + 监听按键、实时显示已捕获组合（见 ⑤ + S2）。native（移动端）不可用。
  5. 5
    捕获态（本行示范）：行进入 rowCapturing 高亮，键帽区实时显示捕获到的组合（⌘⇧K）；捕获到 ≥1 组合即出现完成（保存 override）；取消放弃返回原值；Backspace 删最后一个组合。空捕获时显「按下快捷键…」（见 S2-②）。
  6. 6
    行级重置（仅 override 行）：该行已自定义（左侧蓝色圆点 #0969da + 键帽为自定义组合）。点击 removeOverride 恢复该行默认键位。无 override 的行不显「重置」。
  7. 7
    待校正行：「审查变更」help.keys=⌃⇧G（灰显）但实际 combo=⌘⇧G——已标 待校正。仍可改键，校正在后续统一批次处理。
状态（含布局）
布局
桌面优先：master-detail，左导航 248 + 右详情滚动。5 组分段卡纵向堆叠，行 = 动作名(左) + 键帽 + 改键/重置(右)。紧凑：右详情占满、键帽可换行。
默认
键帽显当前绑定；改键常驻 faint；无 override 行不显重置；无 capture 时全列表静止。
悬浮
行 hover 显淡灰底（accent #f6f8fa）+ 改键加深（见 ④）。
选中 / 捕获中
改键中的行 rowCapturing 高亮 + 内描边；键帽区转实时捕获显示 + 完成/取消（见 ⑤ + S2）。同一时刻只有一行处于捕获态；切走 tab 自动取消。
禁用
移动端（native）整段不可用：仅显「键盘快捷键仅在桌面端可用」（见 S2 底）。
空
动作集固定 41，无空态；无 override 时段头不显「全部重置」、各行不显「重置」。
加载 / 错误 / 离线
本地客户端配置，不涉及加载 / 网络 / 离线（与已连接 Host 无关）；改键失败极少，保存失败则保留捕获态可重试。
右键菜单
本页无右键菜单——每行用内联「改键 / 重置」按钮 + 捕获态「完成 / 取消」承载全部操作，不走右击上下文菜单；段头「全部重置」亦为常驻按钮。

交互注解 · 改键状态机
捕获流程
  1. 1
    默认 → 捕获：点「改键」进入捕获，行高亮，全局监听 keydown（preventDefault 拦截组合，不触发实际快捷键）。
  2. 2
    等待：未捕获到完整组合时显「按下快捷键…」+ 闪烁光标；仅「取消」可用。按住的修饰键实时回显（heldModifiers）。
  3. 3
    已捕获：捕获到 ≥1 组合即实时显示键帽并出现「完成」；支持多段 chord（连续组合）；Backspace 删最后一段。
  4. 4
    完成 / 取消：完成 = setOverride 存自定义键位（行出现蓝色圆点 + 重置）；取消 = 丢弃返回原值。切走 tab（失焦）自动取消。
  5. 5
    重置 / 全部重置：行级重置只清本行；段头「全部重置」清所有 override（均仅在存在 override 时出现）。
冲突提示 + 紧凑换行

改键撞车的冲突提示（开放项预览 · 待闸-1）· 与紧凑窄栏自适应

左 = 改键冲突提示：当捕获到的组合已被别的动作占用时如何呈现。 开放项预览——对应 requirement §6 开放项 2，本轮架子默认不含冲突检测，此屏仅供闸-1 决策「是否纳入后续轮次」的形态参考，非已承诺范围。 右 = 紧凑窄栏：右详情变窄时，动作名自动换行、键帽组保持整组右对齐不拆行（组合键绝不跨行断开），改键按钮始终可点。

交互注解 · 冲突提示 + 紧凑
点击效果（逐控件 · 冲突为开放项预览）
  1. 1
    冲突提示条（开放项预览）：捕获到的组合（⌘P）已绑定别的动作（打开文件）→ 红色描边行 + 红键帽 + 下方提示「⌘P 已被『打开文件』占用」。仅当闸-1 决定纳入冲突检测时存在；否则改键允许重复绑定、不提示。
  2. 2
    「完成」禁用 + 两出口（预览方案）：冲突时「完成」置灰不可保存；提供 覆盖它（夺走「打开文件」的绑定，使其变未绑定）/ 换一个（清空重新捕获）；「取消」放弃返回原值。最终交互口径以闸-1 决策为准。
  3. 3
    紧凑窄栏自适应：右详情变窄（紧凑桌面 / 分栏）时，动作名换到第二行，键帽组（如 ⌘⇧⌥←）保持整组完整、右对齐不拆行——组合键绝不跨行断开以免误读；改键按钮始终右侧可点。纯布局自适应，无新交互。
