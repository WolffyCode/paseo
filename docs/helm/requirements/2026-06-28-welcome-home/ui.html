交互注解
点击效果（逐控件 · 中区首页）
  1. 1
    品牌图标（hero · 蓝圆角块 + gauge）：纯品牌标识、不可点（hover 无态）。与顶栏看板 logo / 应用图标同源，强化「这是 Helm」。
  2. 2
    时段问候语 + 友好提示：问候语随本地时段变化——凌晨「夜深了」/ 早「早上好」/ 午「下午好」/ 晚「晚上好」；提示语随上下文细化：有当前项目 → 含项目名（「在 paseo-main 构建点什么」）、无项目 → 通用版（见 s2）、无模型 → 让位给引导条（见 s4）。纯文案、不可点。
  3. 3
    中央 Composer 输入：首页主入口。点击 / 聚焦 → 进入聚焦态（蓝光晕环 + hero 缩小让位，见 s3）；输入并发送第一条 → 中区切换为消息流（离开着陆态、进入普通对话页）。Composer 内部（附件 / 思考 / 联网 / 模型）以外壳既有 Composer 为准，本需求不重定义。
  4. 4
    模型选择器（Composer 内联）：提供方🔒 · 中转站 · 模型三层，沿用外壳既有级联。本需求不重定义其菜单。无可用模型时转 warning 态、见 s4。
  5. 5
    当前上下文 chips（项目 / 模式 / 分支）：表示「即将发出的这条新对话」的上下文，属外壳 Composer 既有组成（弱化展示）。点项目 chip 可改当前项目、点分支可改分支、点模式可切模式 —— 沿用外壳，本需求不重定义。
  6. 6
    「搜索」（最近对话栏头）：打开全局会话搜索（与左栏「搜索会话」/ ⌘K 同一入口）。hover 浮极淡灰底 #f6f8fa。
  7. 7
    最近对话项：点击 → 进入该对话的消息流（中区切为消息流、左栏对应节点高亮选中）。hover → 整行淡灰底 #f6f8fa + 行尾浮「→」（时间让位）。各取最近几项，不复制整棵左栏对话树。
  8. 8
    运行中状态点（最近对话项）：绿点 + 「运行中」= 该对话后台流式中（与左栏对话树绿运行点同源）；点进即看实时输出。idle / 完成项显相对时间。
  9. 9
    「打开项目」（最近项目栏头 · primary 蓝字）：打开已知项目选择器，选中后以该项目为上下文准备新对话（中区回到首页态、context chips 切到该项目）。hover 浮极淡蓝底。
  10. 10
    最近项目项：点击 → 以该项目为上下文准备新对话（不直接进消息流，停在首页态、Composer 待输入；发出首条才进消息流）。hover 淡灰底 + 行尾「→」。副行 = 工作目录路径（mono）。
  11. 11
    「选择工作目录…」（虚线行）：打开系统目录选择器，挑一个本地目录作为新项目上下文（首次 / 项目不在最近列表时用）。hover → 边框转蓝虚线 + 淡灰底。
状态（含布局 + 深色变体）
布局
桌面优先：照搬外壳 —— 统一顶栏 h40 + 内容行（左栏 240 · 中区 flex-1 · 右栏 / 目录树本屏收起）。中区首页内容居中、max-width 768，自上而下 hero → Composer → 最近两栏（grid 1:1）；内容超高时中区内部滚动。紧凑 / 手机态：后续（随外壳统一适配 —— 最近两栏堆叠为单列、hero 收紧、Composer 贴顶）。
默认
左栏展开且无选中项（中区无活跃对话）；中区呈完整丰富首页；Composer 未聚焦（无蓝环）、发送键 dim；主机在线（绿点）。
悬浮
最近项 / 栏头按钮 / 「选择工作目录」hover 显淡灰底 #f6f8fa（项目栏「打开项目」为淡蓝底）；最近项 hover 行尾浮「→」、时间让位；顶栏 / 左栏 hover 同外壳。
活跃·选中
首页态本身无 active 选中项（无活跃对话）。点击最近对话后该项瞬时 pressed（淡灰 #eaeef2）随即切走。左栏开关为已展开淡灰填充态（外壳态）。
运行
「修复侧边栏 toggle」最近对话显绿点 + 运行中（后台流式）；左栏对话树同步绿运行点。
禁用
Composer 空输入 → 发送键 dim（不可发）。无可用模型 → 整个 Composer 禁用 + 引导条，见 s4。
空
无最近项目 / 对话 → 见 s2（最近区转引导态，hero + Composer 仍在）。
加载
拉取最近项 / 主机刚连上时，最近两栏显骨架占位行（灰条）；切主机时 hero 提示语转「正在连接…」。细节后续。
错误 / 离线
主机离线 → 顶栏主机切换器红点 + 中区顶部细提示条「主机已离线 · 重连」，最近区转只读灰态（不可点开）。细节随主机模块对齐后补。
深色变体
同一套 codePilot 令牌（.dark）自动适配：中区卡 #161b22、字 #e6edf3、边框 #30363d、最近项 hover #1c2128、品牌块 / 主蓝 #1f6feb、运行绿点不变。本稿只渲浅色，深色不另画屏。
右键菜单
最近对话项右键（4 项 · 与左栏对话树根行语义一致）
打开 — 进入该对话消息流（= 左键）。
重命名 — 行内改对话标题。
复制 ID — 复制会话 ID + toast。
从最近移除 — 仅移出最近列表（不删对话本体）。
最近项目项右键（4 项）
以此开始新对话 — 以该项目为上下文准备新对话（= 左键）。
在 Finder 显示 — 打开工作目录所在位置。
复制路径 — 复制工作目录绝对路径。
从最近移除 — 移出最近列表（不删本地目录）。
最近区右键语义对齐既有左栏对话树 / 项目右键，只是入口在首页；危险项仅「从最近移除」（不触碰对话 / 目录本体），归档 / 彻底删除仍走左栏对话树。Composer / hero / 上下文 chips 无右键菜单。
与外壳的边界：顶栏 ··· 菜单（分屏 / 重命名 / 复制 ID / 删除）、左栏对话树右键、模型选择器级联全部照搬 codepilot-layout / home-shell，本需求不改；只新增「最近对话 / 最近项目」两个首页专属右键。

交互注解
点击效果（逐控件 · 无模型态）
  1. 1
    「去配置」引导抓手（primary 蓝按钮）：本屏首要抓手。点击 → 进入配置入口（provider / 模型配置，属另一需求，本屏不含其界面）。默认 = 蓝实底白字；hover = 转深蓝；配置完成回首页 → 引导条消失、Composer 恢复可用、模型选择器显真实模型。
  2. 2
    Composer（禁用态）：整框淡化（opacity 0.55）、占位文案改「先配置模型后即可开始对话…」、不可聚焦发送。这是「为什么开不了工」的就地解释，配合上方引导条。
  3. 3
    模型选择器（warning 态）：橙边 + alert 图标 + 「未配置模型」。点击 → 同样跳配置入口（与「去配置」同目的地，给第二个抓手）。
  4. 4
    最近对话 / 最近项目（仍可查看）：点最近对话 → 进该对话消息流可查看历史；但要继续发消息仍会被拦到配置入口。点最近项目 / 打开项目同样先配置再开干。
状态（含布局 + 深色变体）
布局
同 s1 外壳；中区在 hero 与 Composer 之间插入一条引导条（整宽、警告橙边）。紧凑 / 手机态：后续（引导条「去配置」按钮换行到条下方）。
错误 / 边界（本屏主态）
无可用 provider / 模型：引导条 + Composer 禁用 + 模型选择器 warning；最近区可读不可发。这是边界引导态，非崩溃错误。
默认 / 悬浮
「去配置」hover 转深蓝；模型选择器 warning hover 加深橙边；最近项 hover 同 s1。
活跃 / 运行
无可用模型 → 无法发起新运行；已存在的后台运行对话（若有）仍在左栏显运行点、可查看。
禁用
Composer 输入 + 发送整体禁用（直到配置完成）；思考 / 联网 / 附件等工具一并灰置。
空 / 加载
「无模型」与「无最近项」可叠加 → 引导条 + 空态引导卡（s2）同现；检测模型可用性时引导条先显加载骨架。细节后续。
离线
主机离线时优先显「主机离线」提示（高于无模型引导）；恢复在线再判模型可用性。
深色变体
引导条 = 橙边 #d29922 + 暗橙底；模型选择器 warning 同色；「去配置」#1f6feb；禁用 Composer 降透明度逻辑不变。
右键菜单
引导条 / 禁用 Composer 无右键菜单；最近对话 / 最近项目右键同 s1（但「打开 / 以此开始新对话」最终仍引导去配置）。
边界声明：本屏只负责「引导去配置」这一步；provider / 模型的配置界面本身属另一需求（主机 / 设置相关），不在本首页设计范围。配置完成后回首页即恢复 s1 正常开干态。
