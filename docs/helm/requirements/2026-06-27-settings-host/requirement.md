## 1. 目标 & 用户价值

**一句话目标**：把 Helm 设置的**主机段**按 **CodePilot GitHub-light** 主题**换肤重出**，并把这一期的**实现切到一条清晰边界上**——本期只建「主机段界面 + 主机配置的结构化读写」，**暂不把所配的中转站 / 模型接进对话流**。主机段的**功能结构 100% 沿用 2026-06-26-settings，不增不减**。

**用户为什么需要**（用户价值沿用 2026-06-26，本需求新增视觉一致性）：

- **观感与新主布局统一**：并行的「CodePilot 主布局」项目把主界面换成了 GitHub-light 的浅色专业观感；设置作为同一个产品的一部分，**若停留在旧配色就会割裂**。主机段换上同一份 token，用户在主布局 ↔ 设置之间来回不出戏。
- **普通用户也能配通一条链路**（沿用）：选内置提供方（CLI）→ 接中转站（base_url + key）→ 放出模型；界面只留常用项、少术语，会用就行。
- **主机是根上下文**（沿用）：连接配对 / 运维 / 模型与提供方 / 用量都挂在「当前选中的主机」下；主机段默认进当前主机，有明确「当前」标识。
- **高级用户有逃生舱**（沿用）：「打开配置文件（JSON）」入口兜底界面没暴露的高级项——有它，GUI 才敢做简单。

> **为什么这样切实现边界**（项目层价值，详见 §6 决策）：此刻有多个项目同时在改对话流；若现在就把中转站 / 模型接进 composer，会和它们抢同一片代码、互相冲突。把「主机配置」与「对话接入」**切开**后，主机配置可在**隔离**下完整建好、**可评审、可单测**，不动别人正在改的对话流——风险最低、返工最少。

## 2. 范围

> 本需求的范围用三栏看：**做 · 本期建（UI + 协议）** / **做 · 视觉（换肤）** / **不做 · 暂缓 + 沿用搁置**。功能 WHAT 全部沿用 2026-06-26-settings，本文不重定义。

### 做 · 本期建 ①：设置 UI（主机段换肤重出）

- **设置外壳**（沿用）：master-detail；左导航 = 主机段（host 选择器默认当前主机 + 「当前」标识；主机 / 模型与提供方 / 用量）+ 配置文件（JSON）入口 + 灰置「应用设置 · 后续阶段」占位；进入 / 退出 / 默认落点 / 外壳级状态。
- **B1 主机**（沿用 + ★守护进程移入）：连接与配对 + 运维合并；常用留界面（身份一处 / 连接概览 / 重启 / 远程访问开关 / 配对手机二维码+链接 / **守护进程区块**）；relay 全字段 / TCP / service-proxy 折叠「高级」或入 config.json。**守护进程（本机内建）区块** = 董事长本次从应用段移入、挂当前主机、仅桌面的**已存在真实能力**（运行状态 / PID / 版本 + 版本不匹配警告 / 开停 / 退出后保持运行 / 日志 / 完整状态弹窗 / 高级设置外链），归**本期建 · 真实可用**、详见 §3。
- **B2 模型与提供方 · 三级**（沿用）：L1 提供方列表（内置 5 + 从真实目录新增 ACP、每行 CLI 已装/未装检测、未装一键安装、右键「删除」门控）→ L2 提供方设置（基础信息含命令 + 中转站列表 + 启停顶部一处）→ L3 中转站详情（base_url / key 密码态 / 端点测速 / 放出模型多选 + 设默认 / 高级折叠 + config.json 链接，非弹框面包屑返回）。
- **B3 用量**（沿用）：每提供方余额 / 已用 / 额度条；三处缺口（时间切换 / 估算金额 / 按模型）克制标「目标态」；依赖能力门 providerUsageList。
- **B4 配置文件（JSON）编辑器**（沿用）：现成 JSON 编辑器组件（行号 / 高亮 / 校验 / 格式化·恢复·保存），映射所选主机真实配置文件；界面没暴露的高级项在此改。
- **全交互态**（沿用）：默认 / 悬浮 / 活跃 / 运行 / 禁用 / 空 / 加载 / 错误 / 离线 + 右键菜单 + 即时持久化语义。**态不增不减，只换皮**。

### 做 · 本期建 ②：协议 / 配置 schema（主机配置的结构化读写）

- 让主机段所有 GUI 操作落到**所选主机的配置文件**（即 B4 逃生舱所编辑的同一份）上，做到**完整落地、读回、隔离生效、不影响别处**。覆盖的结构化配置项（能力级，不列字段名）：
  - **提供方**：内置 5 + 从目录新增项的**启停 / 新增 / 删除**状态。
  - **中转站（Vendor）**：每个中转站的 **base_url + key + 放出模型（多选）+ 默认模型 + 「当前」标记 + 高级项**的增删改。
  - **模型（Model）**：中转站放出的模型集合与「设默认 / thinking」等选择。
  - **relay / 中继**：远程访问通道的相关配置（折叠「高级」的全字段）。
- **隔离与可测**：在主机 A 配的提供方 / 中转站 / 模型 / relay **不串到**主机 B；切回主机 A 配置如实读回；配置读写的纯逻辑**可单测**（不渲染即可测）。
- **GUI 与 JSON 单一真相源**：本期建的结构化项在 B4 编辑器里可见，GUI 与 JSON 重叠字段**双向反映**。

### 做 · 视觉：换肤（CodePilot GitHub-light token）

- 主机段全段视觉规范 = **CodePilot GitHub-light token**，与「CodePilot 主布局」项目**共用单一 token 来源**，任何页面不得自带与之冲突的独立配色。
- token 性格（设计目标值，细节以 ui.html 色板为准）：**GitHub 蓝强调** + **冷灰中性盘** + **圆角卡片** + **柔阴影**。
  - ⚠️ 与并行「CodePilot 主布局」requirement 当前文字（暖灰/石色 + charcoal 近黑单色主色〔明确非彩色 accent〕+ 含深色）**存在出入**；二者既要求「同一份 token」，须在闸-1 调和（见 §7 开放项 ★）。本需求**不擅自二选一**。
- **仅视觉层**：换肤**不改**主机段任何功能、布局结构、信息架构、交互态数量——只换配色 / 圆角 / 阴影 / 卡片质感。

### 不做 · 暂缓（本需求显式排除）：对话接入

> 「协议影响的地方」——即把所配的**中转站 / 模型真正接进对话流**——本期**全部暂缓**。原因见 §1 / §6：多个项目正在同时改对话流，现在接会冲突。

- **不做**：home 对话框 composer 用「所选中转站 + 模型」发起对话。
- **不做**：「设为当前中转站」「设为默认模型」**真正驱动** composer（控件在 UI 上**照常出现、可操作**，选择**持久化进主机配置并在设置内回显**，但其**驱动对话流的接线**本期不实现——标「对话接入暂缓」）。
- **不做**：composer「管理中转站…」深链**回写** / 跨模块联动（深链**入口**本身在主机段一侧照常可达；composer 侧的回写联动暂缓）。
- **不做**：切 host 时 composer 中转站 / 模型**跟随切换**。
- 一句话：**本期把中转站 / 模型的「配置真相源」完整建好并落盘；它被对话流「消费」的那一段接线，本期不接。**

### 不做 · 沿用 2026-06-26 的搁置 / 删除 / 延后（不变）

- **应用段剩余 7 tab 整段搁置**（通用 / 外观 / 快捷键 / 集成 / 权限 / 诊断 / 关于）——移出 nav、灰置占位、设计稿留附录，内容不删。**注：原应用段的「守护进程」本次由董事长移入主机段 B1（见 §3），不再属应用段搁置项、改归本期建 · 真实可用。**
- **不做「自造命令 / 名字」的自定义提供方**——「+ 新增提供方」只从真实 ACP 目录选已存在的加入；任意自造走 config.json。
- **删除 tab**：Agents（主壳已覆盖）/ 工作区（主壳已能操作）/ 终端（PTY profiles 经 config.json 改）。
- **延后隐藏**：插件（MCP / 技能 / 目录），设计稿留附录、内容不删。
- **手机 / 紧凑态**：本轮桌面 only；添加主机归主壳 / onboarding，host 选择器只切换已连主机。
- **本需求不重定义主机段任何功能结构**：上述 B1–B4 的功能 WHAT 全部沿用，本文只复述 + 引用。

## 3. 主机段结构与每块 WHAT（沿用现有 + 引用）

> 每块：**沿用源** + 一句话作用 + 本需求的**换肤 / 边界增量** + UI 锚点。**功能 WHAT、全交互态均沿用 2026-06-26-settings 的同屏定义，本文不重列**（避免重复造）；下方只标「本需求改了什么」。

### 外壳 + host 选择器（沿用）
- 沿用：单一 master-detail；段头 host 选择器（当前主机 + 在线色 + 「当前」标识 + 展开切换，**不负责添加主机**）；切主机右详情重载、默认进当前主机「主机」tab。
- 本需求增量：**换肤** GitHub-light；功能不变。
- UI：`ui.html#hm1`（外壳现役范例）

### B1 主机（Host · 连接与运维合并 · 沿用 + ★守护进程移入）
- 沿用：身份一处（名 + 在线 + 版本 + 「当前主机」+ 重命名）/ 连接概览 / 重启服务 / 远程访问开关 / 配对手机；relay 全字段 / TCP / service-proxy 折叠「高级」或入 config.json；离线态红点 + callout + 常用禁用、高级标「离线只读，config.json 仍可编辑」。
- **★ 守护进程（本机内建）区块（董事长本次从应用段移入主机段 · 挂当前主机 · 仅桌面）**——本机内建的后台服务（这台主机的本地后台进程，**≠ relay 远程通道、≠ 中转站 vendor**）。这是**已存在的真实能力**、不涉对话流 → 归【本期建 · 真实可用】，**不属暂缓**，换肤即用。区块 WHAT：
  - **运行状态**：运行中 / 未运行 + **PID** + **版本**；**版本与 app 不匹配时给警告**。
  - **管理开关**：开 / 停内建守护进程。
  - **退出后保持运行**：开关——退出 app 后守护进程是否继续运行。
  - **日志**：查看日志路径 / 打开日志 / 复制路径（无日志则不可用）。
  - **完整状态**：查看完整运行状态（弹窗显示输出 + 复制）；含加载 / 失败态。
  - **高级设置**：外链入口。
  - 边界：**仅桌面、本机内建**（管理的是本机的内建后台服务）；加载 / 错误态显占位与重试；当前主机为远程时该内建区块不适用（远程主机后端由「连接概览 / 重启服务」覆盖，不在此堆死胡同）。
- 本需求增量：**换肤**；**守护进程区块 = 真实能力换肤即用（本期建·真实可用，非暂缓）**；**relay / 连接 / 配对相关结构化配置项的落盘读回 =【本期建②】**（relay = 加密远程通道，**≠ 中转站 vendor**）。配对只生成 offer、无设备列表 / 吊销（沿用）。
- UI：`ui.html#hm1`（守护进程区块在此屏内）、`ui.html#hm1-pair`、`ui.html#hm1-offline`；完整状态弹窗态见 `#hm1` 同屏注解。

### B2 模型与提供方（三级 · 沿用结构 · 本需求边界关键落点）
- 沿用：以提供方为中心配通链路——提供方（CLI）→ 中转站（base_url + key）→ 放出模型。
  - **L1 提供方列表**：内置 5 + 从目录新增 ACP；每行 CLI 已装/未装、模型数、开关、⚙ 设置；顶部「+ 新增提供方」= 从真实目录选已存在的加入；右键「删除」（目录新增＝真实删除、内置 5＝需新主机能力门控、不假装能删）；未装行「一键安装」（agent CLI 一键装可能需新主机能力，缺则回落安装指引外链）。UI：`ui.html#mp1`、`ui.html#mp1-add`
  - **L2 提供方设置**：基础信息（名称 / 图标 / 命令）+ 中转站列表 +「新增中转站」；启停顶部一处；面包屑返回。UI：`ui.html#mp2`
  - **L3 中转站详情**（drill-in 页 · 非弹框）：base_url / key（密码态）/ 端点测速 / 放出模型（多选 + 设默认 + thinking）；高级折叠（JSON / 限额 / 倍率）+ config.json 链接；删除中转站。UI：`ui.html#mp3`、`ui.html#mp5`
- 本需求增量：
  - **换肤** 三级全屏。
  - **【本期建②】**：L1 启停 / 新增 / 删除、L2 命令、L3 的 base_url / key / 测速结果 / 放出模型 / 设默认 / 「当前」标记，**全部落盘、读回、按主机隔离生效**。
  - **跨模块依赖的「拆半」（关键）**：2026-06-26 原文写「中转站配置是**唯一真相源**，驱动 home-shell composer 的中转站 / 模型选择与默认；composer『管理中转站…』深链回写；切 host 跟随」。本需求把它**拆成两半**：
    - **真相源本身 =【本期建】**：这份配置在主机配置里**完整建好、落盘、读回、隔离**，并在设置内回显（含「当前 / 默认」标记被记录）。
    - **真相源被 composer『消费』=【暂缓·对话接入】**：composer 读取它发起对话、深链回写、切 host 跟随——**本期不接线**（见 §2 暂缓栏、§4 流程暂缓点）。

### B3 用量（Usage · 只读 · 沿用）
- 沿用：每提供方余额 / 已用 / 额度条；时间切换 / 估算金额 / 按模型三处缺口克制标「目标态」；真实只到提供方维；依赖能力门 providerUsageList，缺失显「更新主机」。
- 本需求增量：**换肤**；功能不变。**无对话接入暂缓点**。
- UI：`ui.html#us1`、`ui.html#us2`

### B4 配置文件（JSON）编辑器（逃生舱 · 沿用）
- 沿用：现成 JSON 编辑器组件（行号 / 高亮 / JSON+schema 校验非法禁存 / 格式化·恢复·保存 / 查找）；映射所选主机真实配置文件；GUI 没暴露的高级项在此改；不自造区段导航、不自定义右键；离线只读。
- 本需求增量：**换肤**（编辑器主题随 GitHub-light）；**【本期建②】的结构化配置项**（提供方 / 中转站 / 模型 / relay / 「当前·默认」标记）应在此编辑器**可见、与 GUI 双向反映**——逃生舱即本期协议 schema 的「完整兜底面」。**无对话接入暂缓点**。
- UI：`ui.html#cfg1`

## 4. 核心流程（端到端 · 标注暂缓接线点）

> 每条有入口、有出口、无死胡同。流程结构沿用 2026-06-26；本需求只在涉及对话流的步骤上插入 **【暂缓·对话接入】** 截止标记。

1. **配通一条模型链路（三级）**：设置「模型与提供方」L1 启用提供方 → ⚙ 进 L2 → 选 / 新建中转站 → L3 配 base_url + key → 测速 → 放出模型 + 设默认 → 设为当前 →
   - **【本期建 · 流程在此完整闭合】**：以上每步**落盘 + 设置内回显**（含「当前 / 默认」标记被记录）；切 host / section 栈重置回 L1。
   - **【暂缓·对话接入 · 本期止步于此】**：home 对话框 composer 跟随生效（用所选中转站 + 模型发起对话）= **本期不接线**；主壳级联「管理中转站…」深链直达 L2 / L3 的**入口在**，但其**回写联动**暂缓。
2. **配对手机远程**：主机段「主机」→ 配对手机 → 展开二维码 / 链接 → 手机扫码接入。**relay / 配对配置落盘 =【本期建】**，**无暂缓点**。
3. **看用量**：主机段「用量」→ 每提供方余额 / 额度条直观；三处缺口标「目标态」；缺能力门显「更新主机」。**无暂缓点**。
4. **改高级项（逃生舱）**：左导航「配置文件（JSON）」→ 显当前主机配置 → 改界面没暴露的高级项 → 校验通过保存（需重启字段给提示）。各页「在 config.json 编辑」链接直达此处。**【本期建】的结构化项由此兜底完整**，**无暂缓点**。
