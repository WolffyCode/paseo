# 需求 · Claude 主子孙 subagent 对话树（原生树）

> 日期：2026-06-27 · 状态：评审中（待闸 1/闸 2） · 配套 UI：[ui.html](./ui.html) · 架构：[architecture.md](./architecture.md) · canonical：docs/helm/product.md + docs/helm/ui.html
> **只写 WHAT**。不含代码片段、函数名、文件改法、实现路子（见 [standards.md](../../standards.md) §7）。

## 1. 目标 & 用户价值

让用户在左侧对话树里看到一次任务派生出的 **完整主子孙 subagent 嵌套（任意层）**，并能点开 **任意一个** subagent 节点，看它 **真实、完整的对话**（消息 / 工具调用 / 思考），统一体验。

场景：用户在手机上盯着本机的 Claude 跑一个复杂任务。Claude 内部派出 Explore 子、Explore 又触发更深的 agent……用户想知道「那个孙 agent 到底在干嘛、说了啥」，而不是只看到一个 `Task running` 的黑盒。

价值：把 provider 内部编排的多层 subagent 从黑盒变成 **可观察的对话树**，与 paseo-run 调起的真实可控 agent 树 **同一棵树、同一种体验**。

## 2. 范围

**做：**
- 左侧对话树展示 主 → 子 → 孙 → …… **任意层** subagent 嵌套，所有来源并进 **同一棵树**。
- 每个 subagent 节点可点开看 **完整对话时间线**（只读视图）。
- **近实时**：任务跑动时——新节点近实时出现；节点状态（运行 / 完成 / 出错）近实时更新；已打开的 subagent 对话内容近实时增长。
- 统一处理几类 subagent 来源，外观 / 交互一致：
  - **provider 内部子**（Claude 的 Task 子、Codex 自己的 sub-agent）：一律 **只读**——归 provider 编排，用户不能发消息 / 不能中断 / 不能改其行为。**各 provider 可见深度不同，见下：**
    - **Claude = 本需求主交付**：任意层（主→子→孙→……）完整可见、可点开只读对话。
    - **Codex = 仅契约对齐**：本需求只把 Codex **沿用今天已有的「只读直接子」行为**接进同一套对外契约；**Codex 的更深层 subagent 不在本需求范围 / 硬验收内**（作为未来「加一个兼容层」的增量，UI / 公共契约不变）。
  - **paseo-run 子**（paseo 调起的真实独立 agent）：**完整可控**（已实现，树展示已有）——本需求保持不变、复用。
- daemon 重启后，能把 **已落盘** 的整棵树与每个 subagent 的对话 **完整重建**；重启发生在任务跑一半时，恢复后继续近实时更新到完成。

**不做：**
- 不让用户对 provider 内部子 **发消息 / 中断 / 改行为**（只读铁律）。
- 不自造一份 subagent 对话的存储 / 镜像 —— 直接以 provider 自己的原生来源为准。
- 不改 paseo-run 子现有的可控语义、归档 / 拆分 / 关页签策略（复用，不重造）。
- 不把不同 provider 的 subagent「翻译成统一语义」——统一的只是 **树形 + 只读对话的对外契约**，各来源各自读各自的真相。
- 不新增主壳 / 侧边栏布局（复用 home-shell 的对话树外观）；本需求只补「深层 subagent 可见 + 任意层可点开只读对话」这一增量。

## 3. 流程

**正常路径：**
1. 用户在对话树看一个正在跑的 Claude 主对话。
2. Claude 内部派出直接子 → 子节点近实时出现在主对话下，显示标题（任务描述 / agent 类型）+ 运行状态点。
3. 子又派孙 → 孙节点近实时出现在「对应那个子」之下（不是平铺、不是挂错父）；任意更深层同理。
4. 任一有后代的节点可展开 / 收起（复用现有树交互）。
5. 用户点任意层的 subagent 节点 → 对话区展示该 subagent 的 **完整对话**（只读），并随其跑动近实时增长。
6. 节点状态随其生命周期流转：运行中 →（正常结束）完成 / （失败）出错；状态点随之变化。

**分叉路径：**
- **只读约束**：点开的是 provider 内部子 → 对话视图 **无输入框、无中断**，以只读提示条替代；点开的是 paseo-run 子 → 正常可发 / 可中断。
- **主对话归档**：其下 provider 内部 subagent 子树进入只读历史态（本就只读），不可再变。
- **落盘缺失 / 损坏**：某 subagent 的对话读不出 → **只该节点** 的对话视图显示错误态，不连累整棵树的展示。
- **主机离线**：树与对话视图显示离线态（复用现有离线处理），重连后恢复。
- **旧主机（无此能力）**：客户端显示「升级主机以查看深层 subagent」提示，**不降级模拟**、不崩溃、不假装显示（能力门）。
- **daemon 重启**：已落盘的树与每个 subagent 对话被完整重建后照常展示；若重启时任务仍在跑，恢复后继续近实时更新。

## 4. 屏与全状态

> 本需求 **以复用为主**，引用 canonical / home-shell 的既有屏，仅补本需求新增 / 改动的态。详见 [ui.html](./ui.html)。

- **对话树（左栏）`#tree`**：复用 home-shell 对话树。节点态需覆盖：默认 / hover / 选中 / 运行（脉冲点）/ 完成（idle）/ 出错（error）/ 可展开（chevron + 后代计数 badge）/ 收起 / 深层缩进 / 树重建中（加载）。无 subagent 时不显子节点（空）。
- **subagent 只读对话视图 `#readonly-convo`**：复用现有 observed 只读视图。态需覆盖：加载（拉取对话中）/ 正常（只读时间线）/ 增长中（近实时追加）/ 空（尚无消息）/ 错误（该节点落盘读失败）/ 离线 / 顶部或底部 **只读提示条**（替代输入框）。
- **右键菜单**：
  - provider 内部子节点：只读相关项可用；改写类项（发消息 / 独立重命名 / 独立归档）禁用或不显示（与现有 observed 行为一致）。
  - paseo-run 子节点：保持现有可控右键（不变）。
- **能力门提示 `#upgrade-hint`**：连旧主机时，树只显示该主机能力范围内的层级，并给「升级主机」提示。

## 5. 边界与依赖

- **能力门**：依赖主机 daemon 提供「原生 subagent 树 读取 / 订阅」能力；客户端检测能力后才启用深层树，否则给升级提示（单点检测，不在功能里散落兼容分支）。
- **provider 差异（统一对外、各读各的）**：Claude 读其原生 subagent 落盘；Codex 读其真实 thread；paseo-run 用真实 agent 时间线。三者对外统一为「只读对话树 + 只读时间线」契约。**加新 provider = 加一个兼容层**，UI / 公共契约不变。
- **平台**：桌面优先；树与只读对话视图跨平台（复用现有跨平台组件），不新增平台分叉。
- **与现有功能关系**：复用 home-shell 对话树、phase-1 的 observed 只读（直接子）、phase-2 的 paseo-run 子树与其策略。本需求把「只直接子可见」扩成「任意层可见 + 任意层可点开只读对话」，且把直接子的数据来源与深层 **统一**。

## 6. 验收标准

> 每条可「能 / 否」判定，测试逐条核。
> **范围**：n 层嵌套类验收（#1–#3）以 **Claude** 为评测主场；Codex 内部子按其现状「只读直接子」行为核 #6 / #12，**不核 Codex 深层**。

1. 跑一个会派 ≥3 层 subagent 的 Claude 任务，对话树能显示出 主 → 子 → 孙（≥3 层）的正确嵌套。
2. 每个 subagent 节点的父子归属正确：孙挂在 **对的那个子** 之下，不平铺、不挂错父。
3. 点任意层 subagent 节点，能看到该 subagent 的 **完整对话**，内容与 provider 原生记录一致：**无缺漏**。
4. 直接子的对话 **不出现重复消息**（同一条消息不因来源不同而出现两遍）。
5. 任务跑动时，新节点 / 状态变化 / 已打开对话的增长均 **近实时**（秒级内出现）。
6. provider 内部子的对话视图 **无法发送、无法中断**（只读）；paseo-run 子可发可中断。
7. daemon 重启后，已落盘的树与每个 subagent 对话被 **完整重建**，展示与重启前一致。
8. 重启发生在任务跑一半时，恢复后该树 **继续近实时更新** 到任务完成。
9. 单棵树 subagent 文件很多时（数十个），树构建与对话加载 **无明显卡顿**。
10. 连旧 daemon（无能力）时，客户端给出「升级主机」提示，不崩溃、不假装显示深层。
11. 主对话归档后，其 provider 内部 subagent 子树为只读历史，不可改。
12. 同一棵树里 provider 内部子（只读）与 paseo-run 子（可控）能 **共存且各自行为正确**。
