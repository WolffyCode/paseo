# 需求 · 对话页（消息流 · 套 codePilot）

> 日期：2026-06-28 · 状态：草拟（gate-1 待审）· 配套 UI：[ui.html](./ui.html) · canonical：docs/helm/product.md
> **只写 WHAT**（见 [standards.md](../../standards.md) §7）—— 不含组件名 / 函数名 / 文件名 / 样式变量 / 代码片段 / 实现路子。CodePilot 仅作**观感参照对象**（一个开源项目），所有描述均为**能力级 / 形态级**。
> **一句话定位**：对话页 = 主界面外壳里「**中间画布**」在「**有活跃对话**」时的态 —— 一条对话的**消息流**（用户轮次 / Agent 轮次 / 工具调用 / 底部输入区）。它与[丰富首页](../2026-06-28-welcome-home/requirement.md)是**同一中间画布的两种态**：无活跃对话 = 丰富首页；有活跃对话 = 本消息流。**外壳与主题与首页完全一致**，本需求**只填中间画布的消息流内容与全部交互态**。

---

## 1. 目标 & 用户价值

**一句话目标**：把「中间画布 · 有活跃对话」态填成一条**可读、可控、状态清晰**的消息流 —— 用户能顺畅读懂 Agent 在说什么 / 做什么 / 做到哪了，并随时继续输入或打断。

**用户为什么需要**：
- 用户监控、指挥本地 AI 编码 Agent，**核心时间都花在读这条消息流上**：自己说了什么、Agent 怎么回、调用了哪些工具（读文件 / 改文件 / 跑命令）、改了哪些代码、成功还是失败。
- 现状外壳里这块只是占位（codepilot-layout 已明确「对话流·后续保持现状」）。**没有真正的消息形态、没有工具调用的状态区分、没有错误 / 流式 / 滚动等态** —— 用户读不清、控不住。
- 一条好的消息流要给用户四件事：**读得懂**（用户 / Agent 轮次分明、富文本清晰、代码与改动一眼可辨）、**看得到进度**（工具在跑 / 成功 / 失败、Agent 正在生成）、**控得住**（随时打断、继续追问、出错能重试）、**找得到位置**（长对话能滚、能一键回到底）。
- 整页套 codePilot 主题，与外壳、与首页**一眼是同一个 app**，态之间切换无割裂。

## 2. 范围

### 做（本需求 = 只填「中间画布 · 有活跃对话」的消息流内容与全交互态）

- **用户轮次**：用户发出的消息（靠右、气泡形态）；含其携带的附件 / 引用上下文的**展示**（不重定义输入区如何采集）。
- **Agent 轮次**：Agent 回复，**富文本（Markdown）**呈现 —— 段落 / 列表 / 行内代码 / 链接 / 小标题 / 强调等；带来源标识（哪个 Agent / 模型，弱化）。
- **工具调用块**：Agent 执行的每一次工具调用（读文件 / 改文件 / 跑命令 / 搜索 / 计划 / 待办…）以**可折叠块**呈现，覆盖全状态：
  - **折叠态**：一行摘要（工具名 + 关键参数，如文件名 / 命令）。
  - **展开态**：点开看详情（命令与输出 / 文件改动 / 结果）。
  - **运行中 / 成功 / 失败**：三态在折叠行即可一眼区分（运行有进行指示、成功 / 失败有明确标识，失败带错误摘要）。
  - **计划 / 待办**等结构化结果以对应的结构化卡片呈现（清单、勾选进度）。
- **代码与改动**：Agent 输出的**代码块**（等宽、语法着色、可复制）与**代码改动 diff**（增 / 删行分色、文件名与增删统计），二者清晰可读。
- **流式输出中**：Agent 正在生成时，逐字流入 + 生成光标；伴随的工具调用显示运行中；此时底部输入区进入**可打断**态。
- **空对话刚开始态**：用户刚发出第一条、Agent 尚未产出内容时的过渡态（已显示用户首条 + Agent「准备 / 思考中」指示，无正文）。区别于「无任何对话」的首页空态。
- **错误态**：Agent 轮次失败（如模型请求失败 / 工具执行报错）以**错误卡片**呈现，含错误摘要 + **重试**抓手；用户发送失败亦有**就地提示 + 重试**。
- **底部输入区（沿用外壳既有输入区，本需求只定义其在对话页的几种态与摆位）**：**空**（占位提示，发送不可用）/ **输入中**（有内容，发送可用）/ **发送中·运行中**（Agent 处理中，发送键转为**停止**；继续输入则**排队**）/ **禁用**（主机离线等不可用，整条输入区灰化 + 重连提示）。
- **长对话滚动 + 回到底**：消息流可上下滚动；用户上滚离开底部后出现**回到底部**抓手；Agent 在用户上滚时仍在产出，则抓手带**未读新消息**指示，点一下回到底并恢复跟随。
- **逐轮操作（hover / 右键）**：把指针移到某条消息上浮出该轮操作（复制 / 复制为 Markdown / 引用回复 / 从这一步重试…）；空白与各元素的 hover / 选中 / 运行态齐全。
- **主题**：整页套 **codePilot**（GitHub 浅色，默认），与外壳 / 首页**同一套主题来源**，**深色一并适配**；不自带独立配色。

### 不做（划清边界，防打架）

- **不重画外壳**：统一顶栏 / 4 张浮动卡片 / 三个独立开关 / 主题来源 / 拖拽，由 `2026-06-27-codepilot-layout` 负责；本需求**复用**外壳与主题，**只填中间画布消息流**，不动外壳布局与默认显隐。
- **不重定义底部输入区的内部能力**（附件 / 思考 / 联网 / 模型选择器等属外壳既有输入区）；本需求只定义其在对话页的**摆位与状态**（空 / 输入 / 停止 / 禁用 / 排队）。
- **不做首页**（无对话着陆态属 `2026-06-28-welcome-home`）；二者是同一中间画布的两种态，本需求只管「有对话」这一种。
- **不做右栏工作面板 / 目录树内部**（审查 / 终端 / 浏览器 / 文件预览 / 文件树属各自模块）；对话页只负责中间画布。点击代码 / 文件 / 改动等**跳到右栏**的目标内容不在本需求，本需求只负责**发起跳转这一步**的入口与反馈。
- **不做对话的增删改管理流程本身**（重命名 / 删除 / 归档 / 分屏等属外壳顶栏与左栏对话树）；对话页只**承接**这些已有入口。
- **不改协议 / RPC / 能力标志**；**无老用户 / 迁移**（fork 未上线）。

## 3. 流程（端到端）

> 同一「中间画布」的两种态切换；每态有入口、有出口，无死胡同。

- **进入消息流**：用户在首页中央输入区发出第一条 / 点左栏对话树某条对话 / 点首页「最近对话」某项 → 中间画布切为该对话的**消息流**。
- **首条已发 · 等待产出**：发出首条后 → 显示用户首条 + Agent「准备 / 思考中」指示（空对话刚开始态）→ Agent 开始产出即转入流式。
- **Agent 产出中（流式）**：Agent 文本逐字流入 + 生成光标；其间的工具调用以**运行中**块出现，完成后转**成功 / 失败**；底部输入区进入**停止**态。
- **用户打断**：在 Agent 产出中点**停止** → 停止本轮产出，已产出内容保留，输入区回到可输入态。
- **用户追问 / 排队**：Agent 仍在产出时用户继续输入并发出 → 该条**排队**，待本轮结束后自动接续（或用户可立即发送 / 编辑排队项）。
- **读历史 · 回到底**：用户上滚读历史 → 出现**回到底部**抓手；其间 Agent 仍产出则抓手显**未读新消息**；点抓手 → 回到底并恢复跟随最新。
- **出错 · 重试**：Agent 轮次失败 → 出错卡片 + **重试**；点重试 → 重发本轮 / 继续。发送失败 → 就地提示 + 重试。
- **逐轮操作**：hover 某条 → 浮出该轮操作；点「从这一步重试」→ 回退到该轮并重新产出（破坏性操作需明确，且与外壳既有「回退 / 重试」语义一致）。
- **回落首页**：用户新建对话（尚未发首条）或当前无任何活跃对话 → 中间画布回到[丰富首页](../2026-06-28-welcome-home/requirement.md)，离开消息流。
- **主机离线**：消息流所在主机掉线 → 输入区禁用 + 重连提示；重连后恢复可输入，历史消息保持可读。

## 4. 屏与全状态

> 逐屏对应 [ui.html](./ui.html) 锚点 `ui.html#sX`。桌面优先（紧凑 / 手机态随外壳后续统一适配）。**浅 / 深两套**：每屏默认 GitHub 浅色，深色随 codePilot 自动适配（在各屏注解块标深色变体）。每屏完整交互态以 ui.html 逐屏「交互注解块」（布局 / 状态 / 右键菜单 / 点击效果）为准。

- **s1 标准全景** `ui.html#s1`：消息流主视图 —— 用户气泡 + Agent 富文本 + 一个已完成工具调用（折叠）+ 底部输入区默认态；并示意 hover 某条 Agent 消息时浮出的逐轮操作与其右键 / 操作菜单。建立「外壳 + 消息流」整体观感。
- **s2 工具调用块 · 全态** `ui.html#s2`：同一消息流里集齐工具调用的**运行中 / 成功 / 失败 / 展开**四态，并示意结构化结果（计划 / 待办清单）卡片。
- **s3 代码块 / diff 块** `ui.html#s3`：Agent 富文本里的**代码块**（语法着色 + 复制）与**代码改动 diff 块**（增删分色 + 文件名 + 增删统计）。
- **s4 流式输出中** `ui.html#s4`：Agent 正在生成（逐字 + 光标）+ 伴随运行中工具；底部输入区为**停止**态。
- **s5 空对话 · 刚开始** `ui.html#s5`：用户首条已发、Agent 准备 / 思考中、尚无正文的过渡态。
- **s6 错误态** `ui.html#s6`：Agent 轮次失败的错误卡片 + 重试；并示意发送失败的就地提示 + 重试。
- **s7 输入区 · 全态** `ui.html#s7`：底部输入区在对话页的四态画廊 —— 空 / 输入中 / 发送中·运行中（停止 + 排队）/ 禁用（主机离线）。
- **s8 长对话滚动 + 回到底** `ui.html#s8`：上滚读历史时的截断滚动态 + **回到底部**浮动抓手（含未读新消息指示）。

## 5. 边界与依赖

- **与 `2026-06-27-codepilot-layout` 的关系（填充，非取代）**：本需求**填充**外壳里「中间画布 · 对话流」这一被标「后续保持现状」的内部细节。**外壳（顶栏 / 4 卡片 / 三开关 / 主题来源 / 默认显隐 / 拖拽）一律不动**；回写 canonical 时，原「对话流占位」即等于本消息流全态。
- **与 `2026-06-28-welcome-home` 的关系（同一画布的另一态）**：二者是中间画布「有 / 无活跃对话」两种态，**外壳与主题完全一致、视觉连续**；发出首条从首页切入本消息流，新建 / 清空回落首页。本需求不改首页任何屏。
- **与右栏 / 目录树的关系（只发起跳转）**：点击代码 / 文件 / 改动跳到右栏对应面板属各自模块；本需求只负责**发起跳转**的入口与即时反馈，不画右栏内容。
- **与外壳顶栏 / 左栏的关系（只承接）**：对话的重命名 / 删除 / 归档 / 分屏 / 切换由顶栏与左栏对话树负责；对话页只承接其结果（如标题、选中态）。
- **主题依赖**：依赖外壳提供的统一主题来源（codePilot 浅 / 深）；对话页不得自带冲突配色，须与外壳 / 首页一眼同一 app。
- **数据依赖（能力级）**：依赖「能取到一条对话的消息 / 工具调用 / 改动 / 状态」「能流式接收产出」「能打断 / 重试 / 排队」「能判定主机在线」这类**既有信息与能力**，本需求**只读、只展示、只发起既有动作**，不新增采集 / 协议机制。
- **平台**：桌面优先；紧凑 / 手机态随外壳后续统一适配。
- **无老用户 / 迁移**（fork 未上线）。**不改协议 / RPC / 能力标志**。

## 6. 验收标准（逐条可判定 · 能 / 否）

1. 中间画布有活跃对话时，落地的是这条**消息流**（用户轮次 + Agent 轮次 + 工具调用 + 底部输入区），**取代**外壳里原「对话流占位」；它是中间画布的一种态，**不是**新增独立路由 / 页面。
2. **用户轮次**靠右、气泡形态；**Agent 轮次**以**富文本（Markdown）**呈现（段落 / 列表 / 行内代码 / 链接 / 小标题 / 强调），并有弱化的来源 / 模型标识。
3. **工具调用块**可折叠：折叠态显示工具名 + 关键参数摘要；展开态显示详情；**运行中 / 成功 / 失败**三态在折叠行一眼可区分，失败带错误摘要；计划 / 待办等结构化结果有对应卡片。
4. **代码块**等宽、语法着色、可复制；**代码改动 diff** 增删分色、带文件名与增删统计，二者清晰可读。
5. 存在**流式输出态**：Agent 文本逐字流入 + 生成光标 + 伴随运行中工具；此时底部输入区显示**停止**而非发送。
6. 存在**空对话刚开始态**：用户首条已发、Agent 准备 / 思考中、尚无正文；与「无任何对话」的首页空态**可区分**。
7. 存在**错误态**：Agent 轮次失败显错误卡片 + 重试抓手；发送失败有就地提示 + 重试。
8. 底部输入区在对话页覆盖四态：**空**（发送不可用）/ **输入中**（发送可用）/ **发送中·运行中**（停止 + 继续输入则排队）/ **禁用**（主机离线灰化 + 重连提示）；本需求**未重定义**输入区内部能力（附件 / 模型选择器等仍以外壳既有输入区为准）。
9. 存在**长对话滚动**：可上下滚；上滚离开底部出现**回到底部**抓手；Agent 同时产出则抓手带**未读新消息**指示；点抓手回到底并恢复跟随。
10. 存在**逐轮操作**：hover 某条浮出操作（复制 / 复制为 Markdown / 引用 / 从这一步重试…），破坏性操作（从这一步重试 / 回退）明确，且与外壳既有语义一致。
11. 整页套 **codePilot**（GitHub 浅色，默认），与外壳 / 首页**同一套主题来源**，**深色一并适配**；对话页不自带独立配色，与外壳**一眼同一 app**、与首页**视觉连续**。
12. 本需求**不改**：外壳（顶栏 / 4 卡片 / 三开关 / 主题来源 / 默认显隐 / 拖拽）、首页着陆态、右栏 / 目录树内部、对话增删改管理流程本身、协议 / RPC。
13. s1～s8 八屏在 ui.html 均有对应锚点，各屏交互注解块覆盖 布局（桌面 + 紧凑）/ 默认 / hover / 活跃 / 运行 / 禁用 / 空 / 加载 / 错误 / 离线 / 右键菜单 / 浅深主题及各可点元素点击效果。
