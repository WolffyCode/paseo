## 1. 目标 & 用户价值

定义 Helm 设置界面「应用段（App 级 / 设备级）」的全部内容——挂在已批 master-detail 设置外壳左导航**第一段「应用」**下的 8 个 tab。

- 用户为什么需要：
  - **跟设备走、改一次处处一致**：语言、主题、字号、快捷键、本机 daemon、桌面权限、版本与更新——这些是「我这台电脑 / 这个 App」的偏好，**与连了哪台主机无关**；切主机不该让它们重载或变化。应用段就是这些设备级偏好的**唯一归集处**。
  - **普通用户会用就行**：每页只留常用项、少术语、默认值合理；少数桌面专属高级项（字体族等）折叠或注明专属；**不堆术语、不暴露底层概念**。
  - **本机自托管要可控**：桌面用户的 agent 跑在本机内建 daemon 上——它的开 / 关 / 退出后续跑 / 日志 / 状态要一处看清、一处可控，**且与「所连主机的 daemon」明确区分**（两层）。
  - **诚实不画饼**：底层没有的能力（连接自检、摄像头 / 辅助功能权限、Helm CLI 更新 / 卸载、daemon 重启入口）**要么本轮诚实不列、要么标「目标态」占位，绝不画成能用**——回应「verify functional, not just rendered」教训。

应用段与主机段共用一个外壳：**应用段 = 设备级**（本地偏好 + 桌面壳设置 + 操作系统拥有的权限 / 进程态），**主机段 = 主机级**（所选主机 config.json）。本需求只定义应用段；主机段沿用已批需求不动。

## 2. 范围

**做（应用段 8 tab · 设备级 · 桌面 · 普通用户优先）**

- **外壳接入**：把已批外壳左导航此前灰置的「应用设置 · 后续阶段」占位，落成**可点的「应用」段 8 行**（通用 / 外观 / 快捷键 / 守护进程 / 集成 / 权限 / 诊断 / 关于）；下方仍是「主机」段（host 选择器 + 主机 / 模型与提供方 / 用量 + 配置文件）。**应用段不随 host 选择器切换重载**。
- **8 tab 每项的常用功能 + 全交互态**（按下文「真实能力」据实写，被纠正项以本文为准）。
- **门控标注**：每 tab 标跨平台 / 桌面（Electron）专属；非桌面对桌面专属 tab 在导航置灰（本轮桌面 only，仅注明）。
- **即时持久化**：控件改动即落盘（设备本地），无独立「保存」。
- **能力缺口诚实标注**：缺口处标「目标态 / 待接」或本轮不列，不写降级路径、不画成能用。

**不做（明确划出 · 防蔓延）**

- **不做装 agent CLI（Claude Code / Codex / OpenCode / Copilot / Pi）于「集成」tab**：那是**主机段「模型与提供方」每行 CLI 检测 + 一键安装**的事（同理念、不同位置）；应用段「集成」只做 **Helm 自家 CLI + 技能包**两项，不重复。
- **不做不存在的控件**：守护进程**不拆**出独立 Start / Stop 按钮、不拆出独立「开机自启」开关（真实为单开关合并）；权限**不做**麦克风测试按钮、不做摄像头 / 辅助功能项、不做「打开系统设置」深链。
- **不做 Helm CLI 的更新 / 卸载**（真实仅安装；缺口，开放项③）。
- **不在应用段建第二个 JSON 编辑器**：应用段是设备级、**没有自己的 JSON 文件**；真正的主机级高级项仍由**主机段「配置文件（JSON）」编辑器**兜底。应用段设备项少而简单，无需 JSON 逃生舱。
- **连接自检**（诊断 tab）：当前完全不存在；本轮方向 = **移出、不画成能用**（标目标态，开放项①）。
- **手机 / 紧凑态**：本轮桌面 only（仅注明门控）；无老用户 / 迁移逻辑（fork 未上线）。

## 3. 全局 IA 与设置外壳（应用段如何接入）

### 3.1 外壳结构（沿用已批主机段外壳）
单一 master-detail（左导航 + 右详情），桌面 only。左导航两段：
- **段 A「应用」（本需求落成）**：8 行 tab（通用 / 外观 / 快捷键 / 守护进程 / 集成 / 权限 / 诊断 / 关于），图标 + 名称；一次只一个 active，点击切到该 section、右详情整块替换。
- **段 B「主机」（已批 · 不动）**：段头 host 选择器（当前主机 + 在线色 + 「当前」标识 + 展开切换）+ 主机 / 模型与提供方 / 用量 + 配置文件（JSON）。
- 顶部「返回主壳」（Esc 同效）。

### 3.2 应用段 ≠ 主机段（关键边界）
- **应用段所有 tab 跟设备走、对所有已连主机一致**，**不随 host 选择器切换重载**。在应用段任一 tab 上切换 host 选择器，应用段右详情**不变**。
- **唯一例外**：「关于」tab 的「已连主机」分组**列全部已连主机**的 daemon 版本——它跨主机但**也不跟随 host 选择器**（始终列全部）。
- 段 B（主机段）仍跟 host 选择器走、写所选主机 config.json，与本段无交叉。

### 3.3 进入 / 退出 / 默认落点
- **进入**：主壳左下「设置」→ 落**记忆的上次 section**；首次默认可落「通用」或外壳默认落点（与主机段约定一致，由外壳统一决定）。应用段各 tab 也可经主壳 / onboarding 深链到达（如 onboarding「本机连接失败 → 查看诊断」落「诊断」或其日志入口，见开放项①）。
- **退出**：返回主壳。
- 一次只一个 section active；切 section 右详情整块替换。

### 3.4 平台门控（真实）
- **跨平台 tab**：通用（「服务 URL 打开方式」行除外）/ 外观（「界面字体 / 代码字体」两行除外）/ 诊断 / 关于（「发布频道 / 软件更新」除外）。
- **桌面（Electron）专属 tab**：快捷键 / 守护进程 / 集成 / 权限——非桌面在导航置灰不可点。
- 本轮桌面 only，仅注明门控；非桌面隐藏 / 置灰行为见各 tab 状态要点。

### 3.5 即时持久化
- 应用段所有控件改动**立即落盘（设备本地）**，无独立「保存」；输入类失焦 / 回车提交。
- 即时动作型项（诊断的音频 / 日志、权限的请求 / 测试、守护进程的启停 / 状态、关于的检查更新）**不写偏好**，是一次性动作。

UI：`ui.html#shell`（外壳 · 应用段激活 = 本段外壳现役范例，右详情演示「通用」金标准）

## 4. tab 清单与每 tab WHAT

> 每 tab：作用 / 子功能 / 关键流程 / 状态要点（全态见 ui.html）/ 边界 / UI 锚点。状态命名统一：默认 · 悬浮 · 活跃 · 运行 · 禁用 · 空 · 加载 · 错误 · 离线。

### A1 通用（General）[跨平台] — `#gen1`
- 作用：App 全局基础行为；设备级、对所有已连主机生效。
- 子功能：
  - **代理运行时回车行为**（分段「打断 / 入队」，**默认打断**）：弱化术语——代理正在跑一个回合时，按回车是**打断当前回合**还是把这条消息**排进队列**等回合结束。**注意语义是「忙时回车」，不是普通「回车发送」**（文案不得写成 enter 发送）。
  - **界面语言**（下拉：跟随系统 + 简体中文 / English / Español / Français / Русский / العربية，**默认跟随系统**，选中即时切换全 App）。
  - **服务 URL 打开方式**（下拉：询问 / 内置浏览器 / 系统浏览器，**默认询问**）——**桌面专属**，非桌面隐藏此行。作用：工作区暴露的本地服务点开后用哪个浏览器。
  - **终端回滚行数**（输入：0 – 1,000,000，**默认 10000**；失焦 / 回车按范围夹边界后提交）。
- 关键流程：改任一控件 → 即时落盘 → 全 App 即时生效；无独立保存。
- 状态要点：无空 / 无加载（恒有默认值，本地同步读）；字号 / 行数越界自动夹边界、非数字回退、不弹错；**主机离线照常可改**（设备级与主机无关）；无右键。
- 边界：设备级，不写任何主机 config.json；「服务 URL 打开方式」行桌面专属。
- UI：`ui.html#gen1`

### A2 外观（Appearance）[跨平台；字体族行桌面 / web 专属] — `#ap1`
- 作用：视觉呈现偏好；设备级、对所有主机一致。
- 子功能：
  - **主题**（下拉：浅色 / 深色 / **跟随系统（默认）**；含暗色变体 Zinc / Midnight / Claude / Ghostty，每项带色板）。
  - **字体**：界面字体（留空 = 系统默认，**桌面 / web 专属，native 隐藏**）/ 界面字号 11–24（默认 16）/ 代码字体（同专属）/ 代码字号 9–22（默认 12）。
  - **语法高亮方案**（下拉 8 个：GitHub / Catppuccin / Dracula / Tokyo Night / One / Nord / Gruvbox / Solarized，**默认 One**；带**实时代码预览卡**）。
  - **工作区标题来源**（分段「标题 / 分支」，默认标题）：**本设计正式归入外观**；与 sidebar「显示偏好」菜单为**同一偏好**，任一处改、另一处即时同步（双入口，外观为正式归集处，开放项④）。
- 关键流程：改即时生效；代码预览卡随「代码字体 / 代码字号草稿 + 高亮方案」**边改边实时预览**（只读不可点）。
- 状态要点：字体族非法 → 静默回退上次值；字号越界 → 夹边界；无空 / 无加载；离线照常改；无右键。
- 边界：设备级；「界面字体 / 代码字体」两行桌面 / web 专属，native 隐藏。
- UI：`ui.html#ap1`

### A3 快捷键（Shortcuts）[桌面专属] — `#sc1`（含捕获态 `#sc1-capture`）
- 作用：查看与自定义键盘快捷键。
- 子功能：按 **5 组**列出（导航 / 标签与窗格 / 项目 / 面板 / 代理输入，约 40 个动作）；每行 = 动作名 + 当前键位**键帽** + 「改」；**已自定义行**额外「重置」；标题栏「**全部重置**」（仅存在覆盖时显示）。
- 改键流程：「改」→ **捕获态**（该行高亮、键位区变「按下快捷键…」、监听键盘、按下组合实时回显为键帽；**Backspace** 删最后一段；支持多段 chord）→ 「完成」（捕获到组合后才出现，存为该动作覆盖）/「取消」（或点走 / 失焦，放弃恢复原键位）。见 `#sc1-capture`。
- 键面：**Mac 显 ⌘ / 非 Mac 显 Ctrl**，自动。
- **诚实缺口**：捕获**无冲突检测**——与系统 / 已占用组合冲突仍允许保存、**后到优先、不阻断**（保持现状，本轮不补冲突告警）。
- 状态要点：默认显默认键位、无「重置」、标题栏无「全部重置」；活跃 = 捕获态（`#sc1-capture`）；捕获是即时本地交互无异步运行态；恒有默认绑定（无空 / 无加载）；**非桌面整页置灰**「快捷键在移动端不可用」；无右键。
- 边界：桌面专属；设备级（覆盖存设备本地）；不解析 / 不管理系统级快捷键冲突。
- UI：`ui.html#sc1`、`ui.html#sc1-capture`

### A4 守护进程（Daemon · 本机内建）[桌面 Electron 专属] — `#dm1`（含停止态 `#dm1-stopped`）
- **重要区分**：本 tab 管的是**设备级本机内建 daemon**（这台电脑上 Helm 内建、本地托管 agent / 终端 / 工作区）；它与**主机段 host 选择器所选主机的 daemon**（B「主机」tab，可能远程）**不是同一对象**，分属两层、互不影响。文案须明确区分。
- 作用：看清并控制本机内建 daemon 的生命周期。
- 子功能：
  - **状态**：运行中 / 未运行 + PID + 版本；**daemon 版本与 App 不一致 → amber 警告**「App 和 daemon 版本不匹配，请更新到相同版本」。
  - **管理内建 daemon（单一开关）**：开 = Helm 托管并在打开 App 时自动运行；关 = 二次确认后停止。**诚实：启停 + 开机自启合并为这一个开关**——**没有独立 Start / Stop 按钮、没有独立「开机自启」开关**（据实，不拆造不存在的控件）。
  - **退出后保持 daemon 运行（开关）**：退出 Helm 后 daemon 续跑、agent 不中断；关 = 退出即随之停止。
  - **日志文件**：显示路径 `~/.helm/daemon.log` + 「复制路径」+「打开日志」（弹窗显示文件尾部内容，等宽可选中；空文件提示「（日志文件为空）」）。
  - **完整状态**：「查看状态」运行 `paseo daemon status` 并在弹窗显示输出 + 「复制」；失败显「获取 daemon 状态失败：…」。
  - **高级设置**：外链配置文档（浏览器打开）。
- 关键流程：关「管理内建 daemon」开关 → 确认「这会立即停止内置 daemon，连接它的运行中 agents / terminals 会被停止」→ 停止（状态点转灰、文字「未运行」、`#dm1-stopped`，且不再自动起）；再开 → 立即拉起（状态点琥珀「启动中…」→「运行中」）。
- 状态要点：运行中（绿点 + PID + 版本）/ 未运行（`#dm1-stopped`：灰点 +「PID —」）/ 启动中（琥珀，开关短暂禁用）/ 版本不匹配（amber 警告）/ 状态拉取失败（状态行错误文案）；「打开日志」在无日志路径时禁用、「查看状态」加载中禁用；**本机内建不经网络 / 主机连接，无离线态**（进程意外退出 = 同「未运行」）；无右键。
- **诚实缺口 / 目标态**：**重启 daemon 入口**——真实有后端能力但当前无 UI，**本轮不做**；如要给入口标「目标态 / 待接」（开放项⑥）。
- 边界：桌面 Electron 专属；设备级。
- UI：`ui.html#dm1`、`ui.html#dm1-stopped`

### A5 集成（Integrations · CLI 与技能）[桌面 Electron 专属] — `#int1`（含安装中 `#int1-installing`）
- **重大纠正**：本 tab 真实只 **2 项**，**不是装 agent CLI**。
- 子功能：
  - **Helm 命令行（CLI）**：安装 Helm 自家 `helm` / `paseo` 命令到本机（`~/.local/bin` + 改 shell PATH），让用户能在终端直接用 Helm 命令。状态：已装 ✓ / 未装 → 「一键安装」。**诚实：目前仅安装，无更新 / 卸载**（缺口，开放项③）；附「安装文档」外链。
  - **技能包（Skills）**：安装 Helm 自带的 Claude / Codex 技能包。状态：未装 / 最新 / 有更新；动作：安装 / 更新（确认弹窗**列出变更项**）/ 卸载（**危险二次确认**）。
- 安装态：**只有文案态（无进度条）**——安装中显「正在安装…」并禁用，完成转对应状态，失败显行内错误 + 「重试」。见 `#int1-installing`。
- **诚实 + 交叉引用**：安装 **agent CLI**（Claude Code / Codex / OpenCode / Copilot / Pi）**不在这里**——在**主机段「模型与提供方」每行 CLI 检测 + 一键安装**；二者**同理念、不同位置**（应用段不重复 agent CLI 安装）。
- 状态要点：Helm CLI（已装 ✓ / 未装 + 一键安装 / 安装中文案 / 失败重试）；技能包（未装 / 最新 / 有更新 + 安装 / 更新 / 卸载）；无网络时安装 / 更新失败提示「需要网络连接」；无右键（或仅标准动作）。
- 边界：桌面 Electron 专属；设备级；本轮不做 Helm CLI 更新 / 卸载。
- UI：`ui.html#int1`、`ui.html#int1-installing`

### A6 权限（Permissions）[桌面 Electron 专属] — `#perm1`（含请求态 `#perm1-request`）
- **重大纠正**：真实只 **2 项**。
- 子功能：
  - **通知**：状态（已授权 / 未授权）+ 「请求」+ **「测试」**（已授权时发一条测试系统通知；失败行下红字）。
  - **麦克风**：状态（已授权 / 未授权）+ 「请求」。**诚实：无测试按钮**（麦克风不做测试）。
  - **顶部「刷新」**：重读当前授权状态。
- 请求流程：未授权行「请求」→ 桌面壳弹授权对话框（`#perm1-request`，按钮「正在请求…」）→ 授权后 badge 转绿「已授权」；**被拒 → 只给文案说明**（**无「打开系统设置」深链**）。

## 5. 核心流程（端到端 · 每条有入口有出口）

1. **改一项 App 偏好**：主壳「设置」→ 落应用段某 tab → 改控件（分段 / 下拉 / 输入）→ 即时落盘 + 全 App 即时生效；切 host 选择器**不影响**该偏好。入口 = 主壳「设置」；出口 = 返回主壳。
2. **改一个快捷键**：应用段「快捷键」→ 某行「改」→ 捕获态「按下快捷键…」→ 按组合（Backspace 删段）→「完成」存为覆盖（行出现「重置」）/「取消」放弃；**冲突仍可保存（后到优先，无告警）**；「全部重置」清空所有覆盖。入口 = 快捷键 tab；出口 = 行恢复常态。
3. **开 / 关本机内建 daemon**：应用段「守护进程」→ 关「管理内建 daemon」开关 → 确认（会停运行中 agents / terminals）→ 未运行态（`#dm1-stopped`）；再开 → 启动中 → 运行中。日志：「打开日志」弹窗看尾部；「查看状态」弹窗看 `paseo daemon status` 输出。入口 = 守护进程 tab；出口 = 状态行更新。
4. **装 Helm CLI / 技能包**：应用段「集成」→ Helm CLI 未装「一键安装」→ 安装中文案（`#int1-installing`）→ 已装 ✓ / 失败重试；技能包 安装 / 更新（确认列变更）/ 卸载（危险确认）。**装 agent CLI 不在此**（深链 / 引导用户去主机段「模型与提供方」）。入口 = 集成 tab；出口 = 状态更新。
5. **授权系统权限**：应用段「权限」→ 通知 / 麦克风未授权「请求」→ 系统授权对话框（`#perm1-request`）→ 授权转绿；**被拒只显文案**（无系统设置深链）；通知「测试」发一条测试通知；顶部「刷新」重读状态。入口 = 权限 tab；出口 = badge 更新。
6. **排障 / 自检**：应用段「诊断」→「音频播放测试」播放内置提示音并回填结果 →「查看 daemon 日志」复用守护进程日志视图（亦为 onboarding「本机连接失败 → 查看诊断」落点）。连接自检本轮不提供（目标态）。入口 = 诊断 tab；出口 = 即时结果回填 / 打开日志视图。
7. **看版本 / 更新**：应用段「关于」→ 进入静默检查一次 →（桌面）有更新点亮「安装更新」→ 确认 → 下载安装重启；「已连主机」对照各主机 daemon 版本（不一致 amber、离线灰）；社区按钮外部浏览器打开。入口 = 关于 tab；出口 = 更新完成 / 外链打开。
8. **切 host 不影响应用段**：在任一应用段 tab 上切 host 选择器 → 应用段右详情**不变**（设备级）；仅「关于 · 已连主机」恒列全部、不跟随。入口 = host 选择器；出口 = 应用段保持不变。
9. **边界分叉**：非桌面对桌面专属 tab 在导航置灰（快捷键 / 守护进程 / 集成 / 权限，及通用「服务 URL」、外观「字体族」、关于「频道 / 更新」行隐藏）/ 主机离线照常改应用段 / 安装 / 更新失败（文案 + 重试 / 需要网络）/ 权限被拒（文案，无深链）/ 能力缺口（目标态提示，不写降级）——每条有入口有出口、无死胡同。

## 6. 全局状态与规则

- **本段 scope = 设备级**：应用段所有 tab 写设备本地（App 偏好 + 桌面壳设置 + 操作系统拥有的权限 / 进程态），**不写任何主机 config.json**、**不随 host 选择器切换重载**（「关于 · 已连主机」列全部为唯一跨主机视图，但同样不跟随选择器）。这与主机段（写所选主机 config.json、跟 host 走）**两层不混**——回应主机段开放项⑩的设备级 ↔ 主机级边界。
- **即时持久化**：偏好类控件改动即落盘、无独立保存；即时动作类（播放 / 请求 / 测试 / 启停 / 检查更新）是一次性动作、不写偏好；写入失败必须可见反馈。

- **术语弱化（普通用户）**：少用专业词；**本机内建 daemon ＝「这台电脑上的后台服务」**，与**所选主机的 daemon**（主机段）明确区分、全程不混；回车行为是「忙时回车语义」不是「enter 发送」。

- **本设置无老用户 / 迁移逻辑**（fork 未上线）。

- **应用段（App 级 / 设备级）**：设置外壳左导航第一段；跟设备走、对所有已连主机一致、**不随 host 选择器重载**；含设备本地偏好 + 桌面壳设置 + 操作系统拥有的权限 / 进程态。**≠ 主机段**（主机级 config.json）。
- **本机内建 daemon**：桌面 Electron 在这台电脑上内建、设备级的后台服务（本地托管 agent / 终端 / 工作区）。「守护进程」tab 管它。**≠ 所选主机的 daemon**（主机段「主机」tab，可能远程）。
- **代理运行时回车行为**：代理正在跑一个回合时按回车的语义——打断当前回合 或 把消息入队。**是「忙时回车」语义，不是普通「回车发送」**。
- **Helm 命令行（CLI）**：Helm 自家 `helm` / `paseo` 终端命令（装到 `~/.local/bin` + 改 shell PATH）。**≠ agent CLI**（Claude Code / Codex 等，由主机段安装）。
- **技能包（Skills）**：Helm 自带的 Claude / Codex 技能集，可安装 / 更新 / 卸载。
- **发布频道**：软件更新通道（稳定版 / 测试版），桌面专属。
- **已连主机版本对比**：「关于」中列全部已连主机 daemon 版本并与本机对比的只读视图；列全部、**不跟随 host 选择器**。
- **目标态**：底层暂无、本轮诚实不实现的能力占位（连接自检 / 摄像头 / 辅助功能 / 系统设置深链 / Helm CLI 更新卸载 / daemon 重启入口），**不画成能用**。
