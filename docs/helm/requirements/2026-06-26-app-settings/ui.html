Helm · 设置 · 应用段（8 tab） · UI 设计稿

需求 2026-06-26-app-settings（P1）。本稿覆盖应用段（App 级 / 设备级）8 tab：通用 · 外观 · 快捷键 · 守护进程 · 集成 · 权限 · 诊断 · 关于；外加外壳金标准。与产品经理共享同一份冻结结构。设计原则：普通用户、做减法、能力缺口诚实标「目标态」、不画成能用。已按真实代码纠正旧附录失真（见下）。浅色 · accent #20744A · 桌面 only（跨平台 / 桌面专属差异写进各屏注解，不画进框）。

对旧附录的真实能力纠正（本稿已改正）

  • 守护进程：启停 + 开机自启 = 「管理内建 daemon」一个开关（删独立 Start/Stop 按钮、删独立「开机自启」开关）；停止 = 开关关的确认流。
  • 集成：不是装 agent CLI；单卡只 2 行 —— Helm 命令行（CLI） + 技能包（Skills）。装 agent CLI 在「主机 › 模型与提供方」。
  • 权限：真实只 2 项 —— 通知（可测试）+ 麦克风（无测试）。摄像头 / 辅助功能 / 系统设置深链 = 目标态缺口块。
  • 诊断：极简 —— 音频播放测试 + 查看 daemon 日志（复用入口）。连接自检 = 目标态占位（选 A），真实尚无、与守护进程/主机诊断重叠。
