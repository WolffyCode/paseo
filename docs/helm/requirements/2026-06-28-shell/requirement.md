# 需求 · 壳子（layout shell）重写 · 纯骨架

> 日期：2026-06-28 · 状态：草拟（gate-1 反馈后重写）· 配套 UI：[ui.html](./ui.html) · canonical：docs/helm/product.md
> **只写 WHAT**（见 [standards.md](../../standards.md) §7）—— 全文不含组件名 / 函数名 / 文件名 / 样式变量 / 状态容器名 / 路由名 / 代码片段 / 实现路子。CodePilot 仅作**观感参照对象**。
> **gate-1 董事长反馈（本次重写主轴）**：**壳子的东西太多了**。壳子 = **纯骨架**，只含三样——① 布局规则 ② 全态交互 ③ 驱动布局的壳级按钮。**对话树 / 搜索 / 项目列表 / 消息 / Composer / 右栏 tab 内容 / 目录树文件 / 设置导航项 / host 切换器 / 右键菜单 / 面包屑 / ··· 菜单 / 分支标签**等**全部不属于壳子**——壳里这些区域就是**空容器**，其能力移入 §7「后续内容里程碑（非壳）清单」，壳里不做、但记着不丢。
> **一句话定位**：壳子 = 一组**空容器** + 把容器**划分 / 摆位 / 改宽 / 显隐 / 切页**的**布局规则**与**壳级按钮**及其**全部交互态**。壳子**不关心容器里装什么**。

---

## 1. 目标 & 用户价值

**一句话目标**：把 Helm 主界面重写为一具**纯骨架壳子**——**上 = 顶部栏、下 = 主体区**；主体区两种页面模式（对话页 / 设置页）；每个区域都是**空容器**；壳子只提供「划区 / 摆位 / 改宽 / 展开收起 / 切页 / 返回」这组**布局规则 + 壳级按钮 + 全态交互**，不掺任何区内业务内容。

**用户为什么需要纯骨架壳子**：

- **为什么把内容剥出去？** 把「容器」和「内容」混在一起，是布局规则散乱、行为不一致、难维护、难扩展的根因。先做一具**职责单一、处处一致、稳定可靠**的壳——开关 / 拖拽 / 切页在任何区域、任何页面模式下行为完全一致——再往空容器里逐个填内容，每块内容都站在同一具可靠骨架上。
- **为什么用户要两种页面模式 + 一键返回？** 用户在「干活（对话）」与「调设置」之间高频往返。把设置做成壳内的一种页面模式（对话页↔设置页）、**返回按钮固定在顶栏左上角**、返回后对话页**恢复成离开前的样子**，用户永远不会「跳走又找不回来」。
- **为什么要三个独立开关 + 拖拽？** 用户「时而专注对话、时而铺开多面板」：按需开左侧边栏 / 右侧边栏 / 目录树，三者互不挤占、可任意组合、宽度可拖可记忆——这是真实工作节奏。本骨架先把这套**布局行为**做到位，内容随后填。

## 2. 范围

### 做（本需求 = 纯骨架壳子：空容器 + 布局规则 + 壳级按钮 + 全态交互）

- **① 整体上下布局**：上 = **唯一一条贯穿整宽顶栏**；下 = **主体区**（按当前页面模式呈现两种布局之一）。
- **② 主体区两种页面模式**：
  - **对话页模式**：主体区 = **左侧边栏 + 对话区 + 右侧边栏 + 目录树**（左→右），全部为**空容器**。
  - **设置页模式**：主体区 = **左侧边栏 + 设置区**，二者均为**空容器**；设置区占据左侧边栏右侧全部空间。
- **③ 区域显隐（壳级按钮驱动）**：左侧边栏 / 右侧边栏 / 目录树各有一个**独立 toggle**（落顶栏），只控自己那块；additive、互不连带、可任意组合；对话区**无 toggle、始终在场、自适应**填满剩余宽度，随其它区开 / 关伸缩。**左侧边栏在两种页面模式都在场，且两模式都可 toggle。**
- **④ 区域拖拽改大小**：左侧边栏 / 右侧边栏 / 目录树三个侧区均经**相邻拖拽手柄**左右改宽，受 **min / max** 约束、**按工作区记忆**宽度；对话区无独立宽度、只自适应。
- **⑤ 对话页↔设置页切换 + 返回**：
  - **设置入口按钮**固定在**对话页左侧边栏的最下方槽位** → 单击进入设置页模式。
  - **返回按钮**固定在**顶栏左上角（← 返回）**（**不在左侧边栏**）→ 单击直接回对话页模式，并**恢复进入设置前的对话页布局与选中态**。
- **⑥ 全态交互**：上述壳级按钮与拖拽手柄的**全部状态**——默认 / hover / 活跃（对应面板已展开 / 当前页面模式）/ 按下 / 展开 / 收起 / 拖拽中 / 到 min·max 上下限 / 切页 / 返回——逐态定义、处处一致。
- **⑦ 各区空容器（本需求可见交付物）**：左侧边栏 / 对话区 / 右侧边栏 / 目录树 / 设置区五个容器本需求各显**空容器骨架**（明确「内容后续」），但其**开 / 关 / 拖 / 切页 / 返回当场可用**。
- **平台**：**桌面 only**。

### 不做（明确划出去，防范围蔓延）

- **任何区内业务内容**——对话树 / 搜索 / 项目列表 / 消息 / Composer / 右栏 tab 内容 / 目录树文件 / 设置导航项 / host 切换器 / 右键菜单 / 面包屑 / ··· 菜单 / 分支标签 / 打开位置 / 环境信息 popover / 命令面板 / 级联选择器 …… 一律**不在壳里做**，壳里对应区域就是**空容器**；这些能力逐条移入 §7「后续内容里程碑（非壳）清单」，**记着不丢、壳里不做**。
- **顶栏中部的对话上下文区**：本需求只在顶栏留**空槽位**，不画其中任何内容（标题 / 面包屑 / ··· / 分支等属 §7 内容）。
- **复杂组合交互的内容部分**：如「点左栏子对话 → 展开右侧边栏 + 建侧边聊天」——壳只提供「展开右侧边栏」这一**公共壳行为**；「子对话」「建侧边聊天」属内容（§7）。本需求只交付**原子壳行为**与**对话页↔设置页切换**这一组合。
- **主题取值 / 切换机制**：壳读**既有 codePilot 主题来源**渲染，主题数值 / 浅深切换归 `codepilot-layout` 与后续，本需求不定义。
- **拖拽视觉效果态细节**（手柄高亮样式 / 宽度气泡 / 到限阻力反馈）：标后续；本需求只要求「能拖 + 受 min·max 约束 + 记忆宽度」可用。
- **真实业务分叉**（连接 / 发送 / 离线 / 切主机重载 / 加载 / 取消）：随各区内容后续；本骨架壳行为为纯前端即时态。
- **移动 / 浏览器紧凑态**：deferred，后续由桌面统一适配。

## 3. 流程（端到端）

> 每态有入口、有出口，无死胡同。本需求为壳骨架级，业务分叉随各区内容后续。

- **落地对话页默认态**：连上主机进入主体区 → 对话页模式：顶栏 + 左侧边栏（默认展开 · 空容器）+ 对话区（空容器 · 始终在场）+ 右侧边栏（默认收起）+ 目录树（默认收起）；左侧边栏最下方见**设置入口按钮**。出口 = 可对任一 toggle / 拖拽 / 设置入口操作。
- **切区域 toggle**：点顶栏某 toggle（左 / 右 / 目录树）→ 该区「展开 ↔ 收起」；**收起时整张空容器消失、对话区补位伸缩**，另两区不受影响；toggle 呈「活跃 = 已展开 / 普通 = 已收起」。三 toggle additive、可同时全开（四区并排 · 对话区最窄）、可同时全收（只剩顶栏 + 对话区）、或任意中间组合。
- **拖拽改大小**：拖某侧区相邻手柄 → 该侧区改宽，受 min / max 约束、到限即止；松开后**按工作区记忆**该宽度；对话区随之自适应。
- **进设置页**：点左侧边栏最下方**设置入口** → 主体区切到设置页模式：左侧边栏保持在场（仍可 toggle）、其右侧切为**设置区空容器**；顶栏**左上角出现 ← 返回**，对话上下文空槽位与右侧边栏 / 目录树 toggle 在设置页模式**不呈现**。
- **设置页内 toggle 左侧边栏**：点顶栏的左侧边栏 toggle → 设置页的左侧边栏「展开 ↔ 收起」（与对话页左栏行为一致），设置区补位伸缩。
- **返回对话页**：点顶栏左上角 **← 返回** → 主体区切回对话页模式，并**恢复进入设置前的对话页布局**（三区展开 / 收起态、各区宽度、当前选中均不丢失）。
- **复杂交互 = 组合（后续内容里程碑，沿用本骨架的壳行为）**：内容里程碑里的复杂交互一律由本骨架已有的壳行为（展开 / 收起 / 拖拽 / 切页）**组合**而成，壳子**不新增布局原语**。
- **本需求分叉极少（壳级）**：toggle / 拖拽 / 切页 / 返回均为即时前端态，无加载 / 失败 / 取消。

## 4. 屏与全状态

> 逐屏对应 [ui.html](./ui.html) 锚点 `ui.html#sX`，与 UI 设计师对齐锚点。每屏完整交互态以 ui.html 逐屏「交互注解块」（布局 / 状态 / 右键菜单 / 点击效果）为准。本需求以**壳骨架态 + 空容器态**为主；区内内容态一律标「后续」。布局维度统一：**桌面 only；移动 / 浏览器紧凑态 deferred**。

- **s1 对话页 · 默认态（壳骨架 + 空容器）** `ui.html#s1`：唯一贯穿整宽顶栏 + 左侧边栏（默认展开 · 空容器）+ 对话区（空容器 · 始终在场）+ 右侧边栏（收起）+ 目录树（收起）；左侧边栏最下方见**设置入口按钮**。关键态：顶栏唯一且贯穿整宽、四区基线对齐；三 toggle 默认态（左 = 活跃 / 右 = 普通 / 树 = 普通）；各区空容器可见；顶栏中部为留给后续内容的**空槽位**。
- **s2 对话页 · 四区全展开** `ui.html#s2`：三 toggle 全开——左侧边栏 + 对话区 + 右侧边栏 + 目录树四空容器并排，对话区最窄但仍在场；顶栏三 toggle 均活跃。关键态：additive 全开、相邻容器分隔可见、对话区自适应填充。
- **s3 对话页 · 全收（仅顶栏 + 对话区）** `ui.html#s3`：三 toggle 全收——主体区只剩对话区空容器（占满）；顶栏三 toggle 均普通；左侧边栏收起后其最下方设置入口的去向在注解块说明。关键态：对话区独占、三 toggle 全普通、可逐一点回展开。
- **s4 顶栏 + 壳级按钮 · 拆解** `ui.html#s4`：结构说明屏，逐项拆清两种页面模式下的顶栏——
  - **对话页顶栏**（左→右）：窗口控制安全区 → 左侧边栏 toggle →〔对话上下文**空槽位**〕→ 弹性留白 →〔分支**空槽位**〕→ 目录树 toggle → 右侧边栏 toggle。
  - **设置页顶栏**（左→右）：窗口控制安全区 → **← 返回** → 左侧边栏 toggle → 弹性留白（**无**右侧边栏 / 目录树 toggle、**无**对话上下文）。
  - **左侧边栏最下方** = 设置入口按钮（对话页驱动进设置；设置页模式下呈「当前 / 活跃」态，离开用顶栏 ← 返回）。
  - 逐按钮标 默认 / hover / 活跃（对应面板已展开 · 当前页面模式）/ 按下 四态，以及「设置入口（左栏底）↔ 返回（顶栏左上角）」随页面模式的出现 / 隐藏关系。**这是「壳骨架是否理解对」的核心对照屏。**
- **s5 区域拖拽改大小 · 全态** `ui.html#s5`：三侧区经相邻手柄拖拽改宽——标注各区 min / max / 默认与「按工作区记忆」；对话区无独立宽度、自适应。关键态：手柄 默认 / hover / 拖拽中 / 到 min·max 上下限即止；**拖拽视觉效果态细节标后续**，本屏只画「能拖 + 受限 + 记忆」。
- **s6 设置页 · 默认态（空容器 + 顶栏 ← 返回）** `ui.html#s6`：设置页模式落地态——左侧边栏（与对话页**同一个** · 保持在场 · 仍可 toggle · 空容器）+ 设置区（空容器）+ 顶栏左上角 **← 返回**；对话上下文与右侧边栏 / 目录树 toggle 本模式**不呈现**。关键态：左侧边栏共享在场且可展开 / 收起、设置区空容器、返回按钮在顶栏左上角可达。
- **s7 对话页↔设置页 · 切换流程** `ui.html#s7`：把「点设置入口（左栏底）→ 设置页」「点 ← 返回（顶栏左上角）→ 对话页且恢复进入前态」两步画清——含「进设置前对话页是某三区组合 + 某宽度 + 某选中 → 进设置 → 返回后回到同一组合 / 宽度 / 选中」的恢复语义。关键态：切换为即时态、无中间确认、状态恢复无丢失；**返回按钮位置 = 顶栏左上角（非左侧边栏）**。

**全态交互聚焦**：每屏重点覆盖 默认 / hover / 活跃 / 按下 / 展开 / 收起 / 拖拽（含到限）/ 切页 / 返回 / 空容器 / 浅深主题；区内业务态一律以「后续」在注解块占位，不在本需求画。

## 5. 边界与依赖

### 与既有壳设计的关系：**结构「收敛为纯骨架」，内容能力「移出 · 记着不丢」**

- **收敛（本需求改的）**：既有两套壳结构（`2026-06-25-home-shell` 三区独立、`2026-06-27-codepilot-layout` 统一顶栏 + 4 卡片）收敛为**一具纯骨架**——顶栏 + 4 区 + 3 独立 toggle + 三侧区拖拽 min / max 与记忆的**形态沿用 codepilot-layout**（不重定义视觉 / 数值），但**所有区内业务内容剥离为空容器**，并**新增「设置页」作为主体区第二种页面模式** + **顶栏左上角 ← 返回** + **设置页左栏可 toggle**。
- **设置从「跳走到独立路由」收敛为「壳内页面模式切换」**：既有「左下入口 → 跳 /settings → 返回主壳 / Esc」（`home-shell#s14`、integration `product-flow.md`）形式化为对话页↔设置页壳内切换；语义一致，呈现收敛为页面模式。**修正**：返回控件从「左侧边栏」改到**顶栏左上角（← 返回）**（gate-1 董事长图 #19）；设置入口仍在对话页左侧边栏最下方。
- **内容能力全部移出、记着不丢**：见 §7。一句话定性：**本需求只做「容器骨架 + 壳行为」，所有内容 deferred 到非壳里程碑**。

### 平台与依赖

- **桌面优先**：本需求桌面 only；移动 / 浏览器紧凑态 deferred。平台分界按断点判定布局，不用平台判定。
- **主题来源是依赖项**：壳读**既有 codePilot 主题来源**渲染（浅 / 深），不自带冲突配色；主题取值拥有权属 `codepilot-layout`。
- **与 onboarding 衔接不变**：对话页默认态仍是 onboarding「连接成功」落点；本需求不动连接逻辑（canonical `product.md`「首次启动 onboarding」为准）。
- **壳行为 = 壳对外唯一入口（编程规范对齐）**：依据 `docs/helm/frontend-architecture.md`（模型驱动 UI），壳把布局状态与转移收敛在模型层，对外只暴露这组公共壳行为；区内 / 跨区复杂交互一律**组合**这些行为。本需求据此把「壳行为」定义为产品级 WHAT。
- **无老用户 / 迁移**（fork 未上线）。**不改协议 / RPC / `server_info.features.*`**：纯前端结构 + 壳行为重写。

## 6. 壳清单（本需求做 · 纯骨架 · 只此三类）

> 严格对齐 gate-1「壳子 = 只含三样」。本需求**只**交付以下三类；任何不在此列的都属 §7 内容、deferred。

### A. 布局规则（区域划分 / 位置 / 宽度 / 页面模式）

1. **唯一顶栏**：贯穿窗口整宽、位于最顶部、仅此一条；其下为主体区。
2. **主体区两种页面模式**（任一时刻只呈现其一）：对话页 / 设置页。
3. **对话页布局** = 左侧边栏 + 对话区 + 右侧边栏 + 目录树（左→右），全为空容器。
4. **设置页布局** = 左侧边栏 + 设置区（左侧边栏右侧），全为空容器。
5. **对话区始终在场、自适应填满剩余宽度、无独立 toggle、无独立宽度**。
6. **左侧边栏两种页面模式共享同一个**、两模式都在场、两模式都可 toggle。
7. **三侧区宽度规则**：左 / 右 / 目录树各有 默认 / min / max（数值沿用 codepilot-layout 既定值），**按工作区记忆**宽度。
8. **顶栏窗口控制安全区**：为 OS 窗口控制（交通灯）预留位（布局层关注，非内容）。
9. **顶栏中部为留给后续内容的空槽位**（对话上下文 / 分支等不在本需求画）。

### B. 壳级按钮 · 图标（驱动布局的按钮）

10. **左侧边栏 toggle**（顶栏）：只控左侧边栏展开 / 收起；两种页面模式都在场。
11. **右侧边栏 toggle**（顶栏，仅对话页模式）：只控右侧边栏展开 / 收起。
12. **目录树 toggle**（顶栏，仅对话页模式）：只控目录树展开 / 收起。
13. **设置入口按钮**（**对话页左侧边栏最下方槽位**）：单击 → 进设置页模式。

### C. 全态交互（壳行为的所有状态）

15. **三 toggle 全态**：默认 / hover / 活跃（对应面板已展开）/ 按下；additive 独立、互不连带；收起时整张空容器消失、对话区补位伸缩；可全开 / 全收 / 任意组合。

## 8. 验收标准（逐条可判定 · 能 / 否 · 只测壳骨架）

**整体布局**

1. 主界面存在**且仅存在一条**贯穿整宽顶栏，位于最顶部；其下为主体区。
2. 主体区任一时刻只呈现**两种页面模式之一**（对话页 / 设置页）。

**对话页 · 空容器 + toggle** 3. 对话页主体区为 **4 个空容器**，左→右依次：左侧边栏 / 对话区 / 右侧边栏 / 目录树。4. 对话区**始终在场、自适应填满剩余宽度、无独立 toggle**。5. 顶栏存在 **3 个相互独立的 toggle**（左侧边栏 / 右侧边栏 / 目录树），**各只控自己那块**；任一切换**另两块不受影响**；某块收起时其**整张空容器消失、对话区补位伸缩**，再点可还原。6. 三块可**同时全展开**（四区并排 · 对话区最窄）、**同时全收**（只剩顶栏 + 对话区）、任意中间组合均成立。

**拖拽改大小** 7. 左 / 右 / 目录树**三侧区均可经相邻手柄拖拽改宽**，受 min / max 约束、到限即止；松开后**按工作区记忆**该宽度；对话区无独立宽度、只自适应。

**设置入口 / 切页 / 返回（含三处修正）** 8. **设置入口按钮固定在对话页左侧边栏的最下方槽位**；单击 → 主体区切到设置页模式。9. 设置页模式 = **左侧边栏（与对话页同一个 · 仍在场 · 仍可 toggle 空容器）+ 设置区（空容器）**；对话上下文与右侧边栏 / 目录树 toggle 本模式**不呈现**。10. **返回按钮位于顶栏左上角（← 返回）、不在左侧边栏**；单击 → **直接切回对话页模式**（无中间确认）。11. **设置页的左侧边栏支持展开 / 收起**（与对话页左栏行为一致）。12. 返回后**恢复进入设置前的对话页布局**（三区展开 / 收起态、各区宽度、当前选中均不丢失）。13. 左侧边栏在两种页面模式**共享为同一个**，不是两份各自独立的副本。

**纯骨架边界** 14. 本需求**只交付** §6 三类（布局规则 / 壳级按钮 / 全态交互）；**各区均为空容器**，但其**开 / 关 / 拖 / 切页 / 返回当场可用**——空容器不妨碍任何壳行为。15. §7 内容清单的每一条都在文档中显式标注「**壳里不做 · 空容器 · 后续内容里程碑**」，**无任何一项被声明丢弃**；任何区内业务内容（对话树 / 搜索 / 消息 / Composer / 右栏 tab / 目录树文件 / 设置导航项 / host 切换器 / 右键菜单 / 面包屑 / ··· / 分支等）**不在本需求壳里出现**。

**全态交互** 16. §6.C 的全态交互（默认 / hover / 活跃 / 按下 / 展开 / 收起 / 拖拽含到限 / 切页 / 返回 / 返回恢复）在 ui.html 逐屏注解块齐全可判定。

**复杂交互 = 组合** 17. 文档明确：复杂交互由公共壳行为**组合**而成，壳子**不新增布局原语**；本需求只交付原子壳行为 + 对话页↔设置页切换这一组合。

**平台 / 主题** 18. 文档明确本需求**桌面 only**，移动 / 浏览器紧凑态标 **deferred**。19. 整壳**读既有 codePilot 主题来源**，不自带冲突配色、不在本需求重定义主题取值。
