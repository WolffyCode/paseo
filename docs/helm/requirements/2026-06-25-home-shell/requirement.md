## 1. 目标 & 用户价值

定义 Helm 在「从首页点开始使用、连上主机之后」落地的**主页面**——即 Helm 的 **Codex 式主壳**，覆盖它的**默认空态**、**使用中态**与**全交互态**。这是 onboarding（项目①）明确推迟的「主页本身」。

本轮的形态目标（反馈 B 核心）：主壳是**三个独立整体**，**不是一条横跨整窗的标题栏**——

- **左侧栏**是一个全高独立列（交通灯在它左上区域）；
- **中区画布**自带它**自己的顶栏**（canvas 顶栏）；
- **右侧栏**展开后是一个整体，顶部是它**自己的 tab 页签栏**。

用户为什么需要：

- **新用户**连上主机就要能立刻干活——落地即是一个居中的输入框，打字就能让代理开始工作，不被任何配置步骤挡住。
- **老手**要把日常一处统管——对话历史（按项目目录归类、且**对话内可无限嵌套 subagent**）、改动审查、终端、浏览器、文件、跨主机切换，全在一个主壳里随手可达，不用在多个界面之间跳。
- 本轮还要把主壳的「形」做到位、**逐像素贴 Codex**：三个独立区块各自有自己的 chrome 与拖拽边界，而不是一条横跨整窗的标题栏。

## 2. 范围

**本轮做（桌面端主页面主体 + 空态 + 使用中态 + 全交互态）**

- **三区独立结构（反馈 B）**——能力级描述各区构成与控件落点：
  - **左侧栏（全高独立列）**：自上而下 [**窗口 chrome 细条（左上）= 交通灯 + 侧栏开关 + ‹ › 前进/后退**；**左栏收起时该细条多出 ✎ 新对话图标**] → [主机切换器胶囊（全栏宽）] → [新对话][搜索] → [项目段（树）] → [对话段（树）] → 弹簧 → [设置]；右缘有拖拽手柄可改宽。顶上没有横跨整窗的 chrome 条——**侧栏开关与前进/后退属这条窗口细条（紧挨交通灯），不在中区 canvas 顶栏（反馈 1）**。
  - **中区画布（自带 canvas 顶栏）**：顶栏 左[对话标题][更多 ···]、右[**打开位置 ▾**（真实本机 app 图标，非发送图标）][**环境信息·置顶摘要**（☰，点开浮层 popover）][右面板开关——**仅右面板收起时**出现在此]；顶栏之下才是整块画布（消息流 / 空态 Composer）。**canvas 顶栏左侧不含侧栏开关 / 前进后退**（那组在左栏顶的窗口 chrome 细条）。**空态精简（反馈 ③）：新对话空态时这些对话上下文控件（标题 / ··· / 打开位置 / 环境信息）不显示、对话开始后才出现；空态顶栏只保留右面板开关。**
  - **右侧栏（展开后整体，自带 tab 页签栏）**：**打开后默认 = 启动器列表（反馈 ②）**——刚打开还没选内容 → 竖排 审查⌃⇧G / 终端 / 浏览器⌘T / 文件⌘P，顶部只有 ⤢ 放大 + 收起、**还没 tab 条**；点一行才打开对应 tab。选内容后页签条 [审查（固定第一个）][已开的页签…][新选项卡 +]、最右 [⤢ 放大][▯ 收起]；左缘有拖拽手柄可改宽；**收起时整列消失**，其开关回到 canvas 顶栏右侧。
  - **环境信息 / 置顶摘要 popover（反馈 ①）**：canvas 顶栏右的 ☰ =「切换置顶摘要」（**不是「大纲」**），点开是一个**浮层「环境信息」popover**（**非右侧停靠面板**，与右面板「审查」tab 的完整 diff 区分——这是顶栏**紧凑摘要**）。内容：标题「环境信息」+ [+] · **变更**（工作区 git 改动 + 刷新 / 计数）· **本地 ▾** · **分支 ▾**（如 develop-v4）· **提交或推送** · **GitHub CLI 状态** · **来源**分区（空时「暂无来源」）。扎根真实 git / 分支 / 提交推送能力。
  - **打开位置（反馈 ③）**：canvas 顶栏的「打开位置 ▾」用**真实本机 app 图标**（VS Code / Ghostty / iTerm2 / 访达…）；点开 = 在外部 app 打开当前工作区的列表，**支持手动新增 app**。
  - **主机切换器**是 Helm 净新控件（Codex 不支持多主机故没有），位置在左栏交通灯细条之下、新对话之上，全栏宽。
- **右侧栏 tab 能力（反馈 C）**：新增多页签（「新选项卡 +」小菜单 [终端 / 浏览器 ⌘T / 文件 ⌘P]）、**悬浮显示关闭 ✕**、**拖拽调序**（显放置指示线）、**整列左缘拖拽改宽**（每工作区记忆）。
- **左侧栏拖宽（反馈 D）+ 左右侧栏拖拽效果态（反馈 E）**：左栏右缘拖宽（每工作区记忆）；左右侧栏拖拽时的效果态——手柄高亮、宽度指示、最小/最大宽约束。
- **对话树（反馈 F）**：左栏「对话」不再一层平铺，而是 **项目（目录）→ 对话（根）→ subagent 子对话 → …无限嵌套**；每节点支持展开/收起、状态点、悬浮动作、按节点类型的右键菜单、内联重命名、拖拽排序；扎根代理父子的真实语义（派生 / 剥离 / 级联归档 / 页签与归档的非对称 / 运行态聚合）。

## 3. 流程（端到端）

每态有入口、有出口，无死胡同、无悬挂屏。

- **首跑落地**：onboarding 连上主机 → 落到主页**默认空态**（s1：左栏主机在线、对话段「还没有对话」、中区居中标题 + 居中 Composer 已聚焦、右侧栏收起、无悬浮侧条）。入口 = onboarding 连接成功；出口 = 可立即在 Composer 打字。
- **发首条消息**：在居中 Composer 打字 → 发送键由不可发转为可发 → 发送 → 该草稿落为一条对话，中区从空态切到消息流，左栏「对话」段出现这条对话并选中。入口 = 空态打字；出口 = 进入使用中态（s1 → s2）。
- **使用中（s2）**：
  - **左栏对话树**：项目（目录）→ 对话（根）→ subagent 递归嵌套；点节点把该对话加载到中区；运行中节点显运行点；有子节点的节点可展开/收起；悬浮出动作、右键出按节点类型的菜单、可内联重命名、可拖拽排序。
  - **中区**：消息流（内容保持现状）+ canvas 顶栏（对话标题 / 更多 / 打开位置 / 环境信息·置顶摘要 / 右面板开关）。**侧栏开关 / 前进后退在左栏顶的窗口 chrome 细条（紧挨交通灯），不在 canvas 顶栏。** 点 ☰ 环境信息·置顶摘要 → 弹浮层「环境信息」popover（变更 / 本地 / 分支 / 提交推送 / GitHub CLI / 来源）；点打开位置 ▾ → 外部 app 列表（在该 app 打开工作区 + 添加应用）。
  - **右面板**：点 canvas 顶栏右面板开关 → 展开 → **默认显启动器列表（审查⌃⇧G / 终端 / 浏览器⌘T / 文件⌘P）** → 点一行打开对应 tab → 「新选项卡 +」新增 终端/浏览器/文件 页签 → 页签悬浮浮出 ✕ 可关 → 拖拽页签调序 → ⤢ 放大占满中区 → 还原 → ▯ 收起（整列消失、开关回 canvas 顶栏右侧）。
  - **Composer**：输入 `/` 出彩色命令 token、@ 提及、附件、队列、工具行；点级联胶囊切中转站/模型，下条消息生效（提供方保持锁定）。
  - 按 ⌘K 唤出命令面板跳转 / 执操作 / 快速切中转站；点搜索（或 ⌘K）跨对话/消息/目录搜索并高亮、选中跳转。
- **切主机（s7）**：点左栏主机切换器胶囊 → 下拉（与胶囊同宽、锚其正下）→ 选另一台**在线**主机 → **切换中过场态** → **整工作区重载**（左栏树 + 中区 + 右面板全部重载为该主机数据，先页级加载、后呈现新内容）→ 新主机主壳。入口 = 主机切换器；出口 = 新主机的主壳。
- **离线主机重连（s7）**：点**离线**主机行 → 触发**重连**（退避重试），**不是直接切走** → 成功后恢复该主机。入口 = 点离线行；出口 = 恢复在线 / 仍离线可再点。
- **拖宽侧栏（s12）**：拖左栏右缘 / 右栏左缘手柄 → 手柄高亮 + 旁出当前宽度指示 → 到达 min/max 约束时手柄变色/阻力 → 松开后记忆该工作区宽度。入口 = 拖手柄；出口 = 松开定宽。
- **对话树操作（s6）**：
  - **展开/收起**：点有子节点节点的箭头 → 展开/收起子树。
  - **内联重命名**：右键「重命名」/ F2 → 行内输入 → Enter 提交 / Esc 取消。
  - **拖拽排序**：拖节点 → 显放置指示线 → 落位。
  - **剥离（detach）**：subagent 节点右键「剥离为独立对话」→ 仅解除父子关系标记 → 子升为根/同级，**不停止、不归档、不移动**其工作区。
  - **归档**：对话节点右键「归档」→ 软删除离开列表（**根对话归档级联归档其全部子代理（递归）**；**subagent 归档仅从父对话的子代理轨道移除**）；运行中先弹确认。
  - **删除**：右键「删除」→ 带确认。
  - **关页签**：关**根对话**页签 = 归档（运行中先确认）；关 **subagent** 页签 = **仅布局**，对话仍留在父对话的子代理轨道、可随时重开。
- **边界分叉（每条有入口有出口，无死胡同）**：
  - **无项目空态**：连上但没有任何项目/对话 → 左栏「对话」段显「还没有对话」+ 单个「开始新对话」入口；中区仍是可打字的居中 Composer。
  - **加载**：切主机 / 拉对话树 / 切对话 / 拉 diff → 对应作用域行级或页级加载指示，完成后进内容态。
  - **发送失败**：发送出错 → 瞬时 toast「发送失败 · 已重新入队 · 重试」，消息留在队列，点重试重发。
  - **主机离线 / 重连**：当前主机断连 → 主机状态点转红 + 左栏 callout（离线 + 重新连接）；运行中对话冻结待重连；点重连退避重试，成功后恢复。
  - **取消**：代理运行中发送键变取消键，点取消停止当前回合；连接中过场（切主机 / 重连）可在加载态退出回到原态。

## 4. 屏与全状态

逐屏对应 [ui.html](./ui.html) 的锚点 `ui.html#sX`（s1–s15，与共享规格「最终屏图」完全一致），每屏一句话「是什么」+ 关键交互态。每屏的「布局」维度本轮均为：**桌面 only；紧凑/手机态暂缓，后续由桌面统一适配**。

- **s1 主页 · 默认空态（桌面）** `ui.html#s1`：新三区结构的落地态——左栏（交通灯细条 → 主机切换器胶囊 → 新对话/搜索 → 项目段 → 对话段「还没有对话」→ 弹簧 → 设置）；中区 **空态精简 canvas 顶栏（反馈 ③：只剩右面板开关，打开位置 / 环境信息 / 标题等上下文控件对话开始后才出现）** + 居中标题 + 居中 Composer（已聚焦）+ 三层级联胶囊；右侧栏收起（开关在 canvas 顶栏右侧，**无悬浮侧条**）。关键态：空态可直接打字、空输入发送键不可发、主机在线点、左栏右缘可拖宽、空态顶栏精简。

- **对话树扎根的真实语义（只描述语义，不写实现符号；语义对齐 docs/agent-lifecycle.md）**：
  - subagent 由代理在工作中**派生**（创建时声明父子关系），系统据此把子对话挂到父对话名下、出现在父的**子代理轨道**；某对话只要有 subagent，就在其下加缩进一级的子对话，递归无限嵌套。
  - **父对话归档则级联归档其全部子代理（递归）**——避免子代理在编排者消失后泄漏。
  - **剥离（detach）= 仅解除父子关系标记** → 子升为根/同级，不停止、不归档、不移动其工作区。
  - **关根对话页签 = 归档**（页签是根对话的「家」；运行中先弹确认）；**关 subagent 页签 = 仅布局**（subagent 的「家」是父的子代理轨道，对话仍在、可重开）。
  - **运行态聚合**：运行中的 subagent 把「运行」贡献给**父对话所属工作区**的聚合状态（按父代理的工作区归属，而非子代理当前所在目录）；父对话自身状态仍按其自己的回合算（子在跑不等于父在跑）。
