# 需求 · Helm 主页面（Codex 式主壳）

> 日期：2026-06-25 · 状态：草拟（待闸 1） · 配套 UI：[ui.html](./ui.html) · canonical：docs/helm/product.md
> **只写 WHAT**（见 [standards.md](../../standards.md) §7）。本文不含任何实现细节（代码片段 / 组件名当实现指令 / 文件改法 / 实现路子）——那是架构与开发的事。能力级表述（"复用既有审查/终端能力"）可以，实现符号不可。

## 1. 目标 & 用户价值

定义 Helm 在「从首页点开始使用、连上主机之后」落地的**主页面**——即 Helm 的 **Codex 式主壳**：左侧对话历史 / 中区单对话 / 右侧停靠面板，外加它的**默认空态**与**使用过程中态**。这是 onboarding（项目①）明确推迟的「主页本身」。

- 用户为什么需要：
  - **新用户**连上主机就要能立刻干活——落地即是一个居中的输入框，打字就能让代理开始工作，不被任何配置步骤挡住。
  - **老手**要把日常一处统管——历史对话、按项目（目录）归类的会话、改动审查、终端、浏览器、文件、跨主机切换，全在一个主壳里随手可达，不用在多个界面之间跳。

## 2. 范围

**做（主页面主体 + 空态 + 使用中态 + 全交互态）**

- **顶栏控件区**：左栏开关 / 布局（单栏↔分屏）/ 全屏（专注）/ 右面板开关。四个开关贴主壳右上，不占整行 topbar，与下方分屏严格对齐。
- **左侧栏**：顶部主机切换器（在线状态色 + 主机名 + 展开）/ 新对话 / 搜索 / 目录（项目）→ 对话两段层级 / 会话历史入口 / 左下设置入口。
- **中区（单对话）**：
  - 空态——居中标题 + 居中 Composer（进去就能打字），下方一行三层级联选择器（提供方🔒 · 中转站 · 模型）。
  - 使用中——消息流（用户消息、代理回复、工具调用折叠卡）+ 贴底 Composer。
  - Composer 全态——队列轨 / 附件盘 / 斜杠命令 / @ 提及 / 工具行（附件、思考、Plan、联网、语音、发送/取消）。
- **右面板（Codex 停靠列）**：标签头（审查 / 终端 / 浏览器 / 文件 + 右上收起）；**收起 = 右上角一个开关，无悬浮侧条**；展开后切标签；终端可右/底停靠；可拖宽。
- **主机切换器**：多主机下拉（在线 / 连接中 / 离线状态色 + relay 标识 + 当前项标记 + 添加主机入口）；切主机后整个工作区重载该主机数据；离线主机点击触发重连。
- **⌘K 命令面板**：跳转 / 操作 / 快速切中转站 三组，上下文感知，键盘导航。
- **搜索浮层**：跨对话标题 / 消息内容 / 目录命中并高亮（与 ⌘K 同一入口）。
- **全交互态**：默认 / 悬浮 / 活跃选中 / 运行 / 禁用 / 空 / 加载 / 错误 / 离线，加右键菜单（对话行、目录行、diff 行/文件、终端、文件树、Composer 输入、队列项——每套列全每项 + 点击效果）、内联重命名、拖拽排序、Toast / Alert / 左栏 callout。
- **响应式**：桌面（分屏多面板 + 右侧停靠）与紧凑/手机（左栏 overlay、右面板转全屏覆盖 + 顶部横滑标签、Composer 贴底）。

**不做（划给后续项目，主壳里只「露出入口」、不画其内部）**

- **设置全部页**：左下设置只是导航入口，点进 /settings 的两级 IA（App 级 + 主机级）详情属另一个项目，本主壳不画设置详情页。
- **三层选型的深度管理**：中转站管理 / 用量 / 代理 / MCP / Skill / Prompts / 插件 / 提供方与中转站底层配置——级联里的「管理中转站…」只露入口。
- **主机 Hub / 添加主机详情**：主机切换器的「添加主机…」只打开既有添加流入口，方式详情（直连 / 粘链 / 扫码）属另一项目。
- **手机配对流程**：紧凑态保留 host-pill 与配对入口，但配对流程本身属 onboarding / 主机项目。
- **会话历史列表本身**：左栏「会话历史」只露入口，历史列表屏属另一屏。

## 3. 流程（端到端）

- **首跑落地**：onboarding 连上主机 → 落到主页**默认空态**（左栏主机在线、对话段「还没有对话」、中区居中标题 + 居中 Composer 已聚焦、右面板收起、无悬浮侧条）。入口 = onboarding 连接成功；出口 = 可立即在 Composer 打字。
- **发首条消息**：在居中 Composer 打字 → 发送键由不可发转为可发 → 发送 → 该草稿落为一条对话，中区从空态切到消息流，左栏「对话」段出现这条对话并选中。入口 = 空态打字；出口 = 进入使用中态。
- **使用中**：
  - 左栏出现目录（项目）→ 对话两段；点对话行加载到中区；运行中的对话显运行点。
  - 中区消息流渲染用户消息、代理回复、工具调用折叠卡（完成打勾，权限请求在流内出权限卡）。
  - 点右面板开关展开右面板（默认「审查」标签），切标签看审查 diff / 终端 / 浏览器 / 文件；终端可在右/底停靠间切换；拖左缘改面板宽度。
  - 点 Composer 下方级联胶囊切中转站 / 模型，下条消息生效（提供方保持锁定）。
  - 按 ⌘K 唤出命令面板跳转 / 执操作 / 快速切中转站。
  - 点搜索（或 ⌘K）跨对话 / 消息 / 目录搜索并高亮，选中跳转。
- **切主机**：点左栏顶主机切换器 → 选另一台在线主机 → 整个工作区（左栏目录/对话 + 右面板内容）重载为该主机数据。入口 = 主机切换器；出口 = 新主机的主壳。
- **边界分叉（每条有入口有出口，无死胡同）**：
  - **无项目空态**：连上但没有任何项目/对话 → 左栏仅「对话」段显「还没有对话」+ 单个「开始新对话」入口；中区仍是可打字的居中 Composer。
  - **加载**：切主机 / 拉目录 / 切对话 / 拉 diff → 对应作用域行级或页级 spinner，加载完成进内容态。
  - **发送失败**：发送出错 → 瞬时 toast「发送失败 · 已重新入队 · 重试」，消息留在队列，点重试重发。
  - **主机离线 / 重连**：当前主机断连 → 主机状态点转红 + 左栏 callout（主机离线 + 重新连接）；运行中对话冻结待重连；点重连退避重试，成功后恢复。
  - **取消**：代理运行中发送键变取消键，点取消停止当前回合；连接中过场（切主机/重连）可在加载态退出回到原态。

## 4. 屏与全状态

逐屏对应 [ui.html](./ui.html) 的实际锚点与标题（s1–s13），每屏一句话「是什么」+ 关键交互态。

- **S1 主页 · 默认空态（桌面）** `ui.html#s1`：刚连上的落地态——左栏顶主机切换器 + 新对话/搜索/会话历史 + 目录·对话两段 + 左下设置；中区居中标题 + 居中 Composer + 三层级联胶囊；右面板收起（开关在右上顶栏控件区，无悬浮侧条）。关键态：空态可直接打字、发送键空输入时不可发、主机在线点。
- **S2 主页 · 使用中（进行中对话）** `ui.html#s2`：左栏目录→对话（选中 + 运行点）；中区消息流（用户消息右对齐、代理回复、工具调用折叠卡✓）+ 贴底 Composer。关键态：运行点、运行中发送键转取消、切对话页级加载、流式回复。
- **S3 顶栏控件区 全态 + 右面板开关位移** `ui.html#s3`：四个 顶栏控件区开关（左栏/布局/全屏/右面板）逐元素的默认 / 悬浮 / 活跃三态。关键规则：**右面板收起时开关在右上顶栏控件区，展开后开关移到右面板 标签头，屏幕位置不变**；布局开关在紧凑态禁用。
- **S4 右面板 · 审查（Codex 停靠列 + 标签头）** `ui.html#s4`：右侧停靠列 标签头（审查/终端/浏览器/文件 + 右上收起）；审查 = 改动 diff（文件 +N −N、增删行、底部 Stage/Revert/行评论）。关键态：标签切换、diff 行悬浮 出行评论、拖左缘改宽、收起回顶栏控件区、空态「没有改动」、非 git 工作区 Stage/Revert 置灰。
- **S5 右面板 · 终端（右/底停靠）+ 浏览器 + 文件** `ui.html#s5`：底部停靠终端（停靠切换 + 底部停靠标识 + 输出）+ 右面板浏览器（地址栏 + 刷新 + 预览）；文件面板（文件树 + 预览）作第三态。关键态：终端底↔右停靠切换、终端流式输出、浏览器加载、文件树空/预览、紧凑态退化为全屏标签。
- **S6 左侧栏交互：悬浮 / 右键 / 内联重命名 / 拖拽** `ui.html#s6`：一屏示意项目悬浮（新建对话 + 更多）、对话悬浮（更多）、对话右键菜单、目录右键菜单、内联重命名、拖拽虚线占位。关键态：native/紧凑动作常驻（悬浮仅桌面网页）、重命名提交/取消、拖拽其它行降透明。
- **S7 主机切换器（多主机 + 远程）· 净新** `ui.html#s7`：主机切换器下拉——主机表头 + 各主机行（在线绿 / 连接中橙 / 离线红 + relay 标识 + 当前项标记）+ 添加主机入口。关键态：切换重载工作区、连接中态、离线行可点触发重连、仅一台时只列本机。
- **S8 ⌘K 命令面板** `ui.html#s8`：命令面板——跳转 / 操作 / 快速切中转站三组 + 快捷键提示。关键态：键盘 ↑↓ 选 Enter 执 Esc 关、上下文感知（对话中才显快速切中转站）、跨主机聚合异步填充、无结果短句。
- **S9 三层级联选择器（展开）** `ui.html#s9`：单胶囊点开两级联菜单——① 提供方锁定 + 中转站列表（健康点/延迟/当前/管理入口）② 模型子菜单（exposed 子集 + 设为默认）。关键态：提供方仅新对话选一次且对话中锁定、中转站/模型对话中可切下条生效、测速延迟、中转站不健康红字。
- **S10 Composer 全态：队列轨 / 附件盘 / 斜杠 / @ / 工具行** `ui.html#s10`：队列轨（待发可重排）+ 附件盘（文件/图片/PR chip 可删）+ 斜杠命令菜单 + @ 提及 badge + 工具行。关键态：工具按钮**按模型能力显隐（非置灰）**、运行中发送转取消并入队列、附件上传/语音转写行级 spinner、发送失败 toast。
- **S11 状态合屏：搜索浮层 + 空 / 加载 / 错误 / 离线** `ui.html#s11`：搜索浮层（对话/消息/目录命中高亮）+ 空态（短名词 + 单个开始新对话）+ 加载（行级 inline / 页级 spinner）+ 错误（Alert + toast，各带重试）+ 离线（左栏 callout + 主机红点）。关键态：搜索高亮 + 跳转、重试/重连按钮进行时转「连接中…」并禁用避免重复。
- **S12 手机响应式（紧凑态）** `ui.html#s12`：三个紧凑态——A 首页列表（顶 host-pill + 目录/对话 + 底部标签栏 对话/项目/会话/设置）/ B 对话全屏（顶返回 + 消息 + 贴底 Composer + 精简胶囊）/ C 右面板全屏覆盖（顶部横滑标签 + 审查 diff + Stage）。关键态：断点判定（非平台判定）、动作常驻、悬浮态常驻、长按触发同桌面右键菜单、紧凑态无分屏。
- **S13 设置入口（只露出，不画内部）** `ui.html#s13`：左下「设置」行 = 进 /settings（App 级 + 主机级两级 IA）入口。本主壳只示意入口位置与各态，**不画设置详情页**。关键态：悬浮、点击进入（设置屏属另一项目）、本主壳无「设置已选中」态。

**每屏的完整交互态（默认 / 悬浮 / 活跃 / 运行 / 禁用 / 空 / 加载 / 错误 / 离线 / 右键菜单 / 点击效果）以 [ui.html](./ui.html) 的逐屏注解为准（四维度：布局 / 状态 / 右键菜单 / 点击效果），实现需逐态对齐。**

## 5. 边界与依赖

- **依赖的既有能力（复用，不重造）**：
  - 主机连接 / runtime（连本机与远程、连接状态、显示名、relay）。
  - 代理对话流（用户消息 / 代理回复 / 工具调用 / 权限请求 / 回合）。
  - 改动审查（diff 渲染、Stage/Revert、行评论）。
  - 终端、浏览器（含桌面 webview / web / native 平台差异）、文件浏览。
  - 命令面板与跨主机聚合搜索。
  - 模型与中转站选择（提供方 / 中转站 / 模型的选择能力，及测速健康度）。
  - 草稿→对话、按项目（目录）归类、归档、内联重命名、拖拽排序、空态范式、Toast/Alert/左栏 callout/加载 spinner。
- **平台差异**：
  - **桌面**：左栏 pinned（可拖宽）+ 分屏多面板 + 右侧停靠面板（含终端右/底停靠）；四 顶栏控件区开关贴右上。
  - **紧凑/手机**：左栏退化为 overlay/首页列表 + 底部标签栏；右面板退化为全屏覆盖 + 顶部横滑标签；Composer 贴底、胶囊精简；布局/分屏开关不存在；悬浮时动作常驻、长按代替右键。平台分界用断点判定，不用平台判定做布局决策。
- **与 onboarding（项目①）的衔接**：主页是 onboarding「连接成功」的**唯一落点**——onboarding 不画主页、主页不画连接流；二者在「连上即落空态」处接缝。本主壳没有老用户/迁移逻辑（fork 未上线）。
- **与后续项目的边界（只露入口，不进详情）**：设置（左下 + 紧凑底部标签）、三层选型深度管理（级联「管理中转站…」）、主机详情/添加主机（切换器「添加主机…」）、会话历史列表、手机配对——主壳里只放入口，点击跳到对应项目的屏。

**已知开放项（需董事长闸 1 定夺；本文只列待决问题，不写实现方案）**

1. **主机切换器净新**：左栏顶主机切换器是当前主壳缺口（现状无此控件），需净新加在左栏顶部、对齐 S1/S7 的形态（在线点 + 主机名 + 展开下拉）——确认位置/形态 OK？
2. **终端底部停靠形态**：S5 要终端可在「右/底」停靠；底部停靠是做**专门的底部 dock**（净新概念）还是沿用通用「向下拆分」语义近似表达？倾向后者并在注解标清——需定夺。
3. **空态主标题文案**：S1 暂用「今天想构建点什么？」；沿用，还是换更克制的句首大写无尾点短句（如「开始新对话」）以贴 Helm 排版规则？
4. **单胶囊三层级联 vs 现有分开选择**：设计采用单胶囊（提供方🔒 · 中转站 · 模型）级联；现状是分开的选择控件。采用单胶囊合并方向——确认？

## 6. 验收标准（逐条可判定）

1. 连上主机后落到主页**默认空态**：左栏主机在线、对话段显「还没有对话」、中区居中标题 + 居中 Composer 已可聚焦、右面板收起且**无悬浮侧条**（S1）。
2. 空态在居中 Composer 打字后发送键由不可发转可发，发送后草稿落为一条对话、中区切到消息流、左栏出现该对话并选中（S1→S2）。
3. 使用中消息流能渲染用户消息、代理回复、工具调用折叠卡（完成打勾），权限请求在流内出权限卡（S2）。
4. 右面板从收起到展开默认显「审查」标签；标签头可在审查/终端/浏览器/文件间切换；点右上收起后开关回到右上顶栏控件区、屏幕位置不变（S3/S4）。
5. 右面板可拖左缘改宽（每工作区记忆）（S4）。
6. 终端可在「底部停靠 ↔ 右侧停靠」间切换，停靠标识同步（S5）。
7. 左栏顶主机切换器显在线状态色；切到另一台在线主机后，左栏目录/对话与右面板内容重载为该主机数据；离线主机行点击触发重连而非直接切走（S7）。
8. 三层级联：提供方显锁定且仅新对话可选一次、对话中不可改；中转站与模型在对话中可切且下条消息生效（S9）。
9. ⌘K 命令面板分跳转 / 操作 / 快速切中转站三组，支持键盘 ↑↓/Enter/Esc，且「快速切中转站」仅在对话上下文出现（S8）。
10. 搜索能跨对话标题 / 消息内容 / 目录命中并高亮关键词，选中可跳转（S11）。
