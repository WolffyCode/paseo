> 日期：2026-06-27 · 状态：草拟（gate-1 待审）· 配套 UI：[ui.html](./ui.html) · canonical：docs/helm/product.md
> **只写 WHAT**（见 [standards.md](../../standards.md) §7）。本文不含任何实现符号——不写组件名 / 函数名 / 文件名 / 样式变量名 / 代码片段 / 实现路子；CodePilot 仅作为**观感参照对象**（一个开源项目），所有描述均为**能力级 / 形态级**。颜色 / 尺寸 / 圆角带「约」字的是**设计目标值**（同既有 requirement 引用 accent 色值的惯例），不是实现符号。

## 1. 目标 & 用户价值

**一句话目标**：把 Helm 主界面的**结构骨架与主题色**改版为参照开源项目 CodePilot 的形态——**一条贯穿整宽的统一顶栏** + 顶栏正下方**并排的 4 个浮动卡片区**（左侧侧边栏 / 中间画布 / 右侧侧边栏 / 目录树）+ **3 个各自独立的显隐开关**，并换上命名为 **`codePilot`** 的主题——取参照项目 CodePilot 的「GitHub 浅色」那套中性主题（白底 + 浅冷灰左侧栏 + 很淡的灰选中态 + 近黑文字 + GitHub 蓝点缀 + macOS 式浮动卡片），外加一套 GitHub 深色（**浅色 + 深色**两套），让**首页、主页对话页及所有页面**统一适配。**本版只做骨架与主题，各区内部细节标后续。**

**用户为什么需要**：

- 现状方向（home-shell：三个互不统属的独立区块、每块各自带一截 chrome——左栏顶的交通灯细条、中区自己的画布顶栏、右栏自己的页签条）显得**零碎、没有统一的窗口主线**。董事长评估 CodePilot 的「一条统一顶栏统管 + 4 区浮动卡片」**更高级、更整体、可达性更好**：所有窗口控制与面板开关集中在同一条顶栏一眼可达，主体则是干净的几张卡片。
- **3 个独立开关**让用户按需拼装工作区——要看文件就开目录树、要看工作面板就开右侧栏、要更多对话上下文就开左侧栏；三者**互不挤占、可任意组合**，中间的对话画布**始终在场**、随之伸缩。这正是用户日常「时而专注对话、时而铺开多面板」的真实诉求。
- **统一主题**让所有页面观感一致，并**首次提供深色**；高级感来自克制的中性配色（白底 + 很淡的灰选中态 + 近黑文字 + 浅灰边框）+ macOS「浮动卡片」排布（窗口外壳半透微透、内容卡片实色不玻璃化），强调色（GitHub 蓝）仅作点缀、不大面积铺色——这是董事长指明的「显高级」来源。
- 本版**先把「形」立住**（骨架 + 主题），让董事长在 gate-1 确认方向，再逐区填细节。

## 2. 范围

### 做（本版 = 桌面端结构骨架 + 主题色全局适配）

- **① 一条贯穿整宽的统一顶栏**：横跨窗口整宽、位于最顶部、约 40px 高；其正下方的所有卡片在**同一条水平基线**起。顶栏自左至右承载这些控件**槽位**（本版只立位置与显隐，控件内部行为标后续）：
  - 窗口控制安全区（交通灯）→ 左侧侧边栏开关（紧挨窗口控制）→ 当前对话标题 → 所属项目名（弱化/次要文字）→ 更多操作菜单（···）→ **（中间弹性留白）** → 当前分支标签（信息性、弱化）→ 目录树开关 → 右侧侧边栏开关。
  - **面包屑 / 分支的呈现规格**：顶栏的**项目名面包屑**与**分支标签**为**无边框、无填充的弱化文字按钮**（muted 灰字 + 图标，仅 hover 时浮极淡灰底）；**不得用选中态 / 强调色做边框或底**。
- **② 主体 4 个浮动卡片区**（左→右，每个是一张独立「浮动卡片」，相邻卡片间有可见的分隔）：
  - **左侧侧边栏**：对话列表 / 导航的**容器位**，默认约 240px 宽。
  - **中间画布区**：对话的**容器位**，自适应**填满剩余宽度**，**始终在场**。
  - **右侧侧边栏**：工作面板的**容器位**，默认约 480px 宽。
  - **目录树**：文件树的**容器位**，默认约 280px 宽，位于**最外侧**。
- **③ 3 个各自独立的显隐开关（全部落在统一顶栏，additive 非互斥）**：
  - 左侧侧边栏开关 → **只控**左侧侧边栏的展开 / 收起。
  - 右侧侧边栏开关 → **只控**右侧侧边栏的展开 / 收起。
  - 目录树开关（独立的展开 / 收起按钮）→ **只控**目录树的展开 / 收起。
  - 三者**互不影响、可同时展开、可任意组合**；任一开关只翻自己那一块，**不会**自动关掉另一块；中间画布始终在场，随展开 / 收起**自动伸缩**。
- **④ `codePilot` 主题色（取参照项目 CodePilot 的「GitHub 浅色」那套 · 浅色 + 深色 · 全局适配）**：
  - 主题统一命名 `codePilot`，默认 = **GitHub 浅色**中性主题：白背景、浅冷灰左侧栏、近黑文字、浅灰边框。
  - **选中 / 激活态 = 很淡的灰高亮（浅底深字）**——**绝不是黑底白字 / charcoal 底**。（此前「charcoal 近黑单色主色」是对 primary 按钮色的误读、非选中态，已纠正。）
  - **强调色 = GitHub 蓝**，仅用于主按钮 / 链接 / focus 环，**不铺满选中行、不铺大面积**。
  - **窗口材质分两层（桌面 / macOS）**：**窗口外壳层**（卡片间留白 gutter + 统一顶栏 + 左侧栏）为**半透磨砂材质（macOS vibrancy 观感）、轻微透出桌面背景**；**内容卡片层**（中间画布 / 右侧栏 / 目录树）**保持实色不透明、扁平 + 1px 边框，不做玻璃化**。卡片间留白 gutter 使主体呈「浮动卡片」排布，卡片大圆角（约 16px）。
  - 状态色：成功绿 / 警告橙 / 错误红 / 信息蓝。
  - **浅色（GitHub 浅色）+ 深色（GitHub 深色）两套**（深色为本版新增），**所有页面统一读同一套主题来源（`codePilot`）** → 首页、主页对话页、设置等全局自动适配。
- **三侧区拖拽改宽（本版定 min / max）**：左侧侧边栏 / 右侧侧边栏 / 目录树三个侧区均支持**左右拖拽改宽**，相邻区之间有拖拽手柄，并**按工作区记忆**宽度；各区最小 / 最大 / 默认宽（px）——左 180 / 300 / 240、右 320 / 800 / 480、目录树 220 / 500 / 280；拖到上下限即止。中间画布无独立宽度、自适应填充。（拖拽时的视觉效果态细节标后续。）
- **范围 / 响应式**：本版**桌面 only**——先把桌面做利索。

### 不做（划给后续；本版只在骨架里「留好容器位」，不重画）

- **各区内部细节**（逐区后续填）：
  - 左侧侧边栏内部——对话树语义（项目 → 对话 → subagent 无限嵌套、状态点、悬浮动作、右键菜单、内联重命名、拖拽排序）、主机切换器全流程、新对话 / 搜索入口。
  - 中间画布内部——消息流内容形态（用户气泡 / 代理回复 / 工具调用卡）、Composer 全态（彩色命令 token / @提及 / 队列 / 附件 / 工具行 / 模型选择器）。
  - 右侧侧边栏内部——多页签（审查 / 终端 / 浏览器 / 文件）的新建 / 关闭 / 调序 / 启动器列表 / 放大缩小。
  - 目录树内部——文件树展开 / 选中 / 预览。
  - 顶栏控件内部行为——更多操作菜单项、环境信息 / 置顶摘要 popover、打开位置（外部 app 列表）、分支标签点击、命令面板 / 搜索浮层。
- **卡片间拖拽改宽的「视觉效果态细节」**（注意：可拖拽 + min / max 数值 + 按工作区记忆宽度本版已定，见上「做」）：仅把**拖拽时的视觉效果态**（手柄高亮样式 / 宽度指示气泡 / 到达 min / max 时的阻力反馈）标后续。
- **主题「切换机制」**：本版要求浅 / 深两套都**定义齐全且能正确呈现**、所有页面统一适配；**用户如何切换浅 / 深、是否跟随系统、切换入口**标后续。
- **手机 / 紧凑态**：暂缓，后续由桌面统一适配（桌面优先）。
- **各区真实业务分叉**（加载 / 失败 / 离线 / 取消等）：随各区细节后续；本版骨架的开关显隐为纯前端态、无这些分叉。

## 3. 流程（端到端）

> 每态有入口、有出口，无死胡同。本版为骨架级，业务分叉随各区细节后续。

- **进主页落空态**：从首页 / onboarding 连上主机后落到主页 → 呈现**统一顶栏 + 主体卡片骨架 + 新主题**。默认落点：顶栏 + 左侧侧边栏（默认展开）+ 中间画布（**对话空态占位**）；右侧侧边栏与目录树**默认收起**。入口 = 连接成功落点；出口 = 可对三个开关任意操作、可进入对话区。
- **切左侧侧边栏**：点顶栏左侧侧边栏开关 → 左侧卡片在「展开 ↔ 收起」间切换；**收起时该卡片消失、中间画布补位变宽**，右侧侧边栏与目录树**不受影响**；顶栏开关呈「活跃=已展开 / 普通=已收起」。入口 = 该开关；出口 = 再次点击回切。
- **切目录树**：点顶栏目录树开关 → 最外侧的目录树卡片「展开 ↔ 收起」，**独立于另两个开关**；中间画布随之伸缩。入口 / 出口同上。
- **切右侧侧边栏**：点顶栏右侧侧边栏开关 → 右侧卡片「展开 ↔ 收起」，**独立**；中间画布随之伸缩。入口 / 出口同上。
- **三开关组合（additive）**：三者可**同时全开**（4 区并排、中间画布最窄）、**同时全收**（只剩顶栏 + 中间画布）、或任意中间组合；每个开关只翻自己那块、绝不连带关闭另一块；顶栏三个开关分别独立反映各自面板的展开 / 收起态。
- **主题全局一致**：在所有页面（首页、主页对话页、设置等）切换到本主题后，观感一致；**浅色与深色两套均正确呈现**（深色为本版新增）。切换浅 / 深的触发方式标后续，但两套都要能正确显示、不得只做一套。
- **空态**：中间画布无对话时显示**对话区空态占位**（容器骨架级，居中「开始对话」意象；Composer 等细节标后续），仍可作为进入对话的入口。
- **本版分叉极少（骨架级）**：开关显隐为即时前端态、无加载 / 失败 / 取消；主题应用无失败态。真实业务分叉（连接失败、发送失败、主机离线等）属各区细节，随后续填入，不在本版流程内展开。

## 4. 屏与全状态

> 逐屏对应 [ui.html](./ui.html) 锚点 `ui.html#sX`，与 UI 设计师对齐锚点。每屏的完整交互态以 ui.html 逐屏「交互注解块」（布局 / 状态 / 右键菜单 / 点击效果）为准。本版以**骨架态 + 主题态**为主，各区**内部细节态标后续**。布局维度本版统一为：**桌面 only；紧凑 / 手机态暂缓**。

- **s1 主页 · 默认空态（骨架 + 主题）** `ui.html#s1`：改版后的落地态——一条贯穿整宽的统一顶栏（含全部控件槽位）+ 左侧侧边栏（默认展开、容器占位）+ 中间画布（对话空态占位）；右侧侧边栏与目录树默认收起；整体套用新主题（浅色）。关键态：顶栏唯一且贯穿整宽、4 区基线对齐；三个开关默认态（左=活跃 / 右=普通 / 目录树=普通）；中区空态占位；浮动卡片投影 / 圆角 / GitHub 浅色中性基调 / 很淡的灰选中态 / GitHub 蓝点缀到位。**各区内部内容标后续。**
- **s2 主页 · 4 区全展开（使用中骨架）** `ui.html#s2`：三个开关**全部展开**的极限态——左侧侧边栏 + 中间画布 + 右侧侧边栏 + 目录树**四区并排**，中间画布被压到最窄但仍在场；顶栏三个开关均呈活跃态。关键态：additive 全开、相邻卡片分隔可见、中区自适应填充剩余宽；各卡片内只放容器占位（对话列表位 / 对话流位 / 工作面板位 / 文件树位），**内部细节标后续**。
- **s3 顶栏 + 4 部分 + 3 toggle 布局拆解** `ui.html#s3`：结构说明屏，逐项拆清——
  - **统一顶栏内容**（左→右）：窗口控制安全区 → 左侧侧边栏开关 → 当前对话标题 → 所属项目名（弱化）→ 更多操作菜单 ··· → 弹性留白 → 当前分支标签（弱化）→ 目录树开关 → 右侧侧边栏开关。**项目名面包屑与分支标签为无边框 ghost（muted 灰字 + 图标、仅 hover 浮淡灰底、不用选中 / 强调色做边框或底）。**
  - **4 部分（含拖拽 min / max）**：① 左侧侧边栏（默认 240 · min 180 / max 300）② 中间画布（自适应、始终在场、无独立宽度）③ 右侧侧边栏（默认 480 · min 320 / max 800）④ 目录树（默认 280 · min 220 / max 500、最外侧）；单位 px。三个侧区均可经相邻手柄左右拖拽改宽、按工作区记忆（UI 设计师在本屏补 min / max 与手柄标注）。
  - **3 个独立开关与组合矩阵**：逐一标注每个开关只控自己那块、additive 非互斥；列出关键组合态（仅中区 / 仅左+中 / 全开 / 任一单收 等）下哪些卡片在场、中区如何伸缩。
  - 关键态：每个开关的默认 / hover / 活跃（对应面板已展开）三态；卡片收起时整列消失、其余不受影响。**这是本版「方向是否理解对」的核心对照屏。**
- **s4 主题色板 · 浅 / 深（命名 codePilot）** `ui.html#s4`：主题取值与质感样板——并列展示**浅色（GitHub 浅色）**与**深色（GitHub 深色）**两套：白背景 + 浅冷灰左侧栏、近黑文字、浅灰边框、**很淡的灰选中 / 激活态（浅底深字、绝非黑底白字）**、**GitHub 蓝强调色（仅主按钮 / 链接 / focus 点缀，不铺选中行 / 不铺大面积）**、状态色（成功绿 / 警告橙 / 错误红 / 信息蓝）、大圆角（约 16px）、**材质分层样板**（窗口外壳 = 半透磨砂 vibrancy 微透、透出桌面；内容卡片 = 实色不透明 + 扁平 + 1px 边框、不玻璃化）。关键态：浅 / 深两套均完整；标注「所有页面统一读同一套主题来源（`codePilot`）、自动适配」；**切换浅 / 深的机制标后续**。

**本版骨架态聚焦**：每屏重点覆盖 默认 / hover（顶栏开关）/ 活跃（开关对应面板已展开）/ 展开 / 收起 / 空（中区空态）/ 浅深主题；各区**内部**的运行 / 加载 / 错误 / 离线 / 右键菜单等态**标后续**，在 ui.html 注解块中以「后续」明确占位，不在本版画全。

## 5. 边界与依赖

### 与现有 home-shell 设计的关系：**结构 + chrome + 主题层「取代」，业务能力「融合保留」**

- **取代（本版改的）**——现有主页设计（`requirements/2026-06-25-home-shell/`，本身仍处 gate-1、尚未回写 canonical）是「**三个独立整体、无横跨整窗标题栏**」，与 CodePilot「一条统一顶栏」**正好相反**。本需求在以下层面**取代**该方向：
  - 「三个独立区块」→ 改为「**一条贯穿整宽统一顶栏 + 4 部分浮动卡片**」。
  - 原**分散在各区的 chrome**（左栏顶的交通灯细条 + 侧栏开关 + 前进后退、中区自带的画布顶栏、右栏自带的页签条）→ **全部归并 / 上移进统一顶栏**；各区不再各自带 chrome 截条。
  - 原**文件浏览是右面板内的一个页签** → **升格为独立的第 4 部分（目录树）+ 独立开关**。
  - 原主题（浅色 + 绿 accent `#20744A`、无浮动卡片）→ 换为 **`codePilot`**（取参照项目 CodePilot 的「GitHub 浅色」中性主题：白底 + 很淡的灰选中态 + 近黑文字 + GitHub 蓝点缀）+ macOS 浮动卡片，并**新增深色**。
- **融合保留（本版不丢、只留容器位、细节后续）**——home-shell 定义的 Helm 业务能力**全部保留**，本版只在新骨架里**预留其容器位**，不重画其内部交互：
  - 左侧侧边栏容器 ← 承接对话树（项目 → 对话 → subagent 无限嵌套）、主机切换器、新对话 / 搜索。
  - 中间画布容器 ← 承接消息流 + Composer。
  - 右侧侧边栏容器 ← 承接多页签（审查 / 终端 / 浏览器 / 文件）等工作面板。
  - 统一顶栏槽位 ← 承接对话标题 / 更多操作 / 分支 / 环境信息 / 打开位置 / 命令面板等入口。
  - **主机切换器**是 Helm 净新能力（CodePilot 单主机、无此物），保留；其在新骨架中的**最终落点属细节、后续确定**，本版**不得丢弃**该能力。
- **左栏「外壳 vs 内部」硬边界（董事长明确指令）**——左栏是「CodePilot 外壳 + Helm 内胆」，两层各有归属：
  - **外壳由 CodePilot 接管**：统一顶栏里的左栏开关、左栏卡片位置、默认 240px / 可拖 180–300、codePilot 主题色。
  - **内部内容 + 布局保留 Helm 现有那套**（以 **Helm 现有侧边栏对话树设计**为准）：主机切换器（Helm 净新、左栏顶）/ 新对话 / 搜索会话 / **对话树**（项目 → 对话 → subagent 无限嵌套；chevron 前置、机器人图标、本地目录名、选中 = 淡灰、空态「暂无对话」、hover 工作区卡片、行尾「更多」、右键菜单、双击重命名）/ 设置（底部）。
  - **明确不采用** CodePilot 左栏的内容（「插件 / 素材库」等）——只借其外壳，不借其内容。
- 一句话定性：**本需求换「骨架与主题」，不换「业务能力」**；董事长不必担心上述能力被丢——它们只是改到后续逐区填。

### 平台与依赖

- **桌面优先**：本版桌面 only；紧凑 / 手机态后续由桌面统一适配。平台分界沿用既有约定（按断点判定布局，不用平台判定做布局决策）。
- **主题令牌全局适配是依赖项**：本版依赖存在**一套统一的主题来源**供所有页面读取，含浅 / 深两套——这是「全局一致、自动适配」的前提；任何页面都不得自带与之冲突的独立配色。
- **与 onboarding 的衔接不变**：主页仍是 onboarding「连接成功」的落点；本版只改主页**骨架与主题**，不动 onboarding 流程与连接逻辑。
- **无老用户 / 迁移**（fork 未上线）：不写任何存量兼容。
- **不改协议 / RPC / `server_info.features.*`**：本版是纯前端的结构与主题改版。

## 6. 验收标准（逐条可判定 · 能 / 否）

**统一顶栏**
1. 主页存在**且仅存在一条**贯穿窗口整宽的统一顶栏，位于最顶部、约 40px 高；其下所有卡片从**同一条水平基线**起。
2. 顶栏自左至右依次出现这些槽位：窗口控制安全区 → 左侧侧边栏开关 → 当前对话标题 → 所属项目名（弱化）→ 更多操作菜单（···）→ 弹性留白 → 当前分支标签（弱化）→ 目录树开关 → 右侧侧边栏开关。
3. 原 home-shell 分散在各区的 chrome（左栏交通灯细条 / 中区画布顶栏 / 右栏页签条）**不再各自独立存在**；这些 chrome 控件改由统一顶栏承载。

**4 个浮动卡片区**
4. 主体呈现**恰好 4 个**并排浮动卡片：左侧侧边栏（默认 240px）/ 中间画布（自适应）/ 右侧侧边栏（默认 480px）/ 目录树（默认 280px、最外侧），左→右顺序如述。
5. 中间画布**始终在场**、**填满剩余宽度**，且本身**没有**独立显隐开关。
6. 文件浏览以**独立的第 4 部分（目录树）**呈现，而**不是**右侧侧边栏内部的一个页签。
7. 相邻卡片间有可见留白分隔（gutter）；4 个内容卡片均为实色不透明 + 大圆角 + 1px 边框，呈「浮动卡片」排布。

**3 个独立开关（additive 非互斥）**
8. 顶栏存在 3 个相互独立的开关：左侧侧边栏开关、右侧侧边栏开关、目录树开关，**各自只控自己那一块**。
9. 任一开关切换时，**另外两块不受影响**（不被连带展开或收起）。
10. 三块**可同时展开**（4 区并排同时在场），也可同时收起（只剩顶栏 + 中间画布），及任意中间组合均成立。
11. 某块收起时其**整张卡片消失**、中间画布相应**补位伸缩**；再次点击同一开关可还原。
12. 顶栏每个开关分别反映其对应面板的状态（活跃=已展开 / 普通=已收起），三者状态相互独立。

**面板拖拽改宽（本版定 min / max）**
13. 左侧侧边栏 / 右侧侧边栏 / 目录树**三个侧区均可左右拖拽改宽**，相邻区之间有拖拽手柄，且改后**按工作区记忆**宽度；各区 min / max / 默认（px）分别为——左 180 / 300 / 240、右 320 / 800 / 480、目录树 220 / 500 / 280；拖到上下限即止、中间画布无独立宽度只自适应。

**主题色（命名 codePilot · 浅 / 深 · 全局）**
14. 默认主题**命名 `codePilot`**，取「GitHub 浅色」那套中性主题：白背景、浅冷灰左侧栏、近黑文字、浅灰边框。
15. 选中 / 激活态为**很淡的灰高亮（浅底深字）**，**绝不是黑底白字 / charcoal 底**。
16. 强调色为 **GitHub 蓝**，仅用于主按钮 / 链接 / focus，**不铺满选中行、不铺大面积**。
17. 提供**浅色（GitHub 浅色）与深色（GitHub 深色）两套**完整主题（深色为本版新增），二者均能正确呈现、无缺色 / 错位。
18. 主题在**首页、主页对话页、以及所有页面**统一适配（同一套主题来源 `codePilot`），不存在某页面仍停留在旧配色。
19. 主题包含状态色：成功绿 / 警告橙 / 错误红 / 信息蓝；内容卡片大圆角（约 16px）、实色不透明 + 扁平 + 1px 边框（不玻璃化），呈「浮动卡片」排布。

**分阶段（本版只立骨架，细节确标后续）**
20. 本版交付范围**仅为**统一顶栏 + 4 部分容器骨架 + 3 个独立开关 + 三侧区拖拽改宽（min / max 已定）+ 主题全局适配；4 个区的**内部细节**（对话树语义 / Composer / 多页签 / 文件树内容 / popover / 命令面板 / **拖拽视觉效果态** / 放大缩小 / 主题切换入口 / 手机态）在文档中**均明确标注「后续」**，不在本版实现。
21. home-shell 定义的 Helm 业务能力（左栏对话树、主机切换器、右栏多页签、Composer、命令面板、环境信息 / 打开位置等）在文档中被**显式标注为「保留 · 容器位预留 · 细节后续」**，无任何一项被声明丢弃。
22. 文档明确本版**桌面 only**，紧凑 / 手机态标「后续由桌面统一适配」。

**屏对齐**
