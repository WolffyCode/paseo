# 需求 · 首次启动 onboarding（连主机）

> 日期：2026-06-24 · 状态：需求+UI 已批（闸 1 过）→ 待 P2 架构 · 配套 UI：[ui.html](./ui.html) · 架构：architecture.md（P2 产出）· canonical：docs/helm/product.md
> **只写 WHAT**（见 [standards.md](../../standards.md) §7）。本文不含任何实现细节（代码/文件/函数名）——那是架构与开发的事。

## 1. 目标 & 用户价值

定义 Helm 的「软件首次打开」体验。用户第一次打开 Helm 时，应当**几乎无摩擦地连上本机的 AI 编码代理并开始用**，同时保留连接远程主机的能力。
- 用户为什么需要：新用户打开就想干活，不该被一堆「选连接方式」的步骤挡住；但 Helm 又必须支持连远程主机（手机遥控本机、连别的机器），所以不能只做本地。

## 2. 范围

**做（本需求）**
- 一次性品牌欢迎页（仅真·首次启动出现一次）。
- 桌面端「检测到本地 daemon 即自动连接、直接进主页」的混合首跑。
- 连接中过场、连接失败错误态。
- 连接方式选择器（本地不可用或用户主动连远程时）：直连主机 / 粘贴配对链接 /（手机）扫码。
- 平台差异：桌面 / Web 浏览器 / 手机。

**明确不做（划给后续需求）**
- 提供方 / 中转站 / 模型的配置（连上主机落到主页后，由主页空态引导，属另一个需求）。
- 多主机管理、主机切换器、主机详情（已有/另议）。
- relay 配对协议本身的细节（复用既有能力，不在本需求重做）。
- 主页（落地后的对话主界面）本身的设计。

## 3. 流程（端到端）

- **首装首跑**：欢迎页（一次性）→ 用户点「开始使用」→ 连接中过场 → ①连接成功：进入主页空态；②本地连不上：进入错误态。
- **后续启动**：不再显示欢迎页 → 静默自动连本地 → 直接进入主页。
- **连远程**：在欢迎页点「连接远程主机」，或在错误态点「改用其它连接方式」→ 进入连接方式选择器 → 选 直连/粘链/扫码 → 各自流程完成后连上 → 进入主页。
- **失败重试**：错误态点「重试」→ 回到连接中过场重连。
- 「已看过欢迎」是一次性状态：看过即记住，之后启动不再展示欢迎页。

## 4. 屏与全状态（逐屏对应 [ui.html](./ui.html)）

- **S1 欢迎页（桌面·一次性）** `ui.html#s1`：logo + 「欢迎使用 Helm」+ slogan + 主 CTA「开始使用」+ 次「连接远程主机」+ 版本号。**无设置入口**。
- **S1·态** `ui.html#s1b`：开始使用 / 连接远程链接 的 默认 / hover / pressed / focus 四态。
- **S2 连接中** `ui.html#s2`：spinner +「正在连接本地 daemon…」+ 取消；成功→主页，失败→S4。
- **S3 连接方式选择器** `ui.html#s3`：直连主机 / 粘贴配对链接 / 扫码（手机优先）三方式 + 重试本地。无设置入口。
- **S4 错误态** `ui.html#s4`：错误说明（daemon 未运行 / 端口被占 / 超时）+ 重试 / 改用其它连接方式 / 查看诊断。
- **S5 平台变体** `ui.html#s5`：Web 浏览器（无本地→直接选择器、无扫码）；手机（扫码优先）。
- 每屏的完整交互态（默认/hover/活跃/运行/禁用/空/加载/错误/离线/右键/点击效果）以 ui.html 注解为准，实现需逐态对齐。

## 5. 边界与依赖

- 依赖「连接本机/远程主机」的既有能力（本地 daemon 连接、直连 host:port、粘贴配对链接、扫码配对、relay 端到端）——复用，不重造。
- 平台差异：仅桌面端存在「本地 daemon 自动连」主路径；Web 浏览器与手机无本地 daemon，首跑直接走连接方式选择器（手机以扫码为主）。
- 「诊断」入口在本需求中**仅经 S4 错误态的「查看诊断」可达**（首页/选择器不放设置）。

## 6. 验收标准（逐条可判定）

1. 全新安装首次启动，显示一次品牌欢迎页（logo + slogan + 开始使用 + 连接远程主机 + 版本，**无设置**）。
2. 桌面端点「开始使用」后出现连接中过场（spinner + 正在连接本地 daemon）。
3. 本地连接成功后**自动进入主页**，不停留在欢迎/过场。
4. 之后每次启动**不再显示欢迎页**，静默自动连本地直接进主页。
5. 点「连接远程主机」进入连接方式选择器，含 直连/粘链/扫码 三方式 + 重试本地。
6. 本地连接失败时显示错误态，含 重试 / 改用其它连接方式 / 查看诊断 三操作，点击各自生效。
7. Web 浏览器首跑：开始使用直接进连接方式选择器，且不出现扫码项。
8. 手机端首跑：连接方式以扫码为主 CTA。
9. 首页（S1）与连接选择器（S3）均**无设置入口**。
10. 每屏交互态与 ui.html 一致（抽查 hover / 活跃 / 加载 / 错误 至少各一处）。
