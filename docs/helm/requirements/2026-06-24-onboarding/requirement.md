# 需求 · 首次启动 onboarding（连主机）

> 日期：2026-06-24（**2026-06-25 修订一：web 也先自动连本机 daemon；修订二：采用「延后连接」(Q)——看过欢迎前不发起任何连接；补 重开自愈 / 显示当前主机 两条边界**） · 状态：**修订二 需求+UI 已过闸 1**；架构（Q）待闸 2 · 配套 UI：[ui.html](./ui.html)（修订二已同步：延后连接序列、当前主机由主文案、取消→picker）· 架构：[architecture.md](./architecture.md)（Q 延后连接）· canonical：docs/helm/product.md
>
> **无老用户兼容**：Helm 是未上线的 fork，没有老用户，**不做任何老用户/迁移/存量兼容逻辑**。「有保存主机就直接进主页、不显欢迎」由启动路由（host 优先级在欢迎闸之前）天然保证。
> **只写 WHAT**（见 [standards.md](../../standards.md) §7）。本文不含任何实现细节（代码/文件/函数名）——那是架构与开发的事。

## 1. 目标 & 用户价值

定义 Helm 的「软件首次打开」体验。用户第一次打开 Helm 时，应当**几乎无摩擦地连上本机的 AI 编码代理并开始用**，同时保留连接远程主机的能力。

- 用户为什么需要：新用户打开就想干活，不该被一堆「选连接方式」「手输 host」的步骤挡住；但 Helm 又必须支持连远程主机（手机遥控本机、连别的机器），所以不能只做本地。
- **「先自动连本机」对所有有本机 daemon 的入口一视同仁**——不论用户从桌面应用还是从本机浏览器打开 Helm，第一动作都应是「自动找本机的 daemon 并连上、直接进主页」，全程不手输 host。只有当本机确实没有可连的 daemon（如远程浏览器、或本机没装/没起 daemon）时，才请用户选连接方式。

## 2. 范围

**做（本需求）**

- 一次性品牌欢迎页（仅真·首次启动出现一次，桌面与浏览器首访都过一次）。
- **「检测到本机 daemon 即自动连接、直接进主页」的混合首跑——桌面与 Web 浏览器同样适用**（桌面连内建 daemon，浏览器连本机 daemon）。
- 连接中过场、连接失败错误态/兜底。
- 连接方式选择器（**本机无可连 daemon 时的兜底**，或用户主动连远程时）：直连主机 / 粘贴配对链接 /（手机）扫码。
- 平台差异：桌面 / Web 浏览器 / 手机。

**明确不做（划给后续需求）**

- 提供方 / 中转站 / 模型的配置（连上主机落到主页后，由主页空态引导，属另一个需求）。
- 多主机管理、主机切换器、主机详情（已有/另议）。
- relay 配对协议本身的细节（复用既有能力，不在本需求重做）。
- 主页（落地后的对话主界面）本身的设计。

## 3. 流程（端到端）

- **延后连接（Q，核心）**：**用户看过欢迎并点「开始使用」之前，App 不发起任何主机连接**——欢迎页只展示品牌，不在后台偷偷连。点「开始使用」才触发「连接本机」。
- **首装首跑（桌面或本机浏览器，本机有 daemon）**：欢迎页（一次性）→ 用户点「开始使用」→ 连接中过场（**这时才**自动找本机 daemon，显示当前主机标识+状态）→ ①连接成功：进入主页空态；②本机连不上：桌面进入错误态、浏览器/手机落连接方式选择器兜底。**全程不手输 host。**
- **本机无可连 daemon（远程浏览器 / 本机没装没起 daemon / 手机）**：「开始使用」→ 尝试自动连 → 连不上 → 落连接方式选择器（兜底，显示「本机 daemon · 连接失败」上下文）。
- **后续启动（已看过欢迎）**：不再显示欢迎页 → **静默重试连本机一次** → ①连上：直接进入主页；②仍连不上：落连接方式选择器（桌面落错误态），并显示当前主机标识+状态。即「重开自愈」：daemon 起来了就直接进主页，没起来就回到上次那个界面。
- **连远程（主动）**：在欢迎页点「连接远程主机」，或在错误态/兜底点「改用其它连接方式」→ 进入连接方式选择器 → 选 直连/粘链/扫码 → 各自流程完成后连上 → 进入主页。**「连接远程」是会话级意图，不跨重启记忆**：重开仍先静默重试本机。
- **失败重试**：错误态/兜底点「重试」、或选择器点「重试本地」→ 回到连接中过场重连本机。
- **连接中取消**：连接中过场点「取消」→ 落连接方式选择器。
- 「已看过欢迎」是一次性持久状态：看过即记住，之后启动不再展示欢迎页。

## 4. 屏与全状态（逐屏对应 [ui.html](./ui.html)）

- **S1 欢迎页（一次性·桌面与浏览器首访都过）** `ui.html#s1`：logo + 「欢迎使用 Helm」+ slogan + 主 CTA「开始使用」+ 次「连接远程主机」+ 版本号。**无设置入口**。
- **S1·态** `ui.html#s1b`：开始使用 / 连接远程链接 的 默认 / hover / pressed / focus 四态。
- **S2 连接中** `ui.html#s2`：spinner +「正在连接本机 daemon…」（**此文案即当前主机+状态，不显具体端口**）+ 取消；成功→主页，失败→S4（桌面）或 S3 兜底（浏览器/手机）。**点「开始使用」在桌面与本机浏览器都进 S2**。
- **S3 连接方式选择器** `ui.html#s3`：直连主机 / 粘贴配对链接 / 扫码（手机优先）三方式 + 重试本地。**副标题「没检测到本地 daemon，或连接到其它主机。」即体现当前本机状态（不显端口）**。无设置入口。**这是兜底/远程入口，不是浏览器的默认首屏。**
- **S4 错误态（桌面）** `ui.html#s4`：错误说明 Alert「无法连接本机 daemon」（**即当前主机+失败状态**）+ 原因（daemon 未运行 / 端口被占 / 超时，**合并文案不显具体端口**）+ 重试 / 改用其它连接方式 / 查看诊断。
- **S5 平台变体** `ui.html#s5`：**Web 浏览器先自动连本机 daemon（同桌面走 S2），连不上才落选择器、无扫码**；手机无本机 daemon→自动连失败兜底到选择器、扫码优先。
- 每屏的完整交互态（默认/hover/活跃/运行/禁用/空/加载/错误/离线/右键/点击效果）以 ui.html 注解为准，实现需逐态对齐。

## 5. 边界与依赖

- 依赖「连接本机/远程主机」的既有能力（本机 daemon 自动探测与连接、直连 host:port、粘贴配对链接、扫码配对、relay 端到端）——复用，不重造。
- 平台差异：**「自动连本机 daemon」对桌面与 Web 浏览器都适用**（桌面经内建 daemon，浏览器经本机 daemon）；区别只在 transport 与连不上时的兜底（桌面→错误态可重启内建 daemon；浏览器/手机→连接方式选择器）。**不得把「浏览器」等同于「无本地、直接选择器」**——只有「本机确实无可连 daemon」（远程浏览器/未装未起/手机）才落选择器。
- 「诊断」入口在本需求中**仅经 S4 错误态的「查看诊断」可达**（首页/选择器不放设置）。

## 6. 验收标准（逐条可判定）

1. 全新安装首次启动（桌面或本机浏览器），显示一次品牌欢迎页（logo + slogan + 开始使用 + 连接远程主机 + 版本，**无设置**）。
2. 点「开始使用」后出现连接中过场（spinner + 正在连接本机 daemon，**不显具体端口**）——**桌面与本机浏览器都如此**。
3. 本机连接成功后**自动进入主页**，不停留在欢迎/过场，**全程不手输 host**。
4. 之后每次启动**不再显示欢迎页**，静默自动连本机直接进主页（桌面与本机浏览器一致）。
5. 点「连接远程主机」进入连接方式选择器，含 直连/粘链/扫码 三方式 + 重试本地。
6. 桌面本机连接失败时显示错误态，含 重试 / 改用其它连接方式 / 查看诊断 三操作，点击各自生效。
7. **Web 浏览器首跑且本机有 daemon：点开始使用进连接中过场 → 自动连上本机 daemon → 进主页**（与桌面一致，不停在选择器、不手输 host）。
   7b. **Web 浏览器本机无可连 daemon（远程浏览器/未装未起）：点开始使用尝试自动连 → 连不上 → 落连接方式选择器（兜底，无扫码项）+ 重试本地**。
8. 手机端首跑（无本机 daemon 可达）：自动连失败后落连接方式选择器，扫码为主 CTA。
9. 首页（S1）与连接选择器（S3）均**无设置入口**。
10. 每屏交互态与 ui.html 一致（抽查 hover / 活跃 / 加载 / 错误 至少各一处）。
11. 任一屏/任一平台**全程不要求用户手动输入 host:port 才能连本机**——本机有 daemon 时自动连上。
12. **延后连接（Q）**：看过欢迎并点「开始使用」之前，App **不发起任何主机连接**（欢迎页停留期间无连接中、无后台连接、不会因 host 突然上线而被弹走）。
13. **重开自愈边界**：首跑「开始使用」→ 本机连不上 → 落 picker；关闭 App 再打开 → **静默重试本机一次** → ①daemon 已起：直接进主页；②仍连不上：回到 picker（桌面回错误态），**不重弹欢迎页**。
14. **显示当前主机（由主文案体现）**：连接中 / 错误态 / picker 的**主文案本身**写明当前主机标识（本机 = 「本机 daemon」、远程 = 其名称）+ 状态（连接中 / 连接失败），**不显具体端口**；**不另设与主文案重复的状态元素**（如顶部状态药丸 chip）。
