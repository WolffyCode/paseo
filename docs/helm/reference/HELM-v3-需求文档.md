# Helm v3 · 产品级需求文档（架构 / 排期基线）

> **取代 v1/v2**（v1=网络调研、v2=cc-switch 源码校准，均覆盖不全）。v3 重新以 **Paseo 真实产品 + Codex 真实外壳 + cc-switch 管理** 三个真相源为基，做**产品级、覆盖每个功能**的需求与设计，作为后续架构方向与排期的依据。
>
> **配套设计稿**：`HELM-v3-设计稿.html`（分阶段交付，直接浏览器打开）。**设计稿即需求**：每屏的布局/状态/交互以设计稿为准，本文定 IA、功能边界、排期。
>
> **平台策略**：**一套响应式 UI**（桌面优先，紧凑/手机态自适应；若后续证明无法一套，再单独设计手机端）。本版优先保证**桌面客户端**。

---

## 0. 三条不可丢的底线（前三版翻车根因）

1. **不丢 Paseo 主机机制**：多主机连接（本机/远程 daemon）、主机切换器、添加主机（本地/手动/配对）、连接状态、守护进程管理。
2. **不丢手机端远程控制**：手机通过 **配对（QR/链接 `#offer=`）→ relay 端到端加密** 远程连接并操控本机 agent。
3. **外壳严格对齐 Codex**：左对话历史 / 中单对话 / **右侧真正的停靠面板**（审查·diff / 终端 / 浏览器 / 文件，终端可右/底停靠），从右上角 chrome 开合——**不是悬浮图标 rail**。

---

## 1. 产品定位

Helm = **桌面端（+手机远程）AI 编码终端聚合体**。把本机/远程多个 CLI 代理（Claude Code / Codex / OpenCode / Gemini…）统一到一个 **Codex 式对话外壳**里管理；内置 **三层选型**（提供方→中转站→模型）与 **cc-switch 级驾驶舱**（中转站/用量/代理/MCP/Skill/插件/Prompts）；保留 Paseo 的**多主机 + 手机远程**。运行时与生产 Paseo 隔离（`~/.helm` / 端口 `7070` / `helm://`）。

## 4. Phase 1 详细需求（外壳 + 全交互细节 + 设置 IA 骨架 + 响应式）

> 本阶段交付 `HELM-v3-设计稿.html` 的外壳与交互全集，先评审定调。

- **FR1.1 左侧栏**：顶部主机切换器（`● 主机名 ▾` + 在线状态色）；新对话(`SquarePen`)/搜索(`Search`)；**目录(项目)→对话** 两段（目录可展开/折叠，目录下挂对话；无目录的临时对话单列）；会话历史；设置(左下)。
- **FR1.2 主机切换器**：点开下拉 = 主机列表(在线/连接中/离线 状态点) + 「添加主机」入口。
- **FR1.3 中区·空态**：居中标题 + 居中 Composer（进去就能打字）；Composer 下方一行三层级联选择器。
- **FR1.4 中区·对话**：消息流（用户/代理/工具调用/diff/计划），复用 Paseo agent-stream。
- **FR1.5 Composer 全态**：输入框内 模型/访问/附件；框下 项目/mode/branch chips；**model-driven** 功能按钮随能力显隐；slash 菜单、@ 提及、附件盘、队列轨、语音、模式 全态。
- **FR1.6 右面板（Codex 停靠，修 rail）**：收起时仅右上角 chrome 一个开关（无悬浮 rail）；展开为停靠列，**顶部 tab 头**（审查/终端/浏览器/文件）+ 关闭；可拖拽改宽；**终端支持右侧或底部停靠**；审查支持逐行评论 + chunk/文件 stage/revert（目录绑定面，同 cwd 共享）。
- **FR1.7 顶部 chrome**：左栏开关（交通灯↔←→）/ 布局 / 右面板开关 / 全屏；不占整行 topbar。
- **FR1.8 ⌘K 命令面板**：全局搜索 + 命令（新对话/切主机/打开设置/切中转站/跳目录…）。
- **FR1.9 搜索浮层**：跨对话/目录搜索。
- **FR1.10 侧栏交互细节**：目录行 hover→[+新对话][⋮]；对话行 hover→[⋮]；**右键上下文菜单**（对话：重命名/归档/复制/在Finder显示/删除；目录：新对话/重命名/移除/在Finder显示）；内联重命名；拖拽排序；选中(surface2)/运行态(状态点)。
- **FR1.11 状态态**：空对话、加载(inline spinner/page spinner)、错误(Alert/Toast)、**离线/连接断开**（侧栏 callout + 主机状态红点 + 重连）。
- **FR1.12 设置 IA 骨架**：两级导航全貌（App 级 8 节 + 主机级 7 节），打开「通用」示例；其余节为 Phase 2/3 内容占位。
- **FR1.13 响应式·手机态**：紧凑时 左栏 overlay、右面板转**全屏 tab**、Composer 贴底、顶部精简、配对入口；同一套设计稿标注桌面/紧凑两态。

## 7. 设计规范 / 平台门 / 协议兼容

- **视觉**：Paseo dark（`theme.ts`，绿 accent `#20744A`），严格遵循 `docs/design.md`（字重>字号、accent 每屏≤1 CTA、句首大写无尾点、card/row、List+Detail、原语复用 Button/StatusBadge/DropdownMenu/Combobox/AdaptiveModalSheet/ContextMenu/confirmDialog）。
- **平台门**（`@/constants/platform`）：`isWeb`/`isNative`/`getIsElectron()`/`useIsCompactFormFactor()`；一套 UI 用断点做布局分支，不用 Platform.OS 当布局代理；hover 用 `isHovered||isNative||isCompact`。
- **协议**：后向兼容；新字段 `.optional()`；新能力走 `server_info.features.*` 能力门 + `COMPAT()` 注释；新 RPC 点号命名空间。
- **MVVM**：UI 由 view-model 驱动（如 Composer 能力 = `(提供方,中转站,模型)` 纯函数）。

## 8. 架构方向 / 排期建议

- **M1（外壳+交互）**：Phase 1 设计 → 实现外壳重构（沿用 Paseo workspace/sidebar/composer，折叠为目录→对话+单对话+Codex 右面板）+ 主机切换 + ⌘K + 交互细节。
- **M2（三层+管理）**：vendor 一等实体 + 级联选择器 + 中转站/模型/用量/代理/MCP/Skill/Prompts/插件 + cc-switch 同步（Phase 2 设计）。
- **M3（手机+远程+打磨）**：手机响应式全套 + 配对/relay 打通 + 项目/会话 + onboarding（Phase 3 设计）。
- 每阶段：设计评审 → per-feature spec(`docs/specs/`) → 实现 → 真机像素核对 + 端到端功能验证（[[verify-functional-not-just-rendered]] 教训：不靠 text-grep）。
