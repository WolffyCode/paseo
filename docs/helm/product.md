## 0. 产品定位

Helm 是移动端、Web 与桌面端的 AI 编码代理控制台：连接本机或远程主机上的代理运行环境，让用户从一个界面监控、进入和控制工作。代码留在用户自己的机器上。

## 1. 全局信息架构（IA）

- 首次启动：一次性 onboarding，负责把用户连接到一个可用 host。
- 连接后：进入 host 主页与 workspace/agent 工作界面。
- 诊断与配置：通过 settings 进入；首次启动流程只在本地连接失败时暴露诊断入口。

## 2. 入口与导航

- `/welcome` 是首次启动 onboarding 路由。路由名保留为 welcome，但承载完整 onboarding 状态机。
- `/` 负责启动落点：**已看过欢迎且有可用 host 时进入主页；全新用户先进 onboarding 欢迎页（看过欢迎前不自动连接）；没有可落地 host 且需要连接时进入 onboarding**。
- `/pair-scan?source=onboarding` 是 onboarding 扫码连接入口。
- `/settings` 是本地 daemon 连接失败后的诊断入口。

### 首次启动 onboarding（连主机）

- 作用：让首次打开 Helm 的用户先看一次品牌欢迎，然后尽快连上本机 daemon（桌面与本机浏览器同样自动连），并保留连接远程主机的能力。
- 入口：全新安装/清空本地状态后启动；无 host 的启动路径；用户在欢迎页选择“连接远程主机”；本地连接失败后选择其它连接方式。
- 流程（延后连接 Q）：**看过欢迎前不发起任何连接**。首启显示欢迎页 →“开始使用”才触发连接本机 daemon（连接中显示当前主机标识+状态）→ 成功进主页；本机连不上：桌面进错误态、Web/手机落连接方式选择器。“连接远程主机”进连接方式选择器；错误态可重试 / 改用其它方式 / 查看诊断。**后续启动不再显示欢迎页，静默重试本机一次：连上进主页，仍连不上落选择器/错误态（重开自愈）。**
- 边界：不配置 provider、relay 协议或模型；直连、粘贴配对链接、扫码复用既有能力；**“自动连本机”对桌面与本机浏览器一视同仁（桌面经内建 daemon、浏览器经 localhost daemon）；只有真无本地可连 daemon（远程浏览器/未装未起/手机）才落选择器**，Web 选择器不显示扫码，手机扫码优先。

## 4. 核心流程

### 首次连接 host

1. 首装首跑显示一次欢迎页，包含 Helm 标识、slogan、开始使用、连接远程主机和版本号。**看过欢迎前不发起任何连接。**
2. 点“开始使用”后进入连接中过场，**这时才**检测并连接本机 daemon——**桌面（内建 daemon）与本机浏览器（localhost daemon）一致**；过场显示当前主机标识+状态（不显端口）。
3. 本机连接成功后由启动路由进入主页，不停留在欢迎或过场，全程不手输 host。
4. 本机连接失败：**桌面**显示错误态（重试 / 改用其它连接方式 / 查看诊断）；**Web/手机**落连接方式选择器兜底（含“重试本地”，显示当前主机失败上下文）。
5. 用户选择远程连接（欢迎页“连接远程主机”或失败后“改用其它连接方式”）进入连接方式选择器，使用直连 host、粘贴配对链接或扫码完成连接。

### 后续启动与重开自愈

1. 已看过欢迎后，启动不再显示欢迎页。
2. 启动静默重试本机一次：daemon 已起 → 直接进主页；仍连不上 → 落连接方式选择器（桌面落错误态），显示当前主机标识+状态。
3. “连接远程”是会话级意图，不跨重启记忆——重开仍先静默重试本机。

## 5. 全局状态与规则

- “已看过欢迎”是本地一次性持久状态；看过后后续启动不再展示欢迎页。
- **看过欢迎前不发起任何主机连接（延后连接 Q）**：连接由“开始使用”触发，或在已看过欢迎的后续启动时静默发起（重试本机一次）。
- **“自动连本机 daemon”对桌面与本机浏览器一视同仁**：桌面经 Electron 内建 daemon 的 directPipe/socket，浏览器经 localhost daemon 的 WebSocket；区别只在 transport 与连不上时的兜底（桌面→错误态、浏览器/手机→连接方式选择器）。只有真无本地可连 daemon 才落选择器。
- 连接中 / 错误态 / 选择器**通过各屏主文案**体现当前主机标识（本机 daemon / 远程主机名）+ 状态，不显具体端口；不另设与主文案重复的状态元素。
- 新 host 连上后的主页落点由启动路由统一决定，onboarding 只渲染阶段并派发动作；onboarding 屏自身不做“跳主页”。
- 本需求不修改协议、RPC 或 `server_info.features.*`。

## 6. 术语表

- Host：Helm 连接的本机或远程 daemon 环境。
- Daemon：本机或远程运行的 Paseo/Helm 后端进程。
- Onboarding：首次启动连接 host 的一次性引导流程。
- 连接方式选择器：直连主机、粘贴配对链接、扫码连接的选择界面。
